Microsoft Update Management (MUM) Section Date Parameters

Definitions

Scheduler - Lets you define the period covered by a section. If it is checked, the section will use any schedule(s) you assign to a report containing the section. If the "Scheduler" configuration option is unchecked, you will be able to define the coverage period for the date type (Detect, Download, Install, Removal, Error, Last Change, Next Action) whose "Scheduler" configuration option you unchecked.

Restrict - Lets you restrict the coverage period defined to a single date type (Detect, Download, Install, Removal, Error, Last Change, Next Action). For example, suppose you check the "Restrict" configuration parameter for the “install date” date type parameter. The section will report only information about Microsoft software updates whose installation dates match the dates defined in any schedule assigned to reports using the section, or the date range entered via the start and end dates, if the "Scheduler" configuration parameter is disabled.

Consider the following usage of the restrict configuration parameter for the “install date” date type:

  • When defining or editing a section, you check "Restrict" for the “install date” type. If "Scheduler" is checked, a report using this section will only include status entries with installation date matching the period of any schedule(s) attached to the report. For example, if you assign to a report a schedule that runs daily at 1 AM, the report will only include updates that were installed from yesterday at 1 AM to the current time (1 AM today).

  • When defining or editing a section, you check "Restrict" for the “install date” type. If "Scheduler" is not checked, a report using this section will only include status entries with installation date matching the period specified. For example, you could enter a start date of 1/1/07 and an end date of 2/1/07 to get a list of updates installed in January 2007.

You can enable the "Restrict" configuration option for only one date type (Detect, Download, Install, Removal, Error, Last Change, Next Action) at a time. When you do, all other "Scheduler" and "Restrict" configuration options are disabled.

IMPORTANT NOTE

The “Status” configuration parameter in the “Add/Copy/Edit a Section” pages, lets you retrieve MUM update records by their current status on a machine-by-machine basis.

The “date type” configuration parameters in the “Add/Copy/Edit a Section” pages (Detect, Download, Install, Removal, Error, Last Change, Next Action), let you retrieve MUM update records based on the occurrence of update activity of a certain type.

You should be careful about configuring sections where you select values for both the “Status” and “date type” configuration parameters. Most combinations of “Status” and “date type” configuration parameter values result in no MUM update records retrieved.

Example 1 – Restricted section with a defined coverage period. Suppose you defined a section as follows:

  • Status – Detected

  • Install Date – “Restrict” checked, “Scheduler” unchecked, Start Date – 03/01/07, End Date – 4/1/07

A section defined in this way will retrieve all updates which have a detected state and installation activity in March 2007, regardless of the schedule assigned to a report using this section. This could happen if the disk on a system was replaced, or one or more updates were removed.

Example 2 – Restricted section with a periodic coverage period. Suppose you defined a section as follows:

  • Status – Installed

  • Detect Date – “Restrict” checked, “Scheduler” checked

A section defined in this way will retrieve all updates which have installed status and detect activity during the period covered by the schedule assigned to the report.

Example 3 – Status section. Suppose you defined a section as follows:

  • Status – Installed

  • Date types – All scheduler configuration options checked

When you use this section in a report, the report will include all MUM update records with installed status, which had MUM related activity during the period covered by the report. Please note that if no updates with installed status had any MUM related activity during the period covered by the report, the report will be empty.

Notes

If you do not want to specify ranges for date configuration parameters on this "Edit a Microsoft Update Management Section" page, leave all "Schedules" configuration parameter boxes checked. In this way, when this section is included in a report, it will use the schedule(s) assigned to the report.

Please remember that you can enable the "Restrict" configuration option for only one date type at a time. When you do all other "Scheduler" and "Restrict" configuration parameters will be grayed out.

If you disable the "Scheduler" configuration option for more than one date type, the period covered by the section will be a period including both coverage periods entered in the start and end dates for the date types whose "Scheduler" configuration parameter was unchecked.

For example, suppose you disable the "Scheduler" configuration option for "detect" and "install" date types, and enter: detected date interval -- Start date - 1/1/07 and End date - 2/1/07 install date interval -- Start date - 1/15/07 and End date - 2/15/07

Also assume that the "Restrict" configuration option is not enabled for either "detect" or "install" date types, then a report including a section with date types configured as described above, will include those MUM status entries where the detect date is between 1/1/07 and 2/1/07 OR the install date is between 1/15/07 and 2/15/07 (the OR is important). MUM status entries not matching either date range will not be included.

If you want to specify a coverage period for any of the date configuration parameters on the "Edit a Microsoft Update Management Section" page, you should uncheck the "Schedules" box for the corresponding date configuration parameter, and enter the start and end dates for the desired coverage period. This entry will override for this section, any schedule assigned to a report including this section.

If you want a section to report only MUM events with a date (e.g. installation date) falling within a certain period of time, you should uncheck the "Scheduler" box for the corresponding date configuration parameter, enter the start and end dates for the desired coverage period, and check the "Restrict" configuration parameter box. This entry will override for this section, any schedule assigned to a report including this section, and all other date configuration parameter settings on the "Edit a Microsoft Update Management Section" page will be ignored.

Examples

To clarify the above description of the date type related configuration parameters on the "Edit a Microsoft Update Management Section" page, before you configure any of the date parameters in this "Edit a Microsoft Update Management Section" page, please review the configuration examples listed below.

On-demand report for updates with installation date occurring during a period of time
- Uncheck the "Scheduler" box for "Install date",
- Check the "Restrict" box for "Install date", and
- Enter period start date in the install "Start Date" box, and period end date in the install "End Date" box

On-demand report covering MUM activity occurring during a period of time
- Do not change the date parameters default configuration on the "Add/Copy/Edit a Microsoft Update Management Section" pages.
- Assign to the report using this section a one-time schedule covering the desired period

Periodic report for updates with installation date occurring during the last week (rolling)
- Check the "Restrict" box for "Install date"
- Assign to the report using this section a weekly periodic schedule.

Periodic report for activity occurring during the last week (rolling)
- Do not change the date parameters default configuration on the "Add/Copy/Edit a Microsoft Update Management Section" pages.
- Assign to the report using this section a weekly periodic schedule.

On-demand report for all MUM status (entire table)
- Do not change the date parameters default configuration on the "Add/Copy/Edit a Microsoft Update Management Section" pages.
- In the one-time schedule you assign to the report using this section, leave start date and end date boxes blank.

Periodic report for all MUM status (entire table)
- Uncheck all "Scheduler" boxes for all date parameters on the "Add/Copy/Edit a Microsoft Update Management Section" pages.
- Leave all start and end dates on this "Edit a Microsoft Update Management Section" page blank.
