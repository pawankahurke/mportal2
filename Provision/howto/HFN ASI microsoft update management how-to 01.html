How to enable Microsoft Update Management (MUM) átop

1) Log onto the ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) Go to the MUM wizard page (https://asiservername/patch/wu-confg.php). Please read the text on this page carefully.

4) To enable MUM on one system, a group of systems, a site, or all sites, please click on the "enable the Microsoft Update management Scrip (#237)" link

5) Follow the on-screen instructions

6) Once you complete execution of the MUM activation wizard (Step 5)), the Microsoft Update Management function will be enabled in automated mode on the systems you selected when running the wizard. This means that every morning at 3 AM, software updates will be installed for up to two hours, and systems will be re-started automatically if required by a software update.

7) MUM lets you decide if software updates should be installed automatically (as described in step 6)), or only when you approve them. You make this choice using the "Select update method" wizard on the MUM wizard page (https://asiservername/patch/wu-confg.php). Please note that by "manual update method" we mean that when you select this method, MS software updates will be installed only after you approve them using the "Approve updates" wizard also on the MUM wizard page (https://asiservername/patch/wu-confg.php).

How to change a previously taken MUM “Approve updates” action

1) Log onto your ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) Go to the MUM wizard page (https://asiservername/main/patch/wu-confg.php)

4) In the second paragraph on the MUM wizard page, click on the "See wizard actions taken to date" link. The "Microsoft

Update Wizard Status" will be displayed

5) Scroll down the "Microsoft Update Wizard Status" page to the "Status for:" section, and click on the "Approve updates" link

Identify the “Approve updates” action you want to modify. If you want to:

a) Change the composition of the group the “Approve updates” action was applied to

  • Click on the “Change machine selection.” link. The “Wizard - Edit Machine Group” page will be displayed in a new window

  • Click on one of the [select all], [deselect all], [select/de-select individual] links and follow the on-screen instructions

  • Click on the “OK” button, and close the “Wizard - Edit Machine Group” page.

  • Close the “Machine Group Details” page

How to retrieve Microsoft update status átop

In order to review the status of MS software updates at your sites:

1) Log onto the ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) Go to the MUM status page by clicking on the "satus" link in the "microsoft updates" navigation bar

(https://asiservername/main/patch/wu-stats.php)

4) In the "Search Options" panel, select "installed" from the "Status" search parameter pull-down list

5) Click on the "Search" button. Please note that, if necessary, you can adjust the number of items displayed on the page by changing the value of the "Page Size" display parameter.

6) When the results are displayed look at the dates in the "Install Date" column to determine when an update was installed on a system.

If you want to find out the status of a specific update at all sites, e.g. Security Update for Windows 2000 (KB912919) ( en 0.0.0 SP0.0):

a) Click on the "Reset" button,

b) In the "Search Options" panel, select the update whose status you want to reviewe, from the "Update" search parameter pull-down list.

c) Click on the "Search" button. Please note that, if necessary, you can adjust the number of items displayed on the page by changing the value of the "Page Size" display parameter.

Once you have the information you are looking for, if you want to determine why a certain update may not have been installed, or downloaded as you expected it to be, you should retrieve Scrip 237 (Microsoft Update Management) event logs for the date on which the last action was reported on the status page.

How to change a Microsoft Update Management (MUM) machine configuration átop

Do not make changes to the machine configuration for a group of machines (one or more machines, including built-in groups such as single machines and sites) unless you are confident about the change. If in doubt, please contact HandsFree Networks support.

1) Log onto the ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) Click on the "advanced" link in the "microsoft update" navigation bar

4) Click on the "config" link in the second-level navigation bar (the one with the sites, updates, and config links)

5) Click on the "Machine configuration" link in the body of the page (second paragraph)

6) Click on the "edit" link for the machine group whose configuration you want to edit.

7) Change the machine configuration parameters as desired. For example, scroll down to the section titled "Restart policy", click on the desired configuration setting radio button

8) Click on the OK button.

Please note that for systems that belong to more than one group (e.g. All and Site), the machine configuration of the higher priority group takes precedence.
