ASI Local Interface Activation and Use Guide

Overview

The local ASI interface lets you perform recurring system management and maintenance actions quickly and easily via a wizard based interface. Initially, the local interface lets you perform the following actions:

  • Enable or disable the ASI Microsoft Update Management facility (MUM) on the local system, or on all system on the sub-net

  • Enable or disable malware protection on the local system, or on all system on the sub-net

  • Activate remote control of the system you are accessing the local ASI interface on

Over time, we will add other system management and maintenance activities to mirror the system management and maintenance wizard based actions implemented on the ASI server.

When activating the local ASI user interface you can follow one of thre general strategies:

  • Activate the local ASI interface on one or more of the servers on a sub-net. Typically, system management personnel are the only ones that have access to servers and are the only ones that should have access to the local ASI interface

  • Enable the local interface on all systems at a site (across all sub-nets or on a sub-net by sub-net basis). This is the suggested approach if you want users at your site to be able to activate remote control sessions

  • Enable the local ASI interface on as needed basis. This is the most conservative approach, and is also the one that requires more effort.

Password protection

Access to the Enable / Disable Malware Protection, and Enable / Disable Microsoft Updates functions is password protected. The user id is the same user id you use for directly accessing the ASI client at a site. The default password is dilbert.

If you want to change the local ASI interface access password, perform the following steps:

  • Access the Scrip 43 (HandsFree Client Tools) configuration page on a system at the site where you want to change the local ASI interface access password either via the local interface (http://localhost:2721), or via your ASI server (https://yourasiserverurl/main/config/index.php?act=site)

  • Enter the new password in the Password for simple configuration interface configuration parameter, and in the corresponding Confirmation field. Depending on your needs keep the Password for simple configuration interface configuration parameter global, or change it to local.

  • Click on the Submit Changes button

Local ASI interface activation

You activate the local ASI interface by changing the Scrip 190 (Client as Service) configuration in one of two ways:

    Activation at site

  • At the site where you want to activate the local ASI interface, open a browser window on the system you want to activate the local ASI interface on. If you want to activate the local interface on all systems at a site you can use any system.

  • In the browser address box, type

  • http://localhost:2721

  • Enter the user id and password combination for directly accessing the ASI client

  • Click on the IP address-system name at the top of the page

  • Click on the entry for Scrip 190

  • If you want to enable the local ASI interface only on the system whose ASI client you logged, click on the local radio button to the left of the Enable taskbar icon configuration parameter, otherwise leave the Enable taskbar icon configuration parameter set to global

  • Click on the Enable taskbar icon configuration parameter check box

  • Click on the Submit Changes button

    Important note. Before clicking on the Submit Changes button, please check the content of the File or command for taskbar double-click configuration parameter in Scrip 190. Its default value should be ui.url. If this configuration parameter is empty, please enter ui.url in it.

    Please note that in the File or command for taskbar double-click configuration parameter you could enter the name of a different html file where you could give users access to functions and information of your choosing.

    Activation via ASI server

  • Alternatively, log onto your ASI server's site configuration page at https://yourasiserverurl/main/config/index.php?act=site

  • Click on the [view machines] link for the site where you want to activate the local ASI interface

  • If you want to activate the local interface only on one system, click on the [configure Scrips] link for that system, otherwise click on the [configure Scrips] link for any system

  • Click on the entry for Scrip 190

  • If you want to enable the local ASI interface only on the system whose ASI client you logged, click on the local radio button to the left of the Enable taskbar icon configuration parameter, otherwise leave the Enable taskbar icon configuration parameter set to global

  • Click on the Enable taskbar icon configuration parameter check box

  • Click on the Submit Changes button

    Important note. Before clicking on the Submit Changes button, please check the content of the File or command for taskbar double-click configuration parameter in Scrip 190. Its default value should be ui.url. If this configuration parameter is empty, please enter ui.url in it.

    Please note that in the File or command for taskbar double-click configuration parameter you could enter the name of a different html file where you could give users access to functions and information of your choosing.

Local ASI interface use

Performing system management via the local ASI interface is easy and fast.

Enabling – disabling Microsoft Update Management (MUM)

Enabling or disabling MUM is equivalent to enabling or disabling Scrip 237 (Microsoft Update Management). To do this using the local ASI interface, you should perform the following steps:

  • Double click on the ASI client icon in the system tray of a system where you activated the local ASI interface

  • Log onto the local interface with the user id you use to directly access the ASI client, and either the default password (dilbert) or the password you set

  • If you want to enable (or disable) MUM on all systems on the sub-net, click on the This site: link. To enable (or disable) MUM only on the system whose ASI client you logged onto, click on the This machine: link.

Enabling – disabling malware protection

Enabling or disabling malware protection is equivalent to enabling or disabling Scrips 27 (System start-up environment control), and 232 (Intrusion protection control). To do this using the local ASI interface, you should perform the following steps:

  • Double click on the ASI client icon in the system tray of a system where you activated the local ASI interface

  • Log onto the local interface with the user id and password you use to directly access the ASI client

  • If you want to enable (or disable) malware protection on all systems on the sub-net, click on the This site: link. To enable (or disable) malware protection only on the system whose ASI client you logged onto, click on the This machine: link.

Activating remote control

Activating remote control of a system is equivalent to executing Scrips 236 (On-demand Remote Control), or 245 (On-demand GoToAssist) on that system. To do this using the local ASI interface, you should perform the following steps:

  • Activate the local ASI interface on the system(s) where you want to activate remote control.

  • Double click on the ASI client icon in the system tray of a system where you activated the local ASI interface

  • Log onto the local interface with the user id and password you use to directly access the ASI client

  • To activate remote control click on the Start link for either Start Remote Control Access (Ultr@VNC):, or Start Remote Control Access (Citrix® GoToAssist):.

  • To stop remote control of a system, click on the Stop link for either Stop Remote Control Access (Ultr@VNC):, or Stop Remote Control Access (Citrix® GoToAssist):.
