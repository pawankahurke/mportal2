ASI Autotask Support Configuration and Use Guide

Introduction

When you configure and enable Autotask support in event notifications, every time a notification which you configured to be sent via e-mail is produced, it will be sent automatically to your Autotask account at Autotask where a trouble ticket will be automatically generated.

To support integration with Autotask, we have added the following parameters to the Add an Event Notification, Edit an Event Notification, and Copy an Event Notification pages:

  • Per-site e-mail notifications

Enabling this option will generate an e-mail notification message for each site event logs have been retrieved from.

  • Add e-mail footer

When this option is enabled, the e-mail footer entered in "E-mail footer" is appended to the end of the e-mail notifications.

  • E-mail footer

By default, the E-mail footer is pre-configured with the following entry:

<Autotask>

<ID name="%ticketuser%"/>

<PW name="%ticketpassword%"/>

<Customer name="%site%"/>

<SubIssue name="%name%"/>

</Autotask>

If you change the syntax of the above entry, or change or remove the two <Autotask> constants, the resulting event notification will not be successfully sent to your Autotask account.

You can replace the %ticketuser%, %ticketpassword%, %site%, and %name% environment variables in the the above entry for one or more notifications (using the edit multiple notifications function) with a constant. However, if you do that, you lose the benefit of having a single place where you enter and modify this information, and have it automatically used in all notifications configured to be sent to Autotask.

The entry in the E-mail footer event notification configuration parameter will be appended to the end of an e-mail notification when you enable the Add e-mail footer event notification configuration parameter. Note that the Per-site e-mail notifications event notification configuration parameter should also be enabled. The %site% variable corresponds to the site name as shown in the sites section of the admin page on your ASI server. The %name% variable corresponds to the name of the notification being sent. The %ticketuser% and %ticketpassword% variables take on the values entered for the ticket_user and ticket_password entered in the ASI server configuration page.

Before you start enabling Autotask support in existing event notifications on your ASI server, please read the Enabling Autotask support in existing notifications section of this document. It will save you a lot of tedious work.

Autotask support configuration

You can configure existing event notification with Autotask support in the Edit an Event Notification page (e.g. https://ASIserverURL/main/event/notify.php?nid=256&act=edit). You can configure new event notifications with Autotask support in the Add an Event Notification page (https://ASIserverURL/main/event/notify.php?act=addn), or you can add Autotask support when you copy an event notification via the Copy an Event Notification page (e.g., https://ASIserverURL/main/event/notify.php?nid=256&act=copy).

In all three pages, you will need to configure the same three event notification configuration parameters:

  • Per-site e-mail notifications

  • Add e-mail footer

  • E-mail footer

The Per-site e-mail notifications, and Add e-mail footer event notification configuration parameters are boxes that you simply need to check whenever you want to enable Autotask support. We strongly suggest that you leave the E-mail footer configuration parameter in its default configuration.

E-mail footer

To integrate ASI event notifications with Autotask, a special footer is automatically appended whenever a notification is produced. It is added via the E-mail footer event notification configuration parameter. This special footer contains four items:

  • ID name - Your Autotask user id, represented by the %ticketuser% environment variable in the default E-mail footer entry

  • PW name - Your Autotask password, represented by the %ticketpassword% environment variable in the default E-mail footer entry

  • Customer name – the name of the site, represented by the %site% environment variable in the default E-mail footer entry

  • SubIssue name – the name of the notification, represented by the %name% environment variable in the default E-mail footer entry

Using environment variables in the E-mail footer event notification configuration parameter default entry minimizes manual data entry.

For example, the name of a site is generated automatically the first time an ASI client on a system at the site posts an event on the ASI server. Using the %site% environment variable to represent the Customer name Autotask e-mail footer component means that no manual data entry is required when a site is added, or a site name is changed.

ID name and PW name

You can configure the Autotask user id and passwords on your ASI server configuration page as follows:

  1. Access the your ASI server configuration page at https://ASIserverURL/main/acct/server.php

  2. Scroll down to the ticket_password configuration parameter, enter your Autotask account password as supplied by Autotask

  3. Scroll down to the ticket_user configuration parameter, enter your Autotask user id as supplied by Autotask

  4. If ready, proceed to the Event notifications page at https://ASIserverURL/main/event/notify.php, otherwise you can close the page.

Customer name

The default value of Customer name entry in the E-mail footer event notification configuration variable is the name of the site which the event logs reported by the notification are retrieved from. The %site% environment variable is replaced automatically by the name of the site found in the Admin page at https://ASIserverURL/main/acct/admin.php. You should leave the default configuration of the Customer name entry unchanged. In this way, when you add sites, or change the name of an existing site, you will not have to change the E-mail footer event notification configuration parameter.

Please note that the name of a site used as the value for Customer name entry in the E-mail footer event notification configuration variable needs to be exactly the same as the name you use for the customer on the Autotask server. Otherwise, the notification will not be delivered.

Subissue name

The default value of the SubIssue name entry in the E-mail footer event notification configuration variable is the name of the notification as you entered it in the Name event notification configuration parameter. The %name% environment variable is replaced automatically by the name of the notification. You should leave the default configuration of the SubIssue name entry unchanged. In this way, when you add event notifications, or change the name of an existing notification, you will not have to change the E-mail footer event notification configuration parameter.

Enabling Autotask support in existing notifications

You can enable Autotask support in multiple event notifications with one editing action, as follows:

  1. Access the Event Notifications page on your ASI server at https://ASIserverURL/main/event/notify.php

  2. Select the notifications you want to add Autotask support to using the selection parameters in the Search Options panel. For example, if you want to select all enabled event notifications, simply select enabled from the State search parameter pull-down list.

Make sure that the Page Size parameter in the Display Options panel is large enough to display ALL selected event notifications.

  1. Click on the manage link at the top of the Event Notifications page

  2. Click on the Edit Multiple Notifications link in the Manage Event Notifications page

  3. Click on the Check all button, or click in the box next to the name of each notification you want to apply the change to.

  4. Select Yes from the E-mail footer pull-down list

  5. Select Yes from the E-mail per site pull-down list

  6. Modify the content of the E-mail footer event notification configuration parameter as desired

  7. When ready, click on the OK button at the bottom of the page.
