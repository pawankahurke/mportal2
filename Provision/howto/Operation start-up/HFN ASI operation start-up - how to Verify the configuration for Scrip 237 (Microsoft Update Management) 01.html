Verifying the Configuration for Scrip 237 (Microsoft Update Management)

You can use this procedure to verify that on all ASI clients at your sites Scrip 237 (Microsoft Update Management) is configured to minimize any potential CPU related performance issues caused by the Microsoft automated update software.

All systems

  1. Log onto the ASI server (https://asiservername/main ). The ASI dashboard will be displayed.

  2. Click on the Use extended interface link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. At the upper right-hand corner of any page on the ASI server extended interface, click on the wizard link in the sites navigation bar (https://asiservername/main/config/index.php?act=wiz).

  4. Click on the Scrip configuration link (https://asiservername/main/config/scrpconf.php?custom=8). The Scrip Configuration - Select a Group page will be displayed.

  5. Click on the All machines radio button. Then click on the Next button. The Scrip Configuration - Select Scrip page will be displayed.

  6. Scroll down the page, locate the entry for Scrip 237 (Microsoft Update Management), and click on its name. The Scrip 237 configuration page will be displayed.

  7. Locate the schedule configuration parameter In the Name column, find the row that contains schedule configuration parameters (Minute, Hour, etc.)

  8. In that row, look at the Value column. Either a Configure button will appear, or the current values will be listed. These are the correct values:

Minute (0-59):

0

Hour (0-23):

22

Day (1-31):

*

Month (1-12):

*

Weekday (0-6):

*

Random minute:

60

Type (1 or 2):

2

Fail limit:

0

    If the above values are displayed the settings are correct. If the Configure button appears, click on it and compare the values displayed with the above values. If they are the same, the settings are correct.

If different values are displayed, proceed to the next step.

  1. Check the Disable client startup offline scan configuration parameter. If you see a Configure button click on it. Once a check-box is visible, if it is not checked, click on it to enable the configuration parameter. Otherwise, the settings are correct.

  2. If the values of the schedule parameter do not match the ones displayed in step 8) above, change them so that they do.

  3. If you have made any changes, click the Next button and stop here. You will need to allow some time for machines to receive the updated configuration, depending on your settings for Scrip 177 (Scrip Configuration Update) and the number of machines to be configured.

Note

Please note that depending on the Internet browser you use on its configuration, and on the total number of ASI clients at your sites, after you execute a configuration change (by clicking on the “Next” button) affecting all systems (or a very large number of systems) at your sites, the browser may time out. If this happens, close the window, and wait for up to one hour before making any other Scrip configuration changes on all systems at your site. The Scrip configuration change you made will be processed correctly. The fact that the Internet browser timed out has no effect on the correct completion of the Scrip configuration change.



If you have not made changes, proceed with the following steps.

  1. On the Scrip 237 configuration page, click on the "Assigned values" link for the schedule configuration parameter. A new window will open displaying the schedule assigned to all systems at your sites.

  2. Check the schedule entries in the Value column. If they are like this:

0 22 * * * 60 2 0

then the schedule is correct for all machines, and you can close the window. If any entries are different, make a note of the sites and machines where the schedule values are different.

  1. Go back to the Scrip 237 configuration page. Click on the "Assigned values" link for the Disable client startup offline scan configuration parameter. A new window will open. The entries in the Value column should be "Enabled" in all cases. Make note of any sites and machines where this is not the case.

If you need to change the Scrip 237 configuration for a site, or for a single machine:

Site

  1. Log onto the ASI server (https://asiservername/main ). The ASI dashboard will be displayed.

  2. Click on the Use extended interface link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. At the upper right-hand corner of any page on the ASI server extended interface, click on the configuration link in the sites navigation bar (https://asiservername/main/config/index.php?act=site ). The Select a Site page will be displayed.

  4. Locate the entry for the site where you want to configure Scrip 237, and click on its [configure site] link. The Scrip Configuration - Select Scrip - Site <site name> page will be displayed.

  5. Scroll down the page, locate the entry for Scrip 237 (Microsoft Update Management), and click on its name. The Scrip 237 configuration page will be displayed.

  6. Change the values for the schedule and the Disable client startup offline scan configuration parameters as necessary, following steps 7)-11) in section “All systems” of this document.

System

  1. Log onto the ASI server (https://asiservername/main ). The ASI dashboard will be displayed.

  2. Click on the Use extended interface link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. At the upper right-hand corner of any page on the ASI server extended interface, click on the configuration link in the sites navigation bar (https://asiservername/main/config/index.php?act=site ). The Select a Site page will be displayed.

  4. Locate the entry for the site where the system whose Scrip 237 you want to configure is located, and click on its [view machines] link. The Configuration - <site name> Machines page will be displayed.

  5. Locate the entry for the system whose Scrip 237 you want to configure, and click on its [configure Scrips] link. The Scrip Configurator - Select Scrip - Site <site name> - Machine <machine name> page will be displayed.

  6. Scroll down the page, locate the entry for Scrip 237 (Microsoft Update Management), and click on its name. The Scrip 237 configuration page will be displayed.

  7. If you are changing the system's Scrip 237 configuration after you performed the steps in section “All systems” above, select “all machines” from the Group pull-down list for the schedule and the Disable client startup offline scan configuration parameters.

If you are changing the system's Scrip 237 configuration after you performed the steps in section “Site” above, select “site <site name>” from the Group pull-down list for the schedule and the Disable client startup offline scan configuration parameters.

If you are changing the system's Scrip 237 configuration before independently of the site it belongs to, or any other group:

  • Check the values of the schedule and the Disable client startup offline scan configuration parameters. If they match the values for these configuration parameters in section “All systems” of this document, stop here, and close the window.

  • If the values of the schedule and the Disable client startup offline scan configuration parameters do not match the values for these configuration parameters in section “All systems” of this document:

  • Select “machine <machine name>” from the Group pull-down list for the schedule and the Disable client startup offline scan configuration parameters.

  • Change the schedule to match the one listed in step 8) in section “All systems” of this document.

  • Enable the Disable client startup offline scan configuration parameter by clicking in its box

  1. Click on the “Submit Changes” button.

Note

Please note that you could use the Scrip configuration wizard to change the Scrip 237 configuration for a site or a system. If you wanted to use the Scrip configuration wizard, you would follow the procedure described in the “All systems” section of this document with one change. In step 5), instead of clicking on the “All machines” radio button, you would click either on the “A single site”, or the “A single machine” radio button.
