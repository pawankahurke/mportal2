Scheduling a Shutdown of Disk Defragmentation Processes

Scrip 227 (Process and Service Shutdown-Restart) can be configured to perform a regularly scheduled shutdown of disk defragmentation services, preventing a known problem where the defragmentation utility hangs. To set up this configuration for all machines at a site:

  1. Log onto your ASI server (https://asiservername/main).

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below "user login" and date.

  3. In the "sites" navigation bar, click on the "wizard" link (https://asiservername/main/config/index.php?act=wiz). The “Configuration Wizard” page will be displayed.

  4. Click on the "Scrip configuration" wizard entry. The “Scrip Configuration - Select a Group” page will be displayed.

  5. Select "A single site". The “Scrip Configuration - Select Site” page will be displayed.

  6. Click on the radio button for the site you want to configure. The “Scrip Configuration - Select Scrip - Site <site name>” page will be displayed.

  7. Scroll down to the entry for Scrip 227 (Process and Service Shutdown-Restart), and click on the Scrip name. The Scrip configuration page will be displayed.

  8. In the “Scrip enabled” configuration parameter check the box.

  9. In the "Process groups" configuration parameter, enter:
    defrag,DfrgFat.exe,DfrgNtfs.exe

  10. In the “Scheduled restart” configuration parameter, Enter defrag.

  11. Enter these values in the “Schedule” configuration parameter to schedule execution for 11 a.m. daily:

    Minute (0-59):

    0

    Hour (0-23):

    11

    Day (1-31):

    *

    Month (1-12):

    *

    Weekday (0-6):

    *

    Random minute:

    0

    Type (1 or 2):

    2

    Fail limit:

    2

  12. In the “On demand restart” configuration parameter, Enter defrag.

  13. Click the "Next" button.
