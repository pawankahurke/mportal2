Identifying and Grouping Server Systems

Identifying servers and non-server systems

The pre-defined asset queries Servers and Non-servers can be used to determine which machines at a site are servers and which machines are not. To create a report that identifies servers at a given site:

  1. Log onto your ASI server (https://asiservername/main). The ASI dashboard will be displayed.

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. Click on the “legacy reports” link in the “reports” navigation bar found the at upper right-hand corner of any page in the ASI server extended interface. The “Legacy Reports” page will be displayed.

  4. Click on the “Legacy asset reports” link to access the “legacy” asset reports page at https://asiservername/main/asset/report.php. The “legacy” “Asset Reports” pages will be displayed.

  5. Click on the add link in the line of links found above the “Search Options” panel. The “Add an Asset Report” page will be displayed.

  6. Configure a new asset report as follows

  • Report title: Type the report title using the site name, for example, Servers - Site1.

  • Report output: Choose Send as email.

  • E-mail recipients: Type your email address.

  • Default e-mail recipients: Choose no.

  • Saved Queries: Scroll down and highlight Servers.

  • Include: Scroll down and select the desired site.

  • Categorize by: Machine Name

  • Include Links: No

  • Include details: No

  • Run Report: Immediate

  1. When all the information has been entered, click the Add button near the bottom of the page. The report should be delivered by email within a 5-10 minutes.

To run a single report covering all sites, the procedure is similar. In the “Include” box, select “All”, the first choice at the top of the box..

To produce a report that identifies non-server machines, follow the same procedure, substituting Non-servers as the name of the saved query and as the report title.

Grouping server systems into one group

Once server and non-server machines have been identified, you may want to define a group including all server systems, or one including all server systems at one site. To add a group including all server systems you should perform the following steps:

  1. Log onto your ASI server (https://asiservername/main). The ASI dashboard will be displayed.

  2. Click on the “Use extended interface” link on the right side of the ASI Dashboard upper pane, below user login and date. The “Welcome” page will be displayed.

  3. Click on the “sections” link in the “reports” navigation bar found the at upper right-hand corner of any page in the ASI server extended interface. The main “Sections” page will be displayed.

  4. Click the “groups” link in the “tools” navigation bar. The “Select or Add a Group” page will be displayed.

  5. Click the “Add new group” button. The “Group Management - Add a Group” page will be displayed.

  6. Enter the group name (for example, Servers - site1). Check the “Allow other users to use this group?” box if you would like to allow other users to use this group. Click on the “OK” button. A page listing the available sites will be displayed.

  7. To select machines at a site, click on the “select/de-select individual” link for that site. Check the box in the New column for each machine to be added to the group, based on the information provided by the report created following the instructions in the “Identifying server and non-server systems” section of this document (e.g. Servers - Site1). When finished, click on the “OK” button. The page listing the available sites will be displayed once again.

  8. Repeat the steps in 7. until you have selected all the systems you want to include in the group you are defining. When finished, click on the “OK” button to complete definition of the new group. A page confirming the definition of the new group will be displayed with “Return to group management”, and “Go to management wizards” links in the body of the page.

A group consisting of the servers at a single site can be defined in a similar way, the only difference being that you would perform step 7. only once clicking on the “select/de-select individual” link only for the site whose server systems you want to place in a group.

Note

Please note that the procedure described in the “Grouping server systems into one group” section of this document can be used to define any group of systems.
