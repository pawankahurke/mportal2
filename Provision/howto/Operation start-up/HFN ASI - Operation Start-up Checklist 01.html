Automated Support Infrastructure - Operation Start-up Checklist

This document lists the tasks which you should perform to lay the foundation for optimal use of Automated Support Infrastructure (ASI) functions.

  1. Identify servers and non-server systems at each site, and place them in non-server and server groups. If so desired, place them also in site specific server and non-server groups. (Identifying and Grouping Server Systems)

  2. Using the Scrip execution frequency wizard, configure Scrip 177 (Scrip Configuration Changes) to run every three minutes on 1-3 systems per sub-net at each new site including the servers identified in 1. (Configuring Scrip Execution Frequency)

  3. Pre-configure the ASI on-demand remote control function (Scrip 236) on all systems at each new site. On as needed basis, using the ASI on-demand remote control function, install the remote control utility (UltraVNC). (Pre-configuring the ASI On-demand Remote Control Function)

  4. Configure The ASI Process and Service Shutdown-restart function (Scrip 227) on all systems to run daily, and prevent a known problem where the disk defragmentation utility hangs. (Scheduling a Shutdown of Disk Defragmentation Processes)

  5. Retrieve the public IP address of the Internet gateway at each site. Configure two systems at your home site to monitor gateway availability, and enable the corresponding e-mail notification. (Configuring Internet Gateway Monitoring)

  6. Identify the systems at each new site where Symantec Backup Exec, and Microsoft NT Backup are installed. Please note that this same procedure, with the appropriate configuration changes, can be used to identify whether any application is installed on systems at the new site. (Finding out on Which Systems a Back-up Application is Installed and Active)

  7. Update "Permitted items" list in Scrip 27 (System Start-up Environment Control) and 232 (Intrusion Protection Control) on all systems using the list contained in the "HFN ASI permitted items list master.xls" spreadsheet. (Updating Permitted Items for System Start-up and Intrusion Protection Control)

  8. Even if initially it is disabled, check and ensure that Microsoft Update Management (Scrip 237) configuration is as per ASI client default on all systems. (Verifying the Configuration for Scrip 237 (Microsoft Update Management))

  9. Enable the "ASI client not reporting" notification for critical devices only (servers to start with).(Monitoring ASI Clients on Critical Devices )

  10. Set up a version record for ASI client update (Configuring a Version Record the Remote ASI Client Upgrade)

  11. Configure impersonation (Configuring Impersonation)

  12. Enable a baseline set of e-mail notifications (Enabling Baseline Event Notifications)

  13. Configure availability monitoring of all servers (Configuring Device Availability Monitoring)

  14. Configure the system survey to run weekly (Changing System Survey (Scrip 61) Schedule)
