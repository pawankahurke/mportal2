Configuring a Version Record the Remote ASI Client Upgrade

To set up a new version record for the ASI Client update, the most convenient method is to edit or copy an existing record, making any necessary changes.

  1. Log onto your ASI server (https://asiservername/main).

  2. Click on the Use extended interface link on the right side of the ASI Dashboard upper pane, below "user login" and date.

  3. In the sites navigation bar, click on the updates link (https://asiservername/main/updates/index.php). The Set Site Version page will be displayed.

  4. At the right of the page, below the "How to" button, the updates navigation bar is displayed. Click the versions link in this navigation bar. The Available Versions page will be displayed.

  5. If you have not defined any version records, click on the “add” link in the row of links displayed below the page title. The Add a Version Record page will be displayed.

If you created a version record previously, and you want to edit it, click the edit link for that version record, and the Edit a Version Record page will be displayed. To copy an existing version record instead of editing it (for example, in case you cannot edit the version record because it was created by a different user), click the copy link instead, and the Copy a Version Record page will be displayed. If you are copying rather than editing, you will need to change the version record name.

Please note that the Add a Version Record, Edit a Version Record, and Copy a Version Record pages are identical.

  1. In the Download URL field, enter the URL pointing to the installation executable for the latest point release of the ASI client (for example, hfn-inst-2.004.029.0095.00.exe).

  2. From the name of the installation executable in the Download URL field (e.g. hfn-inst-2.004.029.0095.00.exe), select and copy (Ctrl+c) the version number (e.g. 2.004.029.0095.00). Paste this version number into the Version number number field, replacing the existing entry.

  3. Verify that the value of the Username field is the one required to access the location specified in the Download URL field.

  4. If the the entry in the Username field is correct, you do not need to modify the value of the Password parameter (although it would not hurt to re-enter it). Otherwise, enter the password for the user name that you have entered in the Username field.

  5. Make sure that the value of the Command Line parameter is "/VERYSILENT" (without quotes). If you wish, you can add other installation executable command line parameters, separating each command line parameter with a single space.

  6. When finished, if you are editing an existing record, click the Update button, or if you are copying a version record, click the Copy button.

Note

If you are downloading the ASI client installation executable from handsfreenetworks.com, entries for the fields in the Add a Version Record, Edit a Version Record, and Copy a Version Record pages would be like the following:

Version Name: <Any name you choose>

Version Number: Version number extracted from the name of the ASI client installation executable file as described in step 7) above

Global: <At your choice>

Download URL: http://handsfreenetworks.com/download/<name of ASI client installation executable, e.g. http://handsfreenetworks.com/download/ hfn-inst-2.004.029.0095.00.exe

Username: dnload

Password: dnload

Command Line: /VERYSILENT
