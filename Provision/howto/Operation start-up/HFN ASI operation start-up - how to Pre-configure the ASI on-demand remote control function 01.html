Pre-configuring the ASI On-demand Remote Control Function

To maximize support engineer productivity and minimize the likelihood of operational issues caused by mis-configuration, we strongly suggest that you pre-configure the ASI on-demand remote control function as soon as you install the ASI client at a site.

To pre-configure on-demand remote control on all systems at a site please perform the following steps:

  1. Log onto your ASI server (https://asiservername/main).

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below "user login" and date.

  3. In the "sites" navigation bar, click on the "wizard" link (https://asiservername/main/config/index.php?act=wiz).

  4. Click on the "Configure remote control" wizard entry. The “Remote Control – Configuration” page will be displayed.

  5. Select "A single site" option by clicking on the corresponding radio button. The “Remote Control - Select Site” page will be displayed.

  6. Select the site where you want to pre-configure the remote control function by clicking on the radio button for its entry. The “Remote Control - Select Connection Method - Site <site name>” page will be displayed.

  7. If you have a Citrix® GoToAssist™ account that you would like to use, select this option and enter the relevant information on the next page. Otherwise, choose Ultr@VNC. The “Remote Control - Ultr@VNC Setup - Site <site name>” page will be displayed.

  8. Unless you have special requirements, we recommend that you accept the default settings for the Ultr@VNC configuration, i.e. make no changes, and click the "Next" button.

Please note that the UltraVNC default configuration options are:

  • Use intermediate server

  • Connection will be initiated automatically

To pre-configure ASI on-demand remote control on a single machine, you would follow the steps described above except that in step 5. you would select the “A single machine” option by clicking on the corresponding radio button.

Please note that you can pre-configure the ASI on-demand remote control function for one system also as follows:

  1. Click on the “remote control” link in the “tools” navigation bar. The “Remote Control – Select Site” page will be displayed.

  2. Select the site where the system you want to configure the ASI on-demand remote control function on is located by clicking on the corresponding radio button. The “Remote Control - Select Machine” page will be displayed.

  3. Select the system you want to configure the ASI on-demand remote control function on by clicking on the corresponding radio button. The “Remote Control - Select Connection Method - Site <site name> - Machine <machine name>” page will be displayed.

  4. If you have a Citrix® GoToAssist™ account that you would like to use, select this option and enter the relevant information on the next page. Otherwise, choose Ultr@VNC. The “Remote Control - Ultr@VNC Setup - Site <site name> - Machine <machine name>” page will be displayed.

  5. Unless you have special requirements, we recommend that you accept the default settings for the Ultr@VNC configuration, i.e. make no changes, and click the "Next" button.

Please note that the UltraVNC default configuration options are:

  • Use intermediate server

  • Connection will be initiated automatically
