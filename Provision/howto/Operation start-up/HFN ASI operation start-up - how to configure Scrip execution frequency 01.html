Configuring Scrip Execution Frequency

When you initiate a system management, maintenance, or problem management action on the ASI server, for the action to take effect on the target systems, it needs to be retrieved by the ASI clients on the target systems. We call this operation Scrip configuration synchronization.

Scrip 177 (Scrip Configuration Update) performs Scrip configuration synchronization operations. Scrip configuration synchronization actions occur when Scrip 177 (Scrip configuration update) runs. Like any other Scrip with a schedule, Scrip 177 can run as frequently as once per minute. By default, it is configured to run once per hour with a 50-minute random interval.

For more timely execution of remote operations, Scrip 177 should be configured to run more frequently.

If Scrip 177 on every system at every site were configured to run once per minute, the number of event logs posted on the ASI server would increase significantly. Although the size of Scrip 177 event logs is small, in the range of 10 KBytes, in this case impact on bandwidth consumption at the ASI server location could be noticeable, depending on incoming and outgoing ASI server bandwidth.

For this reason, we advise customers to configure Scrip 177 to run frequently (every 2-3 minutes) on a subset of systems on each sub-net at each site (2-4 depending on the total number of systems on each sub-net). In this way, all Scrip configuration changes (applied to either one or multiple systems) will be processed by the targeted ASI clients within minutes of their execution on the ASI server.

  1. Log onto your ASI server (https://asiservername/main).

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below "user login" and date.

  3. In the "sites" navigation bar, click on the "wizard" link (https://asiservername/main/config/index.php?act=wiz).

  4. Click on the "Scrip execution frequency" wizard entry.

  5. Select the machine group type "A single machine".

  6. Select the desired site.

  7. Select the desired machine.

  8. Click the option button next to the "Minute" pulldown. Use the pulldown to choose the frequency (for example, 2 or 3). Click the Next button.

  9. To repeat the process for another machine, click "Go to the Scrip 177 Execution Frequency page".
