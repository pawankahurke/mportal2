Updating Permitted Items for System Start-up and Intrusion Protection Control

You can use the spreadsheet "HFN ASI permitted items list master.xls" to update the “permitted items” configuration parameter in Scrip 27 (System Start-up Environment Control), and Scrip 232 (Intrusion Protection Control). To do this for all machines:

  1. Open the Excel spreadsheet "HFN ASI permitted items list master.xls".

  2. Log onto your ASI server (https://asiservername/main).

  3. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below "user login" and date.

  4. In the "sites" navigation bar, click on the "wizard" link (https://asiservername/main/config/index.php?act=wiz).

  5. Click on the "Scrip configuration" wizard entry.

  6. Click the All machines link.

  7. Click the link for Scrip 27 (System Start-up Environment Control).

  8. Select and delete the entire contents of the Permitted items configuration parameter. Leave the browser window open.

  9. In Excel, click the column header of the first ("A") column to select the entire column.

  10. In the Excel window, go to the Edit menu and choose Copy.

  11. Return to the browser window and click in the empty Permitted items configuration parameter.

  12. Paste the new data in the box by going to the Edit menu and choosing Paste.

  13. Click the Next button.

  14. The new configuration changes will be displayed. Click the link to configure other Scrips for the same group.

  15. Click the link for Scrip 232 (Intrusion Protection Control) and follow the same procedure for this Scrip.

When you add a new site, you should repeat steps 1)-13) above for both Scrips, except for step 4) where you should select A single site instead of All machines.

When you install the ASI client on an additional system at a site where you already updated the Permitted items configuration parameter, you should update the Permitted items configuration parameter in Scrips 27 and 232 on the system where you just installed the ASI client as follows:

  1. Log onto the ASI server (https://asiservername/main ).

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

  3. At the upper right-hand corner of any page on the ASI server, click on the configuration link in the sites navigation bar (https://asiservername/main/config/index.php?act=site ).

  4. Click on the view machines link for the site where the system you want to update the Permitted items configuration parameter is located.

  5. Click on the configure Scrips link for the system you want to update the Permitted items configuration parameter.

  6. Click on the link for Scrip 27 (System Start-up Environment Control).

  7. From the Group the pull-down to the left of the Permitted items box select all machines.

  8. Click on the Submit Changes button.

  9. Click on the <(machine name)> Select another Scrip link below the “What do you want to do” question, and repeat steps f)-h) for Scrip 232 (Intrusion Protection Control).
