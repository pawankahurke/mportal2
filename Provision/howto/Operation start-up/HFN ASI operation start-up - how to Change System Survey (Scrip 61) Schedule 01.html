Changing System Survey (Scrip 61) Schedule

By default, Scrip 61 (System Survey) runs monthly. If you need asset information from client machines stored on the ASI server to be kept more current, the Scrip may be scheduled to run weekly instead. In this example, we will configure the Scrip to execute on all machines every Wednesday. The large value for the "random minute" setting insures that asset logging is spread throughout the day, reducing any impact on server performance and bandwidth consumption.

  1. Log onto the ASI server (https://asiservername/main ). The ASI dashboard will be displayed.

  2. Click on the Use extended interface link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. At the upper right-hand corner of any page on the ASI server extended interface, click on the wizard link in the sites navigation bar (https://asiservername/main/config/index.php?act=wiz).

  4. Click on the Scrip configuration link (https://asiservername/main/config/scrpconf.php?custom=8). The Scrip Configuration - Select a Group page will be displayed.

  5. Click on the All machines radio button. Then click on the Next button. The Scrip Configuration - Select Scrip page will be displayed.

  6. Scroll down the page, locate the entry for Scrip 61 (System Survey), and click on its name. The Scrip 61 configuration page will be displayed.

  7. First, look at the schedule. In the left-hand (Name) column, find the row that contains schedule data (Minute, Hour, etc.)

  8. In that row, look at the Value column. If the Configure button appears, click it to display the current values. Change the Scrip 61 schedule parameter values to match the following:

Minute (0-59):

0

Hour (0-23):

10

Day (1-31):

*

Month (1-12):

*

Weekday (0-6):

3

Random minute:

420

Type (1 or 2):

2

Fail limit:

0

  1. click the Next button.

You will need to allow some time for machines to receive the updated configuration, depending on your settings for Scrip 177 (Scrip Configuration Update), and the number of machines to be configured. Do not be concerned if the browser window displays “page not found”, or similar messages indicating that the browser timed out. The Scrip configuration change action you took will be completed regardless of any browser time-out event.
