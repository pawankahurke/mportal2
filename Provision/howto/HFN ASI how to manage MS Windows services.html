How-to Manage MS Windows Services Using the Automated Support Infrastructure Service Management Console

Overview

Automated Support Infrastructure (ASI) service management is the first and only remote service management facility that lets users manage MS Windows services on any number of systems, simultaneously, regardless of their location.

The ASI service management function duplicates the Microsoft Windows service management function. It lets you start, stop, re-start, enable, disable, and change service type from manual to automatic, and vice versa.

ASI service management includes two major functions not supported by the Microsoft Windows service management function:

  • You can manage services on multiple systems at the same time, remotely, regardless of their location. You can perform any action on one or more services on any number of systems simultaneously.

  • You can configure a service to always be kept running, or never be allowed to run.

In addition to being a powerful system management tool, the ASI service management function provides you with comprehensive information on all services defined on a system, not just those enabled or disabled, and updates this information in real-time.

Enabling the ASI service management function

Before you can start managing MS Windows services on systems at your sites, you need to enable Scrip 224 (MS Windows Service Manager) on the systems where you want to manage MS Windows services.

Scrip 224 can run in polling or event driven mode.

In polling mode, the Scrip runs at a scheduled time at which he surveys MS Windows services on a system, and reports all MS Windows service information to the ASI server.

In event-driven mode, Scrip execution is triggered in real-time by any MS Windows service change.

Please note that when running in event-driven mode, Scrip 224 will tend to run more frequently, accessing the ASI server more frequently.

Scrip 224 performs actions you configured on the MS Windows services console on the ASI server when:

  • An MS Windows service related change on the system causes the Scrip to contact the ASI server

  • Scrip 224 runs at its scheduled time

Accessing the ASI service management console

You can access all MS Windows services information collected by Scrip 224 (MS Windows Service Manager) via the ASI service management console. In order to access the ASI service management console, please perform the following steps:

  1. Log onto your ASI server (https://asiservername/main). The ASI dashboard will be displayed.

  2. If you want to manage MS Windows services, or access MS Windows service information on one system:

    1. If the “Sites” item in the left pane of the ASI Dashboard is not expanded, click on the "+" icon to the left of the “Sites” label to display entries for all sites on your ASI server

    2. Scroll down the left pane to the site where the system whose MS Windows services you want to manage or review

    3. Expand the entry for the site where the system is located (click on the "+" icon to the left of the site name)

    4. Click on the name of the system where you want to manage or review MS Windows services

    5. In the right pane, click on the "services" link in the list of links at the top of the pane. The services management console for the system will be displayed in the left pane.

  3. If you want to manage MS Windows services, or access MS Windows service information for a group of systems:

    1. If the “Groups” item in the left pane of the ASI Dashboard is not expanded, click on the "+" icon to the left of the “Groups” label to display entries for all groups defined on your ASI server

    2. Scroll down the left pane to the entry for the group of systems where where you want to manage or review MS Windows services

    3. Click on the name of the group.

    4. In the right pane, click on the "services" link in the list of links at the top of the pane.

Taking service management actions

At present, the ASI service management function supports the following service related actions:

  • Start - Starts the service only if it's not running, does nothing otherwise.

  • Stop - Stops a service if it is running, does nothing otherwise

  • Restart - If a service is running, it stops and starts it. If a service is stopped, it starts it.

To take any of the above actions for a service on a system, a site, or a group of systems you defined, simply click on the link corresponding to the action you want to take. After you take an action, a message asking for confirmation of the action will be displayed in the right pane, followed by an action confirmation message also displayed in the right pane. Clicking on the “Done” button on the confirmation page will take you back to the “Services” page for the system or group you took the action on.

Advanced service management

If you want to configure settings for a service, you should click on the “advanced” link for a service. When you do that, the “Service settings to be configured on machine <machine name> in group <group or machine name>” page will be displayed in the right pane.

You can make two types of service configuration setting changes:

  • Service status

  • Keep service running – If you choose this option, Scrip 224 (MS Windows Service Manager) will ensure that the service will run at all times, automatically re-starting it if it stops.

  • Keep service stopped – If you choose this option, Scrip 224 (MS Windows Service Manager) will ensure that the service will not run, automatically stopping it if it attempts to start.

  • Startup type

      • Automatic – When you select this option, Scrip 224 (MS Windows Service Manager) will set the service type on the system, or group of systems, to “automatic”

      • Manual – When you select this option, Scrip 224 (MS Windows Service Manager) will set the service type on the system, or group of systems, to “manual”

      • Disabled – When you select this option, Scrip 224 (MS Windows Service Manager) will set the service type on the system, or group of systems, to “disabled”

Please note that you can configure “service status” and “startup type” settings at the same time.

To select a setting for a service on a system, a site, or a group of systems you defined, simply click on the radio button corresponding to the desired setting. Clicking on the “Update” button will cause the content of the “Current Settings” window to be updated with the changes you just made. Clicking on the “Done” button will take you back to the “Services information” page. Please note that you do not have to click on the “Update” button. Clicking on the “Done” button will both update the settings and take you back to the “Services information” page.



After you access the ASI service management console as described in the “Accessing the ASI service management console” section of this document, you can take any of the actions described above by clicking on the corresponding action link the MS Windows service you want to take the action on. In order to access “Advanced” functions, you need to click on the “advanced” action link.

Notes

When taking a service management action on a group of systems, the service management software running on the ASI server doesn't determine which systems in the group the action applies to. Scrip 224 (MS Windows Service Manager) on the systems in the group retrieves the action, and attempts to perform the action. The Scrip doesn't check the status of the service before it attempts to perform the action. If the action does not apply, nothing will happen.

The ASI MS Windows service management function does not replace Scrip 227 (Process and Service Shutdown-restart). Scrip 227 can run periodically to take a shut-down or re-start action on one or more MS Windows services while the ASI MS Windows service management function lets you take one-time actions.

Until we integrate the ASI event notification function with the ASI dashboard, you should continue to use Scrip 176 (Services Restart) to monitor MS Windows services otherwise you won't receive e-mail notifications when monitored services stop running. However, you can stop using the "restart" option in Scrip 176 and, instead, use the "Services Status" setting in the new MS Windows service management function.
