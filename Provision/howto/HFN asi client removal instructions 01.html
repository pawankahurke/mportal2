ASI Client Removal Instructions

In this document we describe two methods for removing the ASI client listed in order of preference, with the preferred method listed first. Please also take the time to read the ASI client removal notes section as it contains important information and answers to frequently asked questions about the ASI client removal.

ASI client removal using Scrip 43

Below, you will find step-by-step instructions for removing the ASI client using Scrip 43.

  1. Access the Scrip 43 configuration page in one of two ways:

    • Locally, at the system you want to remove the ASI client from,

      • Start an Internet browser session

      • Type http://localhost:2721 in the address box

      • Click on the entry for the system at the top of the page. The Scrip configurator page will be displayed

      • Click on the entry for Scrip 43 (HandsFree Client Tools)

    • Via the ASI server, at any system with Internet access,

      • Start an Internet browser session

      • Type https://asiserverurl/main/config/index.php in the address box. Note that you should replace asiserverurl with the URL of the ASI server the ASI clients at your sites report to.

      • Click on the view machines link for the site where the system(s) you want to remove the ASI client from are located.

      • Click on the configure Scrips link for the system where you want to remove the ASI client. The Scrip configurator page will be displayed.

      • Click on the entry for Scrip 43 (HandsFree Client Tools)

  2. Once you are viewing the Scrip 43 configuration page click on the Execute button for one of the following Scrip 43 configuration parameters:

    • HandsFree client uninstall

    • HandsFree client uninstall – silent

    At this point, ASI client removal is completed. If you performed it via the ASI server, it will take place the next time Scrip 177 (Scrip Configuration Update) from the system where you want to remove the ASI client contacts the ASI server for Scrip configuration changes, finds your Scrip 43 execute command, and downloads it to the ASI client on the system. At that point, Scrip 43 will execute the uninstall command.

ASI client removal using cutil.exe

You can also remove the ASI client from a system using the cutil.exe utility found in the client directory. In order to do this, you need to have direct access to the system where you want to remove the ASI client.

  1. Click on the Start button at the bottom left corner of the screen of the system where you want to remove the ASI client

  2. Click on the Run menu entry

  3. In the Open box type

    • [drive]:\Program Files\HandsFree\client\cutil.exe -l

    for silent removal, or

    • [drive]:\Program Files\HandsFree\client\cutil.exe -u

    for non-silent removal

    Please note that instead of typing the path for cutil.exe, you can locate it by clicking on the Browse button in the Run dialog box.

    You should also keep in mind that if you are using a re-branded ASI client the name of the directories will not be HandsFree and Client.

  4. Click on the OK button

Please note that if you attempt to use Scrip 89 , or one of 212-215 (Scheduled Program Execution), to run cutil.exe to remove the ASI client, you will not be able to remove the ASI client as Scrip 89 (or one of 212-215) will still be running when cutil.exe attempts to remove the ASI client.

ASI client removal notes

  1. VERY IMPORTANT. Double clicking on the unins000.exe executable in [drive]:\Program Files\HandsFree\client will not remove the ASI client. Do not attempt that. If, accidentally, you do that please contact HandsFree Networks Immediately.

  2. PLEASE NOTE THAT REMOVING THE ASI CLIENT ALSO REMOVES THE ASI NETWORK DRIVER (IF IT IS INSTALLED). ASI NETWORK DRIVER REMOVAL WILL CAUSE THE NETWORK CONNECTIONS ON THE SYSTEM WHERE IT IS BEING REMOVED FROM TO BE RESET. THIS ACTION IS PERFORMED BY THE OPERATING SYSTEM INDEPENDENTLY OF ANY ACTION TAKEN BY THE ASI SOFTWARE.

  3. On those systems where it is installed, to complete removal of the ASI network driver, you will need to re-start the system. The system re-start will remove the last hook left by the ASI network driver. Please note that the system re-start is not necessary as this last hook performs no function.

  4. The ASI client removal leaves the persistent state data in [drive]:\Program Files\HandsFree\client so that if you reinstall the ASI client it will still be configured as before the removal. It also leaves the UUID registry key so that a re-installation will not cause a change in the ID of the system.

If you really need to eliminate every trace of the ASI client, you should perform the removal using one of the two methods described in the preceding sections, re-start the system, then delete the directory [drive]:\Program Files\HandsFree, and delete the registry key HKLM\Software\HandsFree Networks

You should keep in mind that if you are using a re-branded ASI client the name of the directories will not be HandsFree and Client.
