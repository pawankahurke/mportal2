How to activate an ASI client which may be in dormant mode átop

Open a browser window.

In the address box, enter http://localhost:2721/

In the user logon dialog enter either <direct access user id> / <direct access password>, or hfn / hfn

Click on the name of the system at the very top of the page. The Scrip Configurator page will be displayed.

Click on the entry for Scrip 223 (HandsFree Client Installation and Deployment)

Enter the site registration code for the site the system you are working on belongs to in the Site registration code configuration parameter.

Click on the Submit Changes button.

Close the browser window.

How to check whether the ASI client is running átop

Method A - Services console

- Right-click on My Computer

- Click on Manage

- Double-click on Services and Applications in right-hand panel

- Double-click on Services icon in right-hand panel

- Scroll down the list of services, and locate the Handsfree client service. If the ASI client is running normally, it should have Automatic Startup Type and Status should be Started.

Method B - Task manager

- Press simultaneously the ctrl, alt, and del keys

- Click on the Processes tab

- Click twice on the Image Name column heading

- If the ASI client is running normally, you should be able to locate the following two entries:

  • capp.exe

  • cust.exe

Method C - ASI client direct access

- Open a browser window.

- In the address box, enter http://localhost:2721/

- In the user logon dialog enter <direct access user id>/<direct access password>

If you cannot log onto the ASI client using the direct access user id, it probably means that the ASI client is in dormant mode. Follow the instructions listed below for activating an ASI client out of dormant mode described in section How to activate an ASI client which may be in dormant mode of this document.

How to stop ASI client execution átop

Method A - Services console

- Right-click on My Computer

- Click on Manage

- Double-click on Services and Applications in right-hand panel

- Double-click on Services icon in right-hand panel

- Scroll down the list of services, and locate the Handsfree client service.

- Double-click on the Handsfree client entry

- Click on the Stop button

- Click on the OK button

Method B - Task manager

- Press simultaneously the ctrl, alt, and del keys

- Click on the Processes tab

- Click twice on the Image Name column heading

- Locate the following two entries

  • capp.exe

  • cust.exe

- Click on the entry for cust.exe

- Click on the End Process button

Method C - ASI client direct access (can be used if you are on site)

- Open a browser window.

- In the address box, enter http://localhost:2721/

- In the user logon dialog enter <direct access user id> / <direct access password>

- Click on the name of the system at the very top of the page. The Scrip Configurator page will be displayed.

- Scroll down to the entry for Scrip 18 (Client Shut-down / Re-start), and click on the Scrip name

- Click on the Shutdown this client Execute button

- Close the browser window.

Method D - ASI server (to be used if you are not on site)

- Log onto the ASI server (https://asiservername/main )

- Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

- At the upper right-hand corner of any page on the ASI server, in the sites navigation bar click on the configuration link (https://asiservername/main/config/index.php?act=site )

- Click on the view machines link for the site where the system you want to remove the ASI driver from is located

- Click on the configure Scrips link for the system you want to remove the ASI driver from

- Scroll down to the entry for Scrip 18 (Client Shut-down / Re-start), and click on the Scrip name

- Click on the Shutdown this client Execute button

- Close the browser window.

How to remove the ASI network driver átop

When a system experiences a connectivity problem which you suspect is caused by the ASI client you do not need to remove or disable the ASI client. You should use one of the following methods to remove the ASI network driver. Please note that you can remotely re-install the network driver on any system it was removed from at any time using Scrip 201 (clicking on the Install network driver Execute button with the Group parameter set to machine <name>), and re-starting the system.

Please note that when you install or remove the ASI network driver on a system, the operating system automatically resets the system's network connection. At this point in time, there is no way to prevent this action from taking place.

Method A - ASI server (to be used if you are not on site)

1) Log onto the ASI server (https://asiservername/main )

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) At the upper right-hand corner of any page on the ASI server, click on the configuration link in the sites navigation bar (https://asiservername/main/config/index.php?act=site )

4) Click on link view machines for the site where the system you want to remove the ASI network driver from is located

5) Click on the configure Scrips link for the system you want to remove the ASI network driver from

6) Scroll down to the entry for Scrip 201 (Network Device Driver Management), and click on the Scrip name

7) Set the Uninstall network driver Group parameter to the name of the system where you want to disable the ASI network driver by selecting the entry machine <name> from the pull-down list to the right of the Uninstall network driver Group parameter. Pay special attention to this step. It is VERY IMPORTANT.

8) Click on the Uninstall network driver Execute button

9) Wait until the change is processed by the ASI client on the system in question (as long as 20 minutes but check on the configuration page the site where the system you are removing the ASI network driver from is located to make sure that the change has been processed by making sure that the Changed entry for the system is No).

10) To make sure that the ASI network driver has been removed on the system in question, you might want to ask the user to restart it, although this should not be necessary.

Method B - Direct ASI client access (can be used if you are on site)

1) Open a browser window.

2) In the address box, enter http://localhost:2721/

3) In the user logon dialog enter <direct access user id> / <direct access password>

4) Click on the name of the system at the very top of the page. The Scrip Configurator page will be displayed.

5) Scroll down to the entry for Scrip 201 (Network Device Driver Management), and click on the Scrip name

6) Set the Uninstall network driver Group parameter to the name of the system where you want to disable the ASI network driver by selecting the entry machine <name> from the pull-down list to the right of the Uninstall network driver Group parameter. Pay special attention to this step. It is VERY IMPORTANT.

7) Click on the "Uninstall network driver" Execute button

8) Close the browser window.

9) To make sure that the ASI network driver has been removed on the system in question, you might want restart the system, although this should not be necessary.

How to remove the ASI registry driver átop

When a system experiences a problem which you suspect is caused by interference by the ASI client with registry operations, you do not need to remove or disable the ASI client. You should use one of the following methods to remove the ASI registry driver. Please note that you can remotely re-install the registry driver on any system it was removed from at any time using Scrip 248 (clicking on the Enable driver to be loaded on startup Execute button with the Group parameter set to machine <name>), and Scrip 18 as described below.

Method A - ASI server (to be used if you are not on site)

1) Log onto the ASI server (https://asiservername/main )

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) At the upper right-hand corner of any page on the ASI server, click on the configuration link in the sites navigation bar (https://asiservername/main/config/index.php?act=site )

4) Click on the view machines link for site where the system you want to remove the ASI registry driver from is located

5) Click on the configure Scrips link for the system you want to remove the ASI registry driver from

6) Scroll down to the entry for Scrip 248 (Registry Device Driver Management), and click on the Scrip name

7) Set the Group selection for the Prevent driver from being loaded on startup configuration parameter to the system where you want to remove the registry driver by selecting machine <name> from the Group pull-down list. Pay special attention to this step. It is VERY IMPORTANT.

8) Click on the Prevent driver from being loaded on startup Execute button

9) Wait until the change is processed by the ASI client on the system in question (as long as 20 minutes but check on the configuration page the site where the system you are removing the ASI network driver from is located to make sure that the change has been processed by making sure that the Changed entry for the system is No).

10) Re-start the ASI client as follows:

- On the navigation bar at the upper right-hand corner of any page on the ASI server, click on the "configuration" link in the "sites" navigation bar (https://asiservername/main/config/index.php?act=site )

- Click on the view machines link for the site where the system whose ASI client you want to re-start is located

- Click on the configure Scrips link for the system you want to re-start the ASI client on

- Scroll down to the entry for Scrip 18 (Client Shut-down / Re-start), and click on the Scrip name

- Click on the Restart this client Execute button

- Click on the machines link in the configuration navigation bar about 1/3 down on the right side of the page you are viewing

Method B - Direct ASI client access (can be used if you are on site)

1) Open a browser window

2) In the address box, enter http://localhost:2721/

3) In the user logon dialog enter <direct access user id> / <direct access password>

4) Click on the name of the system at the very top of the page. The Scrip Configurator page will be displayed

5) Scroll down to the entry for Scrip 248 (Registry Device Driver Management), and click on the Scrip name

6) Set the Group selection for the Prevent driver from being loaded on startup configuration parameter to the system where you want to remove the registry driver by selecting machine <name> from the Group pull-down list. Pay special attention to this step. It is VERY IMPORTANT.

7) Click on the Prevent driver from being loaded on startup Execute button

8) Click on the Scrips link at the upper right-hand corner of the page you are viewing

9) Scroll down to the entry for Scrip 18 (Client Shut-down / Re-start), and click on the Scrip name

10) Click on the Restart this client Execute button

11) Close the browser window.

How to retrieve a site's registration code átop

You can get the site registration code in one of three ways:

1) If you are physically at a system where the ASI client is installed with the site registration code you are looking for:

a) Open a browser window

b) Log onto the ASI client at http://localhost:2721

c) Click on the name of the system

d) Click on the entry for Scrip 223 (HandsFree Client Installation and Deployment)

e) Read the entry in the "Site registration code" configuration parameter

2) Log onto the ASI server (https://asiservername/main), and click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

a) Click on the "configuration" link in the "sites" navigation bar

b) Click on the "view machines" link for the site whose registration code you want to find

c) Click on the "configure Scrips" link for any system in the machine list for the site

d) Click on the entry for Scrip 223 (HandsFree Client Installation and Deployment)

e) Read the entry in the "Site registration code" configuration parameter

3) Log onto the installation server at https://nanoheal.org/main/install with your usual user id and password, then click on the "Sites" link. The Sites page lists all sites for which you have defined a site record, and their registration codes.

How to change HandsFree event management server URL - locally átop

Please take extreme care when changing the URL for the ASI server you want an ASI client to log to. Before you make the change, taking the steps described below, test the URL as follows:

1) Open a browser window

2) In the browser address box enter:

https://<asiservername>:<HTTPS port number>/main/rpc/rpc.php

3) Click on the "Enter" key on your keyboard. (After clicking on the "Yes" button in the security alert dialog, if

necessary). If the following message is NOT displayed do NOT use the URL as the entry in the "HandsFree event management server URL list" configuration parameter in Scrip 43.

"

The XML page cannot be displayed

Cannot view XML input using XSL style sheet. Please correct the error and then click the Refresh button, or try again later.

--------------------------------------------------------------------------------

The stylesheet does not contain a document element. The stylesheet may be empty, or it may not be a well-formed XML document.

"

If you do not see the above message, do not change the the "HandsFree event management server URL list" configuration parameter. Contact HandsFree Networks.

After having confirmed that the "HandsFree event management server URL" entry is correct, as described above, you can proceed with the change of the "HandsFree event management server URL" entry, as follows:

a) Open a browser window

b) Log onto the ASI client at http://localhost:2721 (using your user id and password)

c) Click on the name of the system. The Scrip configurator page for the system will be displayed.

d) Click on the entry for Scrip 43 (HandsFree Client Tools)

e) Scroll down to the "HandsFree event management server URL list" configuration parameter

f) If you want to change the HandsFree event management server URL for the entire site, leave the "HandsFree event management server URL list" configuration parameter set to the site (i.e. the site should be selected from the "Group" pull-down list" found to the left of the configuration parameter). Otherwise, in the the "Group" pull-down list" select the machine whose Scrip 43 configuration page you are viewing.

g) Delete the current entry

h) Enter the "HandsFree event management server URL" you confirmed to be correct

i) Click on the "Submit Changes" button

j) Close the browser window

How to change HandsFree event management server URL - from the ASI server átop

Please take extreme care when changing the URL for the ASI server you want an ASI client to log to. Before you make the change, taking the steps described below, test the URL as follows:

1) Open a browser window

2) In the browser address box enter:

https://<asiservername>:<HTTPS port number>/main/rpc/rpc.php

3) Click on the "Enter" key on your keyboard. (After clicking on the "Yes" button in the security alert dialog, if

necessary). If the following message is NOT displayed do NOT use the URL as the entry in the "HandsFree event management server URL list" configuration parameter in Scrip 43.

"

The XML page cannot be displayed

Cannot view XML input using XSL style sheet. Please correct the error and then click the Refresh button, or try again later.

--------------------------------------------------------------------------------

The stylesheet does not contain a document element. The stylesheet may be empty, or it may not be a well-formed XML document.

"

If you do not see the above message, do not change the the "HandsFree event management server URL list" configuration parameter. Contact HandsFree Networks.

After having confirmed that the "HandsFree event management server URL" entry is correct, as described above, you can proceed with the change of the "HandsFree event management server URL" entry, as follows:

1) Log onto your ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) In the "sites" navigation bar, click on the "configuration" link (https://asiservername/main/config/index.php?act=site)

4) Click on the "view machines" link for the site where you want to change the HandsFree event management server URL. The "Configuration <site name> Machines" page will be displayed.

5) If you want to change the HandsFree event management server URL for the entire site, click on the "configure Scrips" link for any system in the site's machine list. Otherwise, click on the "configure Scrips" link for the system where you want to change the HandsFree event management server URL. The "Scrip configurator" page for the system whose "configure Scrips" link you clicked on will be displayed.

6) Scroll down to the entry for Scrip 43 (HandsFree Client Tools), and click on the Scrip name.

7) Scroll down to the "HandsFree event management server URL list" configuration parameter, and enter the URL for the ASI server you want the ASI client for the system (or entire site) to post event logs to, which you tested as described above.

8) If you want to change the HandsFree event management server URL for the entire site, leave the "HandsFree event management server URL list" configuration parameter set to the site (i.e. the site should be selected from the "Group" pull-down list" found to the left of the configuration parameter). Otherwise, in the the "Group" pull-down list" select the machine whose Scrip 43 configuration page you are viewing.

9) Click on the "Submit Changes" button found both above and below the Scrip configuration parameter table. The page confirming the action you took will be displayed.

How to produce a "Last ASI Client Contact" Report átop

1) Log onto your ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) In the "sites" navigation bar click on the "wizard" link (https://asiservername/main/config/index.php?act=wiz)

4) In the "Configuration Wizard" page, click on the "Export data" entry (https://asiservername/main/acct/export.php)

5) Click on the "core" radio button, then click on the "Next" button

6) Click on the "Census" radio button, then click on the "Next" button

7) De-select the "id", "uuid", "code", "censusuniq", "born", "deleted", "censussiteuniq", "revhost", and "cksum" fields

8) Select 0 from the "Select a maximum number of records to export:" pull-down list

9) Select the desired date format from the "Format time" pull-down list

10) Click on the "Next" button

11) Enter the desired file name in the "Filename" data entry box

12) Click on the "Text" button

13) Save the file to the desired location.

How to produce: Report - Machines whose ASI client has not reported for more than one week (configurable) átop

1) Log onto the ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) In the sites navigation bar click on the "wizard" link (https://asiservername/main/config/index.php?act=wiz)

4) In the "Configuration Wizard" page, click on the "Export data" entry (https://asiservername/main/acct/export.php)

5) Click on the "core" radio button, then click on the "Next" button

6) Click on the "Census" radio button, then click on the "Next" button

7) De-select the "id", "uuid", "code", "censusuniq", "born", "deleted", "censussiteuniq", "revhost", and "cksum" fields

8) Select 0 from the "Select a maximum number of records to export:" pull-down list

9) Enter

last < (unix_timestamp() - 604800)

in the Optional SQL data entry box

  • last -- the last time the ASI client on a system contacted the ASI server (stored as the number of seconds since January 1, 1970)

  • unix_timestamp -- today's date stored as the number of seconds since January 1, 1970

  • 604800 -- the number of seconds in one week

10) Select the desired date format from the "Format time" pull-down list

11) Click on the "Next" button

12) Enter the desired file name in the "Filename" data entry box

13) Click on the "Text" button

14) Save the file to the desired location.

How to diagnose connectivity issues that prevent the ASI client on one or more systems from communicating with the ASI server átop

1) Ensure that the ASI client process (cust.exe) and Services (HandsFree client) is running. If necessary re-start them

(cust.exe is located in \Program Files\HandsFree\Client)

2) Open a browser window

3) In the address box, enter

http://localhost:2721

4) When the user id dialog box is displayed enter the ASI client direct access user id and password for the site as entered in the site record on the installation server (https://nanoheal.org/main/install)

5) Once you have logged onto the ASI client, click on the system entry displayed at the top of the page. This should consist of the name and IP address of the system where you installed the ASI client. The Scrip Configurator page should be displayed.

6) Scroll down to the Scrip 43 (HandsFree Client Tools) entry, and click on the Scrip name. The Scrip's configuration page will be displayed

7) Scroll down to the "Dump failed logging attempts to error log file" configuration parameter, and click on the box to the right of the label, leaving the configuration parameter group selection set to the site (i.e. apply the change to all systems at the site).

8) Click on the "Submit Changes" button

9) Click on the "scrips" link found at the top right-hand corner of the page

10) Scroll down to the Scrip 23 (Send Queued Event Logs) entry, and click on the Scrip name. The Scrip's configuration page will be displayed

11) Click on the "Send event logs now" Execute button found at the bottom of the Scrip 23 configuration page

12) Close the browser window

13) Wait a few minutes (2-3), then browse to the ASI client directory (\Program Files\HandsFree\Client), retrieve a copy of a file named errlog.txt, and send it to HandsFree Networks.

How to check whether the ASI client installation completed successfully on one system átop

1) Ensure that the ASI client process (cust.exe) and Services (HandsFree client) are running on the system where you installed the ASI client. If necessary re-start them (cust.exe is located in \Program Files\HandsFree\client)

2) On the system where you installed the ASI client, open a browser window

3) In the address box, enter

http://localhost:2721

4) When the user id dialog box is displayed enter your user id and password. If you cannot log onto the ASI client with your user id and password try hfn/hfn. If you still cannot log onto the ASI client, please contact HandsFree Networks.

5) Once you have logged onto the ASI client, click on the system entry displayed at the top of the page. This should consist of the name and IP address of the system where you installed the ASI client. The Scrip Configurator page should be displayed.

6) Scroll down to the entry for Scrip 43 (HandsFree Client Tools), and click on the name of the Scrip. The Scrip's configuration page will be displayed.

7) Verify that the name of the site in the "Site" configuration parameter is correct.

8) Click on the "scrips" link at the top right-hand corner of the page. The Scrip Configurator page will be displayed.

9) Scroll down to the entry for Scrip 223 (HandsFree Client Installation and Deployment), and click on the name of the Scrip. The Scrip's configuration page will be displayed.

10) Verify that the the content of the "Site registration code" configuration parameter is correct. Please let me know.

11) Close the browser window.

12) If the entries in the "Site" configuration parameter in Scrip 43 (HandsFree Client Tools), and the "Site registration code" configuration parameter in Scrip 223 (HandsFree Client Installation and Deployment) are correct, then the ASI client installation on the system completed successfully.

Hot to edit-copy an ASI client update version record átop

1) Log onto the ASI server (https://asiservername/main)

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) Go to the Updates module (https://asiservername/main/updates/index.php). The "Set Site Version" page will be displayed. (Click on the "updates" link in the "sites" navigation bar displayed in the top right-hand corner area of every page in the ASI extended interface.)

4) In the second-level navigation bar labeled "updates" found on the right side of the page at the same level as the page title, click on the "versions" link. The "Available Versions" page will be displayed.

5) Click on the "edit" link for the version record you want to edit. The "Edit a Version Record" page will be displayed.

If you did not create the version record (i.e. you cannot edit it) you want to modify, or want to copy an existing version record, just click on the "copy" link. The "Copy a Version Record" page will be displayed. Change the version record name, and proceed with steps 6)-11) below.

6) In the "Download URL" field enter the URL pointing to the installation executable for the latest point release of the ASI client (e.g. hfn-inst-2.004.024.0312.BM.exe).

7) From the name of the installation executable for the point release of the ASI client in the "Download URL" field (e.g. hfn-inst-2.004.024.0312.BM.exe) select and copy (Ctrl+c) the version number (e.g. 2.004.024.0312.BM). Paste the version number (e.g. 2.004.024.0312.BM) in the "Version number" number field (replacing the existing entry).

8) Verify that the value of the "Username" field is the one required to access the location specified in the "Download URL" field.

9) If the the entry in the "Username" field is correct you do not need to modify the value of the "Password" parameter (although it would not hurt to re-enter it). Otherwise, enter the password for the user name entered in the "Username" field.

10) Check and make sure that the value of the "Command Line" parameter is "/VERYSILENT" (without quotes). If you wish, you can add other installation executable command line parameters separating each command line parameter with a single space.

11) Click on the "Update" button (if you are copying a version record, click on the "Copy" button).
