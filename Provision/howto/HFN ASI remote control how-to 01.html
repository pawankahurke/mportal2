How to take remote control of a system where the ASI client is running átop

Before you follow the instructions below, please make sure that:

1) One copy of the Ultr@VNC client process is running in listen mode on the remote system(s) (those used to take control of the target system(s)).

2) If you are using the direct remote control session option (where the UltraVNC server looks for a specific system to give control to) instead of using the intermediate server, and if the remote system(s) access the Internet via a gateway device, the port configured as the listener port (see below) is open for incoming traffic.

3) If you are using the direct remote control session option (where the UltraVNC server looks for a specific system to give control to) instead of using the intermediate server, and if the remote system(s) access the Internet via a gateway device, traffic coming into the listener port, be forwarded to the remote system (the system taking control of the target system(s)).

4) If you are using the direct remote control session option (where the UltraVNC server looks for a specific system to give control to) instead of using the intermediate server, and if the remote system(s) access the Internet via a gateway device, you have the public address (IP or domain) either of the remote system itself, if it has one or, more likely, of the gateway device used by the remote system to access the Internet.

5) No other remote control software is running on either the remote or target systems.

Important – Please remember that when you activate Ultr@VNC based remote control via the ASI server (Method A below), the remote control session on the target system(s) will begin the next time the ASI client from these system(s) contacts the ASI server. This will happen at a frequency determined by the execution schedule of Scrip 177. By default, Scrip 177 runs once per hour with a 50-minute random interval. To change the Scrip 177 execution schedule, you can follow the instructions in the How to change the frequency with which the ASI client retrieves Scrip configuration changes from the ASI server section of this document.

Method A - ASI server (to be used if you want to take remote control of a system at a customer site)

1) Log onto the ASI server (https://asiservername/main )

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) At the upper right-hand corner of any page on the ASI server, click on the remote control link in the tool navigation bar (https://asiservername/main/config/index.php?act=site )

4) Follow the on-screen instructions

Alternatively,

1) Log onto the ASI server (https/asiservername/main )

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) At the upper right-hand corner of any page on the ASI server, click on the wizard link in the sites navigation bar (https://asiservername/main/config/index.php?act=wiz )

4) Click on the Configure remote control wizard entry

5) Follow the on-screen instructions

Method B – User initiated remote control session

1) Open a browser window.

2) Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date.

3) Log onto the ASI server (https://asiservername/main )

4) At the upper right-hand corner of any page on the ASI server, click on the configuration link in the sites navigation bar (https://asiservername/main/config/index.php?act=site )

5) Click on the view machines link for the site of the system where you want to enable the end-user initiated remote control session function

6) Click on the configure Scrips link for the system where you want to enable the end-user initiated remote control session function

7) Scroll down to the entry for Scrip 236 (On-demand Remote Control), and click on the Scrip name

8) Enable the Place remote support activation icon on desktop configuration parameter (check its box), leaving the group selection for the configuration parameter set to site <name> if you want to enable the end-user initiated remote control session function on all systems at the site, or leaving it set to machine <name> if you want only the system whose Scrip 236 configuration page you are viewing to have this function enabled.

9) If you wish to, enter the text for the remote support activation icon in the Remote support icon label (Blank defaults to "Start remote support") configuration parameter, again leaving the group selection for the configuration parameter set to site <name> if you want to enable the end-user initiated remote control session function on all systems at the site, or leaving it set to machine <name> if you want only the system whose Scrip 236 configuration page you are viewing to have the custom remote support activation icon label.

10) Scroll down to the bottom of the Scrip 236 configuration page, and click on the Submit Changes button. The Changes Recorded page will be displayed.

11) Once the Scrip 236 configuration change described in steps 1)-10) above has been retrieved by the ASI client on the system(s) where you want to enable the end-user initiated remote control session function, whenever end-users on systems where this function is enabled click on the remote support activation icon on their desktop, Scrip 236 will run and initiate a remote control session (either direct or via the intermediate server) instantaneously, as long as

  • the system whose IP address is entered in the Address of remote computer configuration parameter (if you are using the direct remote control session mode) is running one copy of the UltraVNC viewer, or

  • a system, located anywhere, with access to the intermediate server, is running one copy of the UltraVNC viewer configured with the correct remote control session id.

Method C - Direct ASI client access (it can be used on site to let a remote system take control of a system where the ASI client is running)

1) Open a browser window.

2) In the address box, enter http://localhost:2721/

3) In the user logon dialog enter <direct access user id> / <direct access password>

4) Click on the name of the system at the very top of the page. The Scrip Configurator page will be displayed.

5) Scroll down to the entry for Scrip 236 (On-demand Remote Control), and click on the Scrip name

6) If you want to activate a remote control session via the intermediate server (the preferred approach), ensure that the Use intermediate server configuration parameter is enabled (check its box). If you wish to use the direct connect remote control session function, in the Address of remote computer configuration parameter, enter the public address (IP or domain) at the site where the remote computer (the one taking control of the target computer) is located.

7) If you want to, enable the Uninstall on shutdown configuration parameter

8) Click on the Install and run now Execute button (it does not matter if Ultr@VNC is already installed on the target system which, in this case is the system you are working on)

Method D - Local ASI user interface (if the local user interface is activated, it can be used on site to let a remote system take control of a system where the ASI client is running)

- Double-click on the ASI client icon in the system tray

- Click on the Start / Stop Remote Control Support wizard entry

- Follow the on-screen instructions
