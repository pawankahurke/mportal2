ASI Connectwise PSA Support Configuration and Use Guide

Introduction

When you configure and enable Connectwise PSA support in event notifications, every time a notification which you configured to be sent via e-mail is produced, it will be sent automatically to your Connectwise PSA server where a trouble ticket will be automatically generated.

To support integration with Connectwise PSA, we have added the following parameters to the Add an Event Notification, Edit an Event Notification, and copy an Event Notifications pages:

  • Per-site e-mail notifications

Enabling this event notification configuration parameter will generate an e-mail notification message for each site event logs have been retrieved from.

  • Use site e-mail address as sender address

When you enable this event notification configuration parameter, the sender e-mail address used by an event notification will be the one you entered for a site in the admin module on your ASI server (https://ASIserverURL/main/acct/admin.php).

Before you start enabling Connectwise PSA support in existing event notifications on your ASI server, please read the Enabling Connectwise PSA support in existing notifications section of this document. It will save you a lot of tedious work.

Connectwise PSA support configuration

You can configure existing event notification with Connectwise PSA support in the Edit an Event Notification page (e.g. https://ASIserverURL/main/event/notify.php?nid=256&act=edit). You can configure new event notifications with Connectwise PSA support in the Add an Event Notification page (https://ASIserverURL/main/event/notify.php?act=addn), or you can add Connectwise PSA support when you copy an event notification via the Copy an Event Notification page (e.g., https://ASIserverURL/main/event/notify.php?nid=256&act=copy).

In all three pages, you will need to configure the same two event notification configuration parameters:

  • Per-site e-mail notifications

  • Use site e-mail address as sender address

The Per-site e-mail notifications, and Use site e-mail address as sender address event notification configuration parameters are boxes that you simply need to check whenever you want to enable Connectwise PSA support. This is all you need to do in order to enable Connectwise PSA support in your event notifications.

Site e-mail address

You can add an e-mail address for an existing site on your ASI server as follows:

  1. Access the Admin page on your ASI server at https://ASIserverURL/main/acct/admin.php

  2. Click on the sites link at the top of the Admin page

  3. Scroll down to the entry for the site whose record you want to add an e-mail address to, and click on the [edit] link for that site

  4. Enter the desired site e-mail address in the Site Email Address configuration parameter

  5. When ready, click on the Assign button at the bottom of the page.

In the unlikely event that you need or want to add a record for a site, you should follow the fives steps listed above except for step 2 when instead of clicking on the sites link, you would click on the new site link, at the top of the Admin page.

Enabling Connectwise PSA support in existing notifications

You can enable Connectwise PSA support in multiple event notifications with one editing action, as follows:

  1. Access the Event Notifications page on your ASI server at https://ASIserverURL/main/event/notify.php

  2. Select the notifications you want to add Connectwise PSA support to using the selection parameters in the Search Options panel. For example, if you want to select all enabled event notifications, simply select enabled from the State search parameter pull-down list.

Make sure that the Page Size parameter in the Display Options panel is large enough to display ALL selected event notifications.

  1. Click on the manage link at the top of the Event Notifications page

  2. Click on the Edit Multiple Notifications link in the Manage Event Notifications page

  3. Click on the Check all button, or click in the box next to the name of each notification you want to apply the change to.

  4. Select Yes from the E-mail per site pull-down list

  5. Select Yes from the Use site e-mail pull-down list

  6. When ready, click on the OK button at the bottom of the page.
