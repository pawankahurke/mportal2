How to Manage Autotask Event Notifications

Previously, users could configure event notifications to be automatically sent to the Autotask server of their choice, and automatically generate tickets.

Extended Autotask support lets users:

  • Create open (or closed) Autotask tickets

  • Enable or disable Autotask integration per ASI notification

  • Optionally append additional instances of a notification to an existing Autotask ticket avoiding unnecessary ticket proliferation

  • Directly access their Autotask server, and configure Autotask settings per user including :

    • Autotask server assignment by site

    • Ticket priority

    • Ticket queue

    • Ticket due date

    • Ticket creation options

      • Always create new tickets

      • Always create new tickets and close them

      • Append multiple instances of the same notification to an existing ticket

    • Ticket Note options

      • Note Type

      • Publish

Extended Autotask support works as described below.

Configuring Autotask settings

  1. Log onto your ASI server (https://asiservername/main). The ASI dashboard will be displayed.

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. At the upper right-hand corner of any page on the ASI server, click on the “notifications” link in the “events” navigation bar found the at upper right-hand corner of any page in the ASI server extended interface. The “Event Notifications” page will be displayed.

  4. If you want to define a default Autotask configuration for all users on the ASI server, click on the “autotask server defaults” link in the line of links above the “Search Options” panel. If you want to define an Autotask configuration for the user currently logged on, click on the “autotask” link in the line of links above the “Search Options” panel. The “Autotask Server Login” page will be displayed.

Please note that whether you click on the “autotask server defaults”, or the “autotask” link, the information you will be asked to provide is the same. In the former case it will apply to all users, in the latter it will apply only to the user currently logged on. The main differences between the Autotask settings pages for one user and for all users are:

  • The word “Default” used in the title of the Autotask settings page for all users

  • In the "user" Autotask settings page, for each ASI site you can either leave the “Autotask account” configuration parameter set to the value entered in the "Autotask server defaults" (leave selection to “Use Default”) if you have a site that doesn’t have an Autotask account set up, or assign the Autotask account corresponding to each ASI site for those sites that have an Autotask customer account set up.

  1. Enter the Autotask server domain name, user name, and password, and click on the “Connect” button. The “Autotask Settings Configuration” page will be displayed.

  2. Follow the on-screen instructions carefully. When you have entered all of the required information, click on the “Save” button. The “Autotask Settings” save confirmation page will be displayed.

Notes

Default settings for all users on the ASI server can only be changed by administrative users.

When a notification with Autotask enabled runs for a particular user, the ASI server software looks for user specific Autotask settings first. If there are no user specific settings, it checks for default settings setup by an administrator. If these default settings do no exist, a note is made in the php log and the audit log to that effect. Please note that an ASI server where the Autotask support extensions have been newly installed does not have any Autotask settingsl.

Configuring event notifications for Autotask

  1. Log onto your ASI server (https://asiservername/main). The ASI dashboard will be displayed.

  2. Click on the "Use extended interface" link on the right side of the ASI Dashboard upper pane, below user login and date. The Welcome page will be displayed.

  3. At the upper right-hand corner of any page on the ASI server, click on the “notifications” link in the “events” navigation bar found the at upper right-hand corner of any page in the ASI server extended interface. The “Event Notifications” page will be displayed.

  4. To retrieve the notifications you want to configure for Auotask, in the "Name Contains" parameter in the "Search Options" panel, enter one or more keywords the name of these notifications have in common. If necessary, adjust the value of the “Page Size” parameter in the “Display Options” panel.

  5. Click on the Search button. The notifications you want to configure for Auotask will be listed in the “Event Notifications” page.

  6. Click on the “edit” action link , found in the “Action” column to the left of the name of the notification. The “Edit an Event Notification” page will be displayed (the name of the page will be slightly different if you are customizing a global event notification owned by another user).

  7. If you need to enable the event notification select “Yes” from the “Enabled” configuration parameter pull-down list

  8. Scroll down the “Edit a Notification” page until you see the “Autotask” configuration option. Click on its box. The “Per-site e-mail notifications” configuration parameter will also be automatically enabled, and grayed out. You will not be able to disable it.

  9. Review the other configuration settings of the event notification, and change them as needed carefully following the on-screen instructions. Once you have completed the review and changes of the event notification configuration, click on the “Update” button (if you are customizing a global event notification owned by another user, you will click on the “Add” button as you will be adding a local copy of the event notification which you will own).

Notes

If you add a notification, or copy a notification for Autotask, the event notification configuration steps are the same as those described in a) to i) above, except that the name of the page you will be configuring the notification on will be called “Add an Event Notification” and “Copy an Event Notification”, respectively.
