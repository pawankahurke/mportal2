Registry Management

  A BaseKey contains the beginning portion of the Key name.

  1.    Root Key is the first part of the key name. These are defined in Windows and can be selected from a pulldown menu in the "Base keys" wizard.
  2.    Sub Key is the second part of the key name. This is a key name that the user can edit, and determines the scope of this base key.


  A Filter selects a registry key for taking action.

  1.   The Filter Type controls which registry types are affected by an action. The user may choose to only affect registry keys or to affect both keys and values.
  2.    Base Key indicates which base key is used as the starting point for the affected key name.
  3.    Key is appended to the base key to give the key name. The entire key name is therefore composed of the root key (from the base key), followed by the sub key (from the base key), followed by the key. The key must start with a backslash ("\").
  4.    Key Wild indicates that the key name is treated as a wildcard, so a question mark ("?") matches a single character and an asterisk ("*") matches any number of characters. For example, with:
          Root key = HKEY_LOCAL_MACHINE
          Sub key = Software
          Key = \Microsoft\Windows\CurrentVersion\U*
       both these keys would be affected:
          HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\URL
       AND
          HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Uninstall
  5.    Value Name selects affected values based on their name.
  6.    Value Name Wild works like "Key Wild" but applies to the value name.
  7.    Value Type selects affected values based on their data type. The available data types are selected from a list in a pulldown menu.
  8.    Value Data selects affected values based on the data they contain.
  9.    Value Data Wild works like "Key Wild" but applies to the value data.
  10.    Global indicates that all users have access to a filter. This option is only available to administrative users and can only be changed by the user that created the filter.
  11.    Recursive indicates that the filter applies to all keys under the selected keys as well as the selected keys themselves.


  Action allows the user to take action on the selected attribute. For example:

  1.   selecting the action button next to a base key shows all the actions that apply to that base key.
  2.   selecting the action button next to a machine shows all the actions that apply to that machine. For example, if a user selects List Action, then they would be prompted with:
    1.   A pull down box of filters to choose from. A list of keys is generated using those that are selected by the chosen filter.
    2.   Max Affected specifies a limit on the number of registry keys that will be affected by this action.
    3.   Clear indicates that any existing values for this registry key will be cleared and replaced with the data from the registry; otherwise, the data from the registry is cumulative.


  Settings control visual settings of the left pane.

  1.   Selecting Restore Factory Settings deletes the contents of the left pane and restores the display to its initial condition.


  List Action displays all relevant properties of the selected action in the right pane and allows modifying that action.

  Machine displays all actions that have run on the selected machine in the right pane. Note that actions that have been deleted are not displayed.

  Site displays all actions that have run at the selected site in the right pane. Note that actions that have been deleted are not displayed.

  Registry Key displays all the values in the selected key in the right pane. Note that the machine from which registry values are taken is shown in the left pane hierarchy.
