Advanced Scrip Configuration Action Status Page

In the Scrip Configuration Action Status page you can access the latest Scrip configuration changes made on your ASI server. By latest Scrip configuration changes we mean, for example, that if you change the configuration of Scrip 96 on system A, call it change X, then 10 minutes later you change it again, call it change Y, the Scrip Configuration Action Status page will display first change X, then 10 minutes later, will display change Y and remove change X from the page, only if change X and change Y are the same change. If Scrip configuration changes X and Y did not change the same Scrip configuration variable(s), they would generate two entries in the Scrip Configuration Action Status page, even if both are changes to the same Scrip on the same system.

In the advanced Scrip Configuration Action Status page, Scrip configuration change actions are broken down into individual machine/Scrip configuration variable combinations. For example, if you made a change to three configuration variables in one Scrip on a group of ten machines, this would result into 30 entries in the advanced Scrip Configuration Action Status page.

On each Scrip configuration change entry listed in the Scrip Configuration Action Status page, you can perform a number of actions.

[modify expiration]

  • You can set an expiration date for one Scrip configuration change (individual change option). Once the expiration date passes, the ASI server software automatically reverts the Scrip configuration change in the entry assigning to the Scrip configuration variable for the machine in the entry the value attached to that variable in the group in the Original Group column.

  • You can set the expiration date for all Scrip configuration changes performed on a variable (variable change option). When the expiration date for all Scrip configuration changed performed on a variable passes, the Scrip configuration change in the entry on all machines that took the value for the variable in the entry from the group in the Change Group column, will be reverted to the value of that variable attached to the group(s) in the Original Group column.

For example, if the Advanced Scrip Configuration Action Status page contains ten Scrip configuration changes where the value of the Percent processor usage above which to report configuration variable attached to group A was assigned to ten different machines at different points in time, if you set an expiration date for all of these changes, when the expiration date passes the value of the Percent processor usage above which to report configuration variable for the ten machines will be reverted to the value attached to the groups it had been assigned from before the Scrip configuration changes took place.

  • You can also set an expiration date for the Scrip configuration changes that used variable values attached to the group listed in the Change Group column (group assignment option). When you do this, once the expiration date passes, the ASI server software will automatically revert all configuration changes that assigned to Scrip configuration variables values attached to the Change Group, assigning to them values attached to the group(s) in the Original Group column. Please note that once a group expiration date passes Scrip configuration variables in the affected entries can take values from multiple Original Group entries.

[revert change]

  • Selecting the individual change option, you can revert one Scrip configuration change assigning to the Scrip configuration variable for the machine listed in the entry the value of that variable attached to the group in the Original Group column.

  • You can perform a revert variable action (variable change option). Doing this will result in reverting the Scrip configuration change in the entry on all machines that took the value for the variable in the entry from the group in the Change Group column, to the value of that variable attached to the group(s) in the Original Group column.

For example, if the Advanced Scrip Configuration Action Status page contains ten Scrip configuration changes where the value of the Percent processor usage above which to report configuration variable attached to group A was assigned to ten different machines at different points in time, if you perform a revert variable action, the value of the Percent processor usage above which to report configuration variable for the ten machines will be reverted to the value attached to the groups it had been assigned from before the Scrip configuration changes took place.

  • Selecting the group change option, you can perform a revert group change action reverting all configuration changes that assigned to Scrip configuration variables values attached to the Change Group, reverting them to the values attached to the group(s) in the Original Group column. Please note that a revert group change action can assign to variables values from multiple Original Group entries.

For example, if the Scrip Configuration Action Status page contains three changes where the value of Scrip configuration variable X attached to group A was assigned to three different machines, if you perform a group revert action, all three changes will be reversed assigning to the three machines the value of configuration variable X from the group(s) it had been assigned from prior to the change, i.e. the group(s) in the Original Group column for the three Scrip configuration changes.

[delete group]

  • When you perform a delete group action, the ASI server software performs a revert group change action on the group in the Change Group column, and deletes the group. All configuration variables with values assigned from the Change Group, will take the value currently attached to the group(s) listed in the Original Group column. Please note that you cannot delete built-in groups (All, Sites, User:X, and Machines) but you can revert Scrip configuration changes involving built-in groups.

Important note

Please note that when an expiration date passes, or you revert a change, the value of the Scrip configuration variable whose value is reverted will not necessarily take on the value it had before you made the Scrip configuration change. This is because since the Scrip configuration change took place, the value of that Scrip configuration variable attached to the group(s) in the Original Group column, the one(s) they are being reverted to, may have changed.

Legend

Site – The name of the site the system where the Scrip configuration change was made belongs to

Machine – The name of the system where the Scrip configuration change was made

Configuration date – The date and time on which the Scrip configuration change was made

Expiration date – The date on which the Scrip configuration change will be reverted. The default value is Never.

Scrip – The number of the Scrip whose variable was changed by the Scrip configuration change

Scrip configuration variable changed – The Scrip configuration variable whose value changed when the Scrip configuration took place, i.e. it was assigned from the group in the Change Group column

Original Group – The group(s) that the variable in the Scrip Configuration Variable Changed column took its value from before Scrip the configuration change took place

Original Value – The value(s) that were attached to the group(s) in the Original Group column, and assigned to the variable in the Scrip Configuration Variable Changed column before the Scrip configuration change took place

Change Group - the group that the variable listed in the Scrip Configuration Variable Changed column took its value from when the configuration change took place

Change Value – The value that was attached to the group in the Change Group column, and assigned to the variable in the Scrip Configuration Variable Changed column when the Scrip configuration change took place

Source – The ASI module which was used to make the Scrip configuration change. The possible values of the Source field are:

  • Machine Scrip configurator

  • Scrip configuration wizard

  • Advanced Scrip configuration wizard

  • Remote control wizard

  • Enable-disable malware protection wizard

  • Enable-disable Microsoft updates wizard

  • Scrip execution frequency wizard
