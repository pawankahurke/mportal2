Scrip 240 coverage

System configuration variables managed by Scrip 240 are divided into two groups manipulated via different sets of configuration parameters.

The Currently enabled items on this machine (for review and known), Currently disabled items on this machine , Items to enable, and Items to disable configuration parameters manage the following system configuration variables:

  • Autoexec.bat

  • Shell= system.ini setting for systems running Microsoft Windows 9x based operating systems

  • Shell extension handlers

  • RunOnce

  • RunOnceEx

  • RunServicesOnce registry keys

  • Browser Helper Objects

  • Internet Explorer Bars

  • Internet Explorer Extensions

  • Internet Explorer Toolbars

  • Hosts file

The Current configuration settings on this machine (for review and known), and Configuration settings to enforce configuration parameters manage the following system configuration variables:

  • Start Page settings

  • Search Pages settings

  • Open verb commands

  • Screen saver settings

  • Critical Winlogon registry keys (Shell, Userinit and DllName) for systems running Microsoft Windows NT based operating systems (NT4, 2000, XP, or Server 2003)

Currently enabled items on this machine

To facilitate, and speed up the review of a system's environment managed by Scrip 240, items currently enabled on a system are divided into two groups:

  • Currently enabled items on this machine - for review

  • Currently enabled items on this machine – known

The Currently enabled items on this machine - for review configuration parameter contains items that do not match any of the entries in the Known items configuration parameter. The Currently enabled items on this machine – known configuration parameter contains items that match entries in the Known items configuration parameter.

When reviewing a system's environment, you can focus your review on the contents of the Currently enabled items on this machine - for review configuration parameter, ignoring the contents of the Currently enabled items on this machine – known configuration parameter. However, we suggest that periodically you review the content of the Currently enabled items on this machine – known configuration parameter to ensure that one or more malware items have not been inadvertently included in the Currently enabled items on this machine – known configuration parameter.

Currently enabled items on this machine configuration parameters (for review and known) entries can have one of the following formats:

  • RegKey, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • Key, followed by a comma with no spaces

  • Key variable name, followed by a comma with no spaces

  • Key variable value

or,

  • IniFile, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • .ini file (either win.ini or system.ini) path, followed by a comma with no spaces

  • .ini file section, followed by a comma with no spaces

  • .ini file variable name, followed by a comma with no spaces

  • .ini file variable value

or,

  • TxtFile, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • Text file path, followed by a comma with no spaces

  • Text file line, followed by a comma with no spaces

or,

  • ExtObj, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • Friendly name, followed by a comma with no spaces

  • Key name, followed by a comma with no spaces

  • Key variable name, followed by a comma with no spaces

  • Key variable data, followed by a comma with no spaces

  • Associated DLL

The Currently enabled items on this machine configuration parameter entries (for review and known) are display-only. They report the content of the disabled items in the start-up environment on the system whose Scrip 240 configuration page your are viewing. Editing the Currently enabled items on this machine configuration parameters (for review and known) has no effect, as their content will be regenerated when Scrip 240 runs.

Note on adding item entries and wild card support

Please note that identical items for different users show up as different entries in the Currently enabled items on this machine and Currently disabled items on this machine configuration parameters, and are handled as separate items in the Items to enable and Items to disable configuration parameters. This allows you to enable or disable a given system configuration item only for a certain user or group of users.

To facilitate the addition of entries to the Items to enable and Items to disable configuration parameters, Scrip 240 supports the use of the * wild card character. For example, if you want to disabled occurrences of 

RegKey,Administrator,HKEY_USERS\S-1-5-21-1855000883-1368534861-315576832-11656\Software\
Microsoft\Windows\CurrentVersion\RunOnce,*MS Setup,C:\WINNT\system32\bkinst.exe ren time:1098905100

for all users on one system or all systems at a site, in the Items to disable configuration parameter you should enter:

RegKey,*,HKEY_USERS\S-1-5-21-1855000883-1368534861-315576832-11656\Software\Microsoft\Windows\CurrentVersion\RunOnce,*MS Setup,C:\WINNT\system32\bkinst.exe ren time:1098905100

Please note that the wild card character can be used only as a value in the User Name field.

Wild card character use as described in this Note on adding item entries and wild card support section, is also supported in the Configuration settings to enforce configuration parameter.

Known items

The Known items configuration parameter contains all start-up environment items which you know to be safe. This does not mean that you may not want to disable items that are in the Known items configuration parameter. All it means is that you do not have to be concerned about any of them being malware.

You can add entries to the Known items configuration parameter manually making sure that they have the format described below. However, we advise against doing this because of the chance of inadvertent error.

The easiest and most reliable way to add entries to the Known items configuration parameter, is to copy safe entries in the Currently enabled items on this machine - for review configuration parameter, or from the Current configuration settings on this machine - for review configuration parameter, and paste them on a new line in the Known items configuration parameter.

After you add entries to the Known items configuration parameter, changes you have made (i.e. additions to the Currently enabled items on this machine – known configuration parameter and the Current configuration settings on this machine - known configuration parameter, and deletions from the Currently enabled items on this machine – for review configuration parameter and the Current configuration settings on this machine - for review configuration parameter, will take effect after you click on the the Update local variables only Execute button, or when you click on the Update intrusion protection now Execute button.

Please note that since the Known items configuration parameter can be global, you can have one known items list for all your sites. If you leave the Known items configuration parameter set to global, any additions made to the parameter will immediately propagate to all systems at a site. In addition, you can use the export function to propagate the contents of the Known items configuration parameter from one system to all sites.

We suggest that you maintain a master known items list in an ASCII text file and update it as you find start-up items that are known to be safe. Periodically, you can copy the master list, paste it into the Known items configuration parameter for a system at one of your sites (at that point it will propagate automatically to all other systems at the site), and then export that system's Scrip 240 configuration to all other sites.

Note on configuration settings

You should take special care when modifying Scrip 240's configuration to change the items in the Current configuration settings on this machine configuration parameter. If you choose to delete entries such as

RegKey,,HKEY_LOCAL_MACHINE\Software\Microsoft\Internet Explorer\Main,Start Page,http://www.microsoft.com/isapi/redir.dll?prd={SUB_PRD}&clcid={SUB_CLSID}&pver={SUB_PVER}&ar=home

and

RegKey,adonnini,HKEY_USERS\.Default\Software\Microsoft\Internet Explorer\Main,Start Page,about:blank

respectively a system's, and user's MS Internet Explorer start page settings, you will render MS Internet Explorer on the system unusable.

If you delete only a user's MS Internet Explorer start page but not a system's MS Internet Explorer start page users on the system will still be able to use MS Internet Explorer as the browser will default to the system's start page.

The above consideration is also true for other MS Internet Explorer configuration settings although deletion of the others does not have the same drastic impact of deletion of the start page setting.

The best way to control MS Internet Explorer Start, Search, Customize Search, and Search Assistant page configurations for both a system and its users is to use the Configuration settings to enforce configuration parameter as described in the preceding section.

Update local variables only

Clicking on the Update local variables only Execute button, will cause Scrip 240 to update the content of the Currently enabled startup items on this machine configuration parameters (for review and known) and the Current configuration settings on this machine configuration parameters (for review and known), and post an event log on the ASI server with the the up-to-date content of a system's configuration areas covered by Scrip 232. This function can be useful for verifying that Scrip 232 (Intrusion Protection control) is protecting a system's configuration areas targeted by intruders without having Scrip 240 clean-up them up, which would prevent you from verifying that Scrip 232 is working.

Update intrusion protection now

Clicking on the Update intrusion protection now Execute button, triggers Scrip 240 immediate execution, even if it's not enabled. The Update intrusion protection now Execute button can be used to execute the Scrip both on the local system, or on all systems at a site. When you click on the Update intrusion protection now Execute button, Scrip 240 runs immediately, regenerating the content of the Currently enabled items on this machine (for review and known), Currently disabled items on this machine parameters, and Current configuration settings on this machine configuration parameters (for review and known). It also enables/disables the system configuration items identified in the Items to enable, and Items to disable configuration parameters, respectively, and enforces the configuration settings contained in the Configuration settings to enforce configuration parameter.

Note about Scrip 240 execution

Scrip 240 runs when one of the following events occurs:

  • When the ASI client starts-up

  • When you click on the Update intrusion protection now Execute button, or on the Update local variables only Execute button

  • When a user logs on. Having Scrip 240 run when a user logs on has the side effect of having it run on ASI client start-up. This is because the user that is currently active is seen as a new user when the ASI client starts up.

  • When it is enabled. When this happens it updates the configuration.

  • Whenever the schedule changes

  • At its scheduled time

When Scrip 240 runs the Items to enable, Items to disable, and Configuration settings to enforce parameters are processed first. After the appropriate system configuration items are enabled or disabled as per the configuration, then the content of the Currently enabled items on this machine and Currently disabled items on this machine configuration parameters is updated.

Note on enabling and disabling items

The easiest way to disable a system configuration item is to copy it from the Currently enabled items configuration parameter, and paste it into the Items to disable configuration parameter. To re-enable an item you can move it (cut and paste) from the Items to disable configuration parameter to the Items to enable configuration parameter.

Please note that you cannot re-enable TxtFile items as described above.

While this method of enabling and disabling start-up items is certainly not the most elegant, it is straightforward and efficient. For example, in order to disable a system configuration item on all systems at a site you only need to add an entry to the Items to disable configuration parameter on one system.

In order to add entries to the Configuration settings to enforce configuration parameter minimizing the probability of making an error, you should perform the following steps:

  • Click on the list all link to the right of the Configuration settings to enforce configuration parameter on the Scrip 240 configuration page on any system

  • Locate the RegKey or IniFile item with a value the configuration setting you want to enforces

  • Copy the desired RegKey or IniFile item

  • Go back to the Scrip 240 configuration page by clicking on your browser Back button

  • Paste the RegKey or IniFile item you copied in the Configuration settings to enforce configuration parameter text box

  • Click on the Submit Changes button if you want to record the changes, or the Execute button if you want to enforce the configuration change immediately

When performing any of the above tasks, please pay special attention to the local/global settings of each variable whose value you are changing. Otherwise, you run the risk of making changes to all systems at a site, when you don't want to, or making changes on only one system when, in fact, the changes should apply to all systems at a site. You should use the same care when executing Scrip 240 on-demand using the Execute button.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
