Scrip 208 Configuration

Configuration Parameters

  • Scrip disabled
  • Names of machines to perform update all machines
  • When enabled, the Scrip runs at 5 AM and 4 PM daily with a random delay of one hour
  • Download software update now

In order for Scrip 208 to run on a system, it has to be enabled, AND its name has to be in the Names of machines to perform update list.

Names of machines to perform update

You use the Names of machines to perform update list to control which systems should download the ASI software and Scrip update. If you want all systems on a sub-net to be download ASI software and Scrip updates, all you need to enter in the list is a *. An empty list is equivalent to the Scrip being de-activated on all systems.

If a system is not in the Names of machines to perform update list, it will not attempt to download ASI software or Scrip updates, even though Scrip 208 may be enabled on it.

When you want to use more than one system to download ASI software and Scrip updated, each system's name should be on a separate line.

The name of each system that you want to use for downloading ASI software and Scrip updates can be found in the Computer name window on the Identification panel in the Network properties in the Microsoft Windows / Windows NT Control Panel. Alternatively, you can simply click on the Home link at the top left-hand corner of the Scrip configuration page. Doing this will take you to them main Scrip configuration page which lists the names of all systems on which the Automated Support Infrastructure client is installed.

Download software update now

With the Execute button, you can run Scrip 208 on demand, even if it's not enabled.

However, if you attempt to run Scrip 208 using the Execute button on a system where the Scrip is enabled but not in the Names of machines to perform update list, the Scrip will not run, and post a log on the event log management database stating that the system where you tried to run Scrip 208 is not in the Names of machines to perform update list.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1Minutes

0-60

2Hours 1-24
3Day of the Month 1-31, depending on the month; a value of * means that the Scrip should be run every day.
4Month 1-12; a value of * means that the Scrip should be run every month.
5Day of the Week 0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.
6Minute Randomizer N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.
7Conditional Execution N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.
8Failed Run Limit N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.

The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
