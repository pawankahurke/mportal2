Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



Because all Scrip 241 information entry actions can be either local or global, before executing any of them it's very important that you make sure that they are set to global or local, depending on what you want to do.

Ask user for information again

Clicking on the Execute button to the right of the Ask user for information again configuration parameter will cause Scrip 241 to immediately generate a dialog box asking the end user on the system(s) where you decided to run it for contact information. The Scrip will continue to generate the dialog box until the end-user enters at least one of the requested information items, or simply clicks on the OK button.

You control the Scrip's execution cycle via configuration parameters Number of seconds to leave up dialog box (zero for no limit), and Number of seconds after which to retry (zero for no retry). Please see below for more information on this two configuration parameters.

Should user dialog box be minimized and flashing? (less obtrusive)

If you enable the Should user dialog box be minimized and flashing? (less obtrusive) configuration parameter, the dialog box asking for information generated by Scrip 241 will be minimized onto the taskbar at the bottom of a system's desktop display. The minimized dialog box will be flashing. The rate of flashing will be determined by the Flashing interval if user dialog box is minimized and flashing (seconds) configuration parameter described below.

Flashing interval if user dialog box is minimized and flashing (seconds)

The Flashing interval if user dialog box is minimized and flashing (seconds) configuration parameter lets you set the interval between flashes of the minimized dialog box asking for information generated by Scrip 241. The default flashing interval is set to 3 seconds. This means that the minimized dialog box will flash once every three seconds.

Title for user dialog box

In the Title for user dialog box parameter, you enter the header of the dialog box asking for the information you would like an end-user to provide.

Descriptive text to display in user dialog box

The Descriptive text to display in user dialog box configuration parameter lets you customize the message inserted at the top of the dialog box asking an end-user for information.

Number of seconds to leave up dialog box (zero for no limit)

In the Number of seconds to leave up dialog box (zero for no limit) configuration parameter you enter the amount of time (in seconds) for which the dialog box asking an end-user for information will persist on the system's screen. By default, the Number of seconds to leave up dialog box (zero for no limit) configuration parameter is set to 900 seconds (15 minutes).

Number of seconds after which to retry (zero for no retry)

The Number of seconds after which to retry (zero for no retry) configuration parameter lets you set the length of the Scrip 241 retry cycle. This is the amount of time, Scrip 241 will wait before running again. It runs again if:

  • An end-user has not entered any information in the dialog box before the time you specify in the Number of seconds to leave up dialog box (zero for no limit) configuration parameter expires and the Ask again if user ignores or cancels configuration parameter is enabled

  • When an end-user clicks on the Cancel button in the dialog box asking for information and the Ask again if user ignores or cancels configuration parameter is enabled

By default, the Number of seconds after which to retry (zero for no retry) configuration parameter is set to 14400 seconds (4 hours).

Ask again if user ignores or cancels

When the Ask again if user ignores or cancels configuration parameter is enabled, Scrip 241 will generate the dialog box asking for information after the period of time specified in the Number of seconds after which to retry (zero for no retry) configuration parameter expires.

This configuration parameter is enabled by default.

Note about Scrip 241 execution

Scrip 241 runs when one of the following events occurs:

  • When the ASI client is installed or upgraded and the ASI client installation executable command line contains the /CI=1 parameter.

  • When you click on the Execute button to the right of the Ask user for information again label.

  • At the end of the end of the time you specify in the Number of seconds after which to retry (zero for no retry) configuration parameter when an end-user takes no action and the Ask again if user ignores or cancels configuration parameter is enabled.

  • At the end of the end of the time you specify in the Number of seconds after which to retry (zero for no retry) configuration parameter when an end-user clicks on the Cancel button in the dialog box asking for information and the Ask again if user ignores or cancels configuration parameter is enabled

  • When you change the value of one or more of the information fields, Scrip 241 will run immediately after you click on the Submit Changes button, and upload the changed values to the ASI server. Please refer to the Information field content section of this configuration help file for more information on the information fields.

Whenever Scrip 241 runs it posts a log onto the ASI server containing the following information:

  • Which of the above events caused its execution

  • The information entered by the end-user

  • The current Scrip configuration

  • Whether it's on a retry cycle, i.e. it scheduled itself to run again
