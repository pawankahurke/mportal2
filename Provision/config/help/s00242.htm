Note on eTrust EZ Antivirus virus definition management

Unlike other anti virus software products ASI supports, eTrust EZ Antivirus software does not keep the date associated with virus definitions anywhere, not in the registry, not on a disk file, and not in the dat file version info. eTrust EZ Antivirus does not even display the date of the current virus definitions in the Help->About dialog box.

Instead, eTrust EZ Antivirus software virus definition updates are driven strictly by the version number of the definitions.

When deciding whether to update virus definition on a system, Scrip 242 uses the version of the local virus definitions on a system, and the version of the latest virus definitions available at a site. There appears to be no way around this. We even tried extracting the File Creation date and File Last Modified date of the definition files themselves but it turns out that eTrust EZ Antivirus does not set those dates correctly.

Scrip 242 compares dat version numbers and not dat dates. Although the Scrip can't know the date of any set of virus definition files, it can track the date when the definitions were last updated (whether they were updated by Scrip 242 or not). This means that instead of the Scrip 242 configurator containing the configuration parameter Maximum age of virus definitions in days like Scrips 12 and 90 it contains the configuration parameter Maximum days since last definitions update. Scrip 242 will post an event log onto the ASI server when then number of days since a virus definition update occurred is equal to or greater than the value you set in the Maximum days since last definitions update configuration parameter instead of logging when the virus definitions are older than a maximum number of days as Scrips 12 and 90 do.

Update from other accessible machines

When you enabled the Update from other accessible machines configuration parameter, Scrip 242 will replace the eTrust EZ Antivirus virus definitions on each system where this option is enabled with the newest virus definitions found on the sub-net where the system is installed. If none of the other virus definition update functions are enabled, Scrip 242 ends execution after this operation is completed.

Update from on-site server

An on-site server is a system where eTrust EZ Antivirus is installed, and that is known to consistently have up-to-date virus definitions. For example this could be one system (or more than one) where the virus definitions are updated manually to ensure that the update is successful.

If the Update from on-site server configuration parameter is enabled, Scrip 242 will look for newer virus definitions at the location(s) specified in the On-site server locations list. If the Update from other accessible machines configuration parameter is enabled searching for newer virus definitions at location(s) specified in the On-site server locations configuration parameter occurs after Scrip 242 has completed the search for newer virus definitions on other accessible systems on the sub-net.

Update from vendor

When the Update from vendor configuration parameter is enabled, Scrip 242 will look for newer virus definitions from the Computer Associates site using eTrust EZ Antivirus software. If Scrip 242 cannot find eTrust EZ Antivirus software, it logs this fact, and ends execution. If either Update from other accessible machines or Update from on-site server, configuration parameters, or both, are enabled, Scrip 242 will look for newer virus definitions from the Computer Associates site only if it was not successful finding newer virus definitions using the other two options.

On-site server locations

Each entry in the On-site server locations configuration parameter contains the full path for an on-site server. Each entry should be on a separate line.

Version of latest definitions

The Version of latest definitions parameter is displayed for information only. It should not be modified under any circumstances. If it were to be modified, Scrip 242 would not function properly.

The purpose of the Version of latest definitions configuration parameter is to track the virus definition version for all systems running eTrust EZ Antivirus software. As a result, when a machine is searching for updates on other systems on the sub-net, it simply accesses this list. It does not have to contact every machine on the subnet. This minimizes local network traffic and bandwidth usage. From the list's entries a system finds out the dates of virus definitions available on the sub-net, and which system(s) has/have them.

The format of Version of latest definitions configuration parameter entries is as follows:

  • version number

For example:

  • 8723

Machines to report virus definition dates

You use the Machines to report virus definition dates configuration parameter to control which systems should report virus definition dates on the systems on the sub-net. If you want all systems on a sub-net to report virus definition dates, all you need to enter in the list is a *. An empty list is equivalent to this function being disabled.

Maximum days since last definition update

When the date of the eTrust EZ Antivirus virus definition database files is older than the current date by the number of days specified by the Maximum days since last definitions update configuration parameter, or more, Scrip 242 will post this information on the ASI server.

Please that the date used by Scrip 242 for this function is the last modified date of the eTrust EZ Antivirus virus definition files.

Maximum random delay before update (minutes)

The Maximum random delay before update (minutes) configuration parameter helps minimize the likelihood that if only one system has the latest (or newer) virus definitions, this system not be flooded with requests for the newer virus definitions from other systems. When the ASI client on a system successfully retrieves virus definitions from the vendor's site, Maximum random delay before update (minutes) ensures that requests for the newer virus definitions from other systems do not occur simultaneously, maxmimizing the probability the the system be able to service at least one of the requests. Once that happens, the number of systems that can provide the newer virus definitions is automatically doubled (at least), ensuring that all requests will be fulfilled.

Maximum retries for obtaining local updates

As its name suggests, the Maximum retries for obtaining local updates configuration parameter lets you set the number of times Scrip 242 will attempt to obtain the newest virus definitions available on the sub-net from other systems on the sub-net. By default, its value is set to 3.

Local update retry delay (minutes)

With the Local update retry delay (minutes) configuration parameter, you set the time interval between attempts Scrip 242 will make to obtain the newest virus definitions available on the sub-net from other systems on the sub-net. By default, its value is set to 60 minutes.

Report when anti-virus product is not installed

If you enable the Report when anti-virus product is not installed configuration parameter, on systems where eTrust EZ Antivirus is not installed Scrip 242 will post a log on the ASI server with this information.

You might want to use this configuration parameter periodically to find out which systems at a site do not have anti virus software installed. In this way, you would be able to intervene pro-actively and add anti virus protection to unprotected systems before a virus infects them.

Command line parameters

Scrip 242 runs in silent mode with the -s parameter in the command line. The -s parameter cannot be deleted.

Update eTrust EZ Antivirus now

With the Execute button, you can run Scrip 242 on demand, even if it's not enabled. In Scrip 242, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
