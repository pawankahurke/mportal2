Scrip 95 Configuration

Configuration Parameters, and their default values

  • Scrip enabled enabled

  • Absolute threshold (KB or %) (e.g. 100000 or 10%)

  • Change threshold (KB)

  • Number of samples for change threshold 1

  • Percentage of free space below which to report for each drive (used for compatibility) 10

  • When enabled, Scrip 95 runs once per hour at 20 minutes past the hour with a random 3-minute delay. This means that Scrip 95 will run from 20 to 23 minutes past the hour.

  • Report disk space (logical) now

The Scrip execution interval can be changed by simply changing the default schedule.

Please note that when either the Absolute threshold (KB or %) (e.g. 100000 or 10%) or Change threshold (KB) configuration parameter is empty, the corresponding function is disabled.

Absolute threshold (KB or %) (e.g. 100000 or 10%)

The Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter is used to alert you about low available logical disk capacity conditions and effectively regulate the Scrip's logging activity.

The value of the Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter can be expressed either in terms of the absolute amount of available disk capacity expressed in KB (e.g. 100000), or in terms of the % of available disk capacity (e.g. 10% -- note the % appended to the number).

When the value of the Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter is expressed in terms of the % of available disk capacity, it can vary between 100% and 0%. When it is set to 100%, this means that Scrip 95 will log available logical disk capacity statistics once per hour (the default Scrip 95 execution schedule). When the value of the Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter is set to X%, where X is between 0 and 100, this means that Scrip 95 will log available logical disk capacity statistics once per hour, if its amount falls below X%. When the value of the Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter is set to 0%, in practice this means that Scrip 95 will log available logical disk capacity statistics very rarely.

When the value of the Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter is expressed in terms of the absolute amount of available disk capacity expressed in KB, it can vary between 0 and a number as large as you like. When it is set to a number greater than the capacity of the largest logical drive on a system, this means that Scrip 95 will log available logical disk capacity statistics once per hour (the default Scrip 95 execution schedule). When the reporting threshold is set to X, where X is between 0 and the capacity of the largest logical drive on a system, this means that Scrip 95 will log available logical disk capacity statistics once per hour, if its amount falls below X. When the value of the Absolute threshold (KB or %) (e.g. 100000 or 10%) configuration parameter is set to 0, in practice this means that Scrip 95 will log available logical disk capacity statistics very rarely.

Change threshold (KB)

You can use Scrip 95 to warn you when available disk capacity decreases by the amount specified in the Change threshold (KB) configuration parameter expressed in number of KBytes.

Every time Scrip 95 runs, for each logical drive on a system, it finds the largest of the available disk capacity values it has retained as specified by the value of the Number of samples for change threshold configuration parameter. It calculates the decrease from that value to the current available disk capacity. If the decrease is greater than the value of the Change threshold (KB) configuration parameter , Scrip 95 will post an event log onto the ASI server.

Number of samples for change threshold

The Number of samples for change threshold configuration parameter represents the number of available disk capacity measurements retained by Scrip 95 for each of the logical drives on a system.

Please note that if you increase the value of the Number of samples for change threshold configuration parameter it will take some time to accumulate the increased number of samples (an amount of time equal to the sum of the interval between Scrip 95 executions necessary to collect a number of samples equal to the increased value of the Number of samples for change threshold configuration parameter). The number of samples retained will be smaller than the increased value of the Number of samples for change threshold configuration parameter for the amount of time it will take to collect a number of samples equal to the value of the Number of samples for change threshold configuration parameter.

Percentage of free space below which to report for each drive (used for compatibility)

The Percentage of free space below which to report for each drive configuration parameter is used to alert the support provider about low available logical disk capacity conditions and effectively regulate the Scrip's logging activity.

The value of The Percentage of free space below which to report for each drive configuration parameter can vary between 100 and 0. When it is set to 100, in practice this means that for Scrip 95 will log available logical disk capacity statistics once per hour (the default Scrip 95 execution schedule). When the reporting threshold is set to X, where X is between 0 and 100, this means that Scrip 95 will log available logical disk capacity statistics once per hour, if its amount falls below X. When the value of The Percentage of free space below which to report for each drive configuration parameter is set to 0, in practice this means that Scrip 95 will log available logical disk capacity statistics very rarely.

Report disk space (logical) now

With the Execute button, you can run Scrip 95 on demand, even if it's not enabled. In Scrip 95, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
