Perform polling now

With the Execute button, you can run Scrip 88 on demand, even if it's not enabled. In Scrip 88, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.

Machines to perform tests

Version 1.x of Scrip 88 allows device and services testing to be carried out from multiple systems whose names are entered by the user in the Machines to perform tests window.

When the name of more than one system is entered, each name should be on a separate line.

The name of each system you want to use to perform the tests can be found in the Computer name window on the Identification panel in the Network properties in the Microsoft Windows / Windows NT Control Panel. Alternatively, you can simply click on the Home link at the top left-hand corner of the Scrip configuration page. Doing this will take you to them main Scrip configuration page which lists the names of all systems on which the Automated Support Infrastructure client is installed.

Machines to be tested

The Machines to be tested parameter has a command line with six fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Enable / Disable

0 (disable) or 1 (enable); this makes it possible to temporarily stop testing a device without deleting it from the list

2

IP name or IP address of device

e.g. nanoheal.org

3

Protocol to Test

TCP or ICMP; TCP opens the port to the TCP/IP service using the port selected and ICMP does a ping

4

Port Number to Test

1-65635; only used for TCP. Set this parameter to 0 for a ping (using ICMP)

5

Number of Times

1-No practical limit; this is the number of times in a row the attempt to contact a device or test a service must fail in order for the device to be to be considered down

6

Timeout Delay (milliseconds)

1-No practical limit; this is the maximum time allowed for a response before the attempt to communicate with a device or test a service is considered a fail
