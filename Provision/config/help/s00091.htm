Scrip 91 Configuration

Configuration Parameters

  • Scrip enabled
  • Disks to scan Scrip 91 scans all local drives, except drives A: and B: ("My Computer" command line parameter)
  • Runs in "silent" mode (/UINONE command line parameter)
  • Scrip runs at 3am every Wednesday and Friday
  • Run McAfee VirusScan now

Disks to scan

You can change the value of this parameter by simply deleting the default value and entering the drive letter of each drive you want scanned followed by a colon, e.g. c:. The entry for each drive you want scanned should be on a separate line.

Command Line Parameters

Version 1.0 of Scrip 91 does not allow the addition of any McAfee VirusScan command line parameters to the execution of McAfee VirusScan except for drive selection.

/UINONE, the "silent" mode command line parameter, cannot be deleted.

Run McAfee VirusScan now

With the Execute button, you can run Scrip 91 on demand, even if it's not enabled. In Scrip 91, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1Minutes

0-60

2Hours 1-24
3Day of the Month 1-31, depending on the month; a value of * means that the Scrip should be run every day.
4Month 1-12; a value of * means that the Scrip should be run every month.
5Day of the Week 0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.
6Minute Randomizer N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.
7Conditional Execution N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.
8Failed Run Limit N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.

The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
