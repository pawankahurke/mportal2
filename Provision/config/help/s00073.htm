Scrip 73 Configuration

Configuration parameters and their default values

  • Scrip disabled

  • Action (0=restart,1=logoff,2=power off,3=restart apps,4=shutdown) set to 0

  • Names of machines to reboot list empty

  • Names of processes to shutdown list empty

  • Max number of seconds to wait for the process to close 30 seconds

  • Max number of dialogs to close 5

  • When enabled, the Scrip runs at 6 AM daily

  • Reboot system now

Action (0=restart,1=logoff,2=power off,3=restart apps,4=shutdown)

In the Action (0=restart,1=logoff,2=power off,3=restart apps,4=shutdown) configuration parameter, you enter the value corresponding to the action you want Scrip 73 to take when it runs. Specifically,

  • Action = 0

Shuts down, and then re-starts a system.

  • Action = 1

Shuts down all processes running in the logon session of the process that called the ExitWindowsEx function, then logs the user off.

  • Action = 2

Shuts down a system, and turns off the power. The system must support, and have enabled the power-off feature.

  • Action = 3

Shuts down the system and then restarts it, as well as any applications that have been registered for restart using the RegisterApplicationsRestart function. This appears to only be for systems running Vista, so it hasn't been tested yet.

  • Action = 4

Shuts down a system to a point at which it is safe to turn off the power. All file buffers have been flushed to disk, and all running processes have stopped.

Names of machines to reboot

Scrip 73 reboots the systems whose names you enter in the Names of machines to reboot list. If you want all systems on a sub-net to be rebooted all you need to enter in the list is a *. An empty list is equivalent to the Scrip being disabled.

When you want to reboot more than one system, each system's name should be on a separate line.

The name of each system you want to reboot can be found in the Computer name window on the Identification panel in the Network properties in the Microsoft Windows / Windows NT Control Panel. Alternatively, you can simply click on the Home link at the top left-hand corner of the Scrip configuration page. Doing this will take you to them main Scrip configuration page which lists the names of all systems on which the Automated Support Infrastructure client is installed.

Names of processes to shutdown

This parameter is used to list all the processes you would like Scrip 73 to shut down in an orderly fashion before starting the reboot sequence. The full name of each process to be terminated should be entered on a separate line. Full location paths are not required.

This configuration entry should be used with care. For example, you should not use either of the masks * or *.* because they would have the effect of terminating both the Client and kernel32.dll, which are necessary for executing Scrip 73 (the Client), and perform the system reboot (kernel32.dll).

At the time of a scheduled or on-demand system reboot, there may be processes running that need to be shut down, and that may require user action or response in order to be terminated. For example:

  • Open network connections that in order to be closed require user approval

  • Applications like Microsoft Word that require a user to save or close without saving files that have been changed and not saved at the time Word is terminated, or

  • Information agents like Intuit Quickbooks' qbdagent.exe.

Scrip 73 handles these situations automatically. You can control process shut-down with the Max number of seconds to wait for the process to close and Max number of dialogs to close

Max number of seconds to wait for the process to close

Once the system reboot sequence has been initiated, this parameter lets you control how long Scrip 73 should wait before shutting down each process that was active at the time.

Max number of dialogs to close

In many instances, requests for user action communicated via dialog boxes are part of process shut down sequences. Scrip 73 handles these requests automatically.

Whenever a dialog box appears during a process' shut down sequence requesting user input, Scrip 73 closes it taking the default action. For example, if the dialog box requests whether a file should be saved or not, and the default response is Yes, Scrip 73 will automatically click on the Yes button.

Reboot system now

With the Execute button, you can run Scrip 73 on demand, even if it's not enabled. In Scrip 73, the Execute button can be used to execute the Scrip only on the local system.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
