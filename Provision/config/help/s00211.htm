Scrip 211 Configuration

Configuration Parameters

  • Scrip Enabled

  • Disks to defragment set to All

  • Maximum run time (minutes) 120 minutes

  • Names of processes to shutdown list empty

  • Max number of seconds to wait for the process to close 30 seconds

  • Max number of dialogs to close five

  • Names of processes to startup list empty

  • Reboot system disabled

  • Scrip runs at 12.01am every Tuesday

  • Run disk defragmenter (Windows XP, .NET Server 2003) now

Disks to defragment

You can change the value of this parameter by simply deleting the default value and entering the drive letter of each drive you want defragmented followed by a colon, e.g. c:. The entry for each drive you want defragmented should be on a separate line.

Command Line Parameters

By default, Scrip 211 executes disk defragmenter in unattended mode. It does not stop and display confirmation messages (/noprompt command line parameter). All files and free space on all local, non-removable drives are defragmented (/all, /f command line parameters).

In order to defragment drives selectively, the drive selection parameter can be changed from /all to [drive]: where [drive] should be replaced with the drive letter of each logical drive you want to have defragmented.

When more than one drive letter is used, each [drive]: should be entered on a separate line.

Scrip 211 does not allow the addition of any Defragmenter command line parameters to the execution of Defragmenter except for drive selection.

The /noprompt and /f parameters (Windows NT4 only) cannot be deleted.

Maximum run time (minutes)

This parameter lets you control the maximum amount of time defragmenter will be allowed to run. When setting a value keep in mind that if a drive or drive(s) have not been defragmented for a long time, disk defragmentation will take a significant amount of time. If the disk drives are large and close to capacity, it could take several hours.

In setting a value for this parameter you should also keep in mind other Scrips that may be scheduled to run during the same time of day to avoid conflict.

Names of processes to shutdown

The Names of processes to shutdown parameter can be used to list all the processes you would like Scrip 211 to shut down in an orderly fashion before starting disk defragmentation. The full name of each process to be terminated, including extension, should be entered on a separate line. Full location paths are not required.

This configuration entry should be used with care. For example, you should not use either of the masks * or *.* because they would have the effect of terminating both the ASI Client and other processes (e.g. explorer.exe) that are necessary for executing Scrip 211 (the ASI Client), and keep the system running (explorer.exe).

One or more of the processes running at the time disk defragmentation is scheduled to start that need to be shut down may require user action or response in order to be terminated. For example:

  • Open network connections that in order to be closed require user approval

  • Applications like Microsoft Word that require a user to save or close without saving files that have been changed and not saved at the time Word is terminated, or

  • Information agents like Intuit Quickbooks' qbdagent.exe.

Scrip 211 handles these situations automatically. You can control process shut-down with the Max number of seconds to wait for the process to close and Max number of dialogs to close

Max number of seconds to wait for the process to close

This parameter lets you control how long Scrip 211 should wait before shutting down each process that was active at the time disk defragmentation was scheduled to start.

Max number of dialogs to close

In many instances, requests for user action communicated via dialog boxes are part of process shut down sequences. Scrip 211 handles these requests automatically.

Whenever a dialog box requesting user input appears during a process' shut down sequence, Scrip 211 closes it taking the default action. For example, if the dialog box requests whether a file should be saved or not, and the default response is Yes, Scrip 211 will automatically click on the Yes button.

Names of processes to startup

This parameter can be used to list all the processes you would like Scrip 211 to start up after disk defragmentation is completed. The full name of each process to be started, including extension, should be entered on a separate line. Full location paths are not required.

Reboot system

When this parameter is enabled, Scrip 211 will restart the system after disk dfragmentation is completed.

Parameters Names of processes to startup and Reboot system are independent of each other. If you want to restart the system after disk defragmentation is completed, Scrip 211 will ignore the list of processes to start up after disk defragmentation is completed. If you don't want to restart the system after disk defragmentation is completed, then Scrip 211 will look for process(es) to start once disk defragmentation is completed.

Run disk defragmenter (Windows XP, .NET Server 2003) now

With the Execute button, you can run Scrip 211 on demand, even if it's not enabled. In Scrip 211, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.

Microsoft defragmenter limitations

According to article Q314848 from the Microsoft knowledge base (http://support.microsoft.com/default.aspx?scid=kb;en-us;314848) The Microsoft XP disk defragmenter tool cannot be scheduled.

In addition, The Microsoft Windows XP Disk Defragmenter tool does not provide support for command-line parameters or switches to automate the disk defragmentation process

This means that the Automated Support Infrastructure provides an inexpensive way to automate this task on systems running Microsoft Windows XP without having to purchase a third-party disk defragmentation utility.
