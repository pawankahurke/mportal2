Delay in seconds from system startup (Scrip,seconds)

Some Scrips are executed when the ASI client starts up. In order to avoid overloading a system and degrading its performance, Scrip 3 delays the execution of selected Scrips for a fixed period of time until after the ASI client starts. Currently, Scrip 23 delays the execution of Scrips 8, 24, 65, 236, and 237. The Delay in seconds from system startup (Scrip,seconds) configuration parameter is not user configurable.

There should be no need to change the default value for the Delay in seconds from system startup (Scrip,seconds) configuration parameter. HandsFree Networks will inform you if this value needs to be changed.
