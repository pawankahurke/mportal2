Note on Trend Micro anti virus virus definition management

Unlike other anti virus software products ASI supports, Trend Micro anti virus software virus definitions dates are not accessible.

Trend Micro anti virus software virus definition updates are driven strictly by the version number of the definitions.

When deciding whether to update virus definition on a system, Scrip 247 uses the version of the local virus definitions on a system, and the version of the latest virus definitions available at a site.

Scrip 247 supports three Trend Micro products

1. PC-cillin

2. OfficeScan server

3. OfficeScan client

Both PC-cillin and OfficeScan use the same definitions so they can update each other using Scrip 247. PC-cillin can be installed on XP, 2K, or 9x. The OfficeScan client can be installed on XP, 2K or 9x but the server can only be installed on XP or 2K. It is not possible to install an OfficeScan client without first installing the OfficeScan server. Once the server is installed a client install executable can be created on the server or a client can be installed using the web console interface created by the server install. When a server is set up, the client also gets installed on the server machine to provide it with AV protection.

The OfficeScan client cannot be updated directly from the vendor; it can only receive vendor updates from the OfficeScan server. When Scrip 247 initiates a vendor update on a client it will look to the server for updates, and when Scrip 247 initiates a vendor update on the server, it will look to Trend Micro's web site. Therefore, the update process on the server machine involves two separate processes; the updating of the server from Trend Micro and the updating of the server's client from the server. The vendor-to-server update will update the Version of latest definitions configuration parameter causing the actual vendor update to propagate to other clients, including the server's client.

Update from other accessible machines

When you enabled the Update from other accessible machines configuration parameter, Scrip 247 will replace the Trend Micro virus definitions on each system where this option is enabled with the newest virus definitions found on the sub-net where the system is installed. If none of the other virus definition update functions are enabled, Scrip 247 ends execution after this operation is completed.

Update from on-site server

An on-site server is a system that is known to have the Trend Micro anti virus server software. Such a system is configured to automatically retrieve virus definition updates at regular intervals.

If the Update from on-site server configuration parameter is enabled, Scrip 247 will look for newer virus definitions at the location(s) specified in the On-site server locations list. If the Update from other accessible machines option is enabled searching for newer virus definitions at location(s) specified in the On-site server locations list occurs after Scrip 247 has completed the search for newer virus definitions on other accessible systems on the sub-net.

Update from vendor

When the Update from vendor configuration parameter is enabled, Scrip 247 trigger a virus definition update cycle on any system where the Trend Micro anti virus server software is installed. If Scrip 247 cannot find Trend Micro anti virus software, it logs this fact, and ends execution. If either Update from other accessible machines or Update from on-site server, configuration parameters, or both, are enabled, Scrip 247 will look for newer virus definitions from the Trend Micro site only if it was not successful finding newer virus definitions using the other two options.

On-site server locations

Each entry in the On-site server locations configuration parameter contains the netBIOS name or IP address for an on-site server. Each entry should be on a separate line.



Please note that if neither the Update from other accessible machines, or the Update from on-site server configuration parameters are enabled, Trend Micro virus definitions are not likely to be updated.

Version of latest definitions

The Version of latest definitions parameter is displayed for information only. It should not be modified under any circumstances. If it were to be modified, Scrip 247 would not function properly.

The purpose of the Version of latest definitions configuration parameter is to track the virus definition version for all systems running Trend Micro anti virus software. As a result, when a machine is searching for updates on other systems on the sub-net, it simply accesses this list. It does not have to contact every machine on the subnet. This minimizes local network traffic and bandwidth usage. From the list's entries a system finds out the dates of virus definitions available on the sub-net, and which system(s) has/have them.

The format of Versions of latest definitions configuration parameter entries is as follows:

  • version number

For example:

  • 2.741.00

Machines to report virus definition dates

You use the Machines to report virus definition dates configuration parameter to control which systems should report virus definition dates on the systems on the sub-net. If you want all systems on a sub-net to report virus definition dates, all you need to enter in the list is a *. An empty list is equivalent to this function being disabled.

Maximum days since last definition update

When the date of the Trend Micro anti virus virus definition database files is older than the current date by the number of days specified by the Maximum days since last definitions update configuration parameter, or more, Scrip 247 will post this information on the ASI server.

Please that the date used by Scrip 247 for this function is the last modified date of the Trend Micro anti virus virus definition files.

Maximum random delay before update (minutes)

The Maximum random delay before update (minutes) configuration parameter helps minimize the likelihood that if only one system has the latest (or newer) virus definitions, this system not be flooded with requests for the newer virus definitions from other systems. When the ASI client on a system successfully retrieves virus definitions from the vendor's site, the Maximum random delay before update (minutes) configuration parameter ensures that requests for the newer virus definitions from other systems do not occur simultaneously, maxmimizing the probability the the system be able to service at least one of the requests. Once that happens, the number of systems that can provide the newer virus definitions is automatically doubled (at least), ensuring that all requests will be fulfilled.

Maximum retries for obtaining local updates

As its name suggests, the Maximum retries for obtaining local updates configuration parameter lets you set the number of times Scrip 247 will attempt to obtain the newest virus definitions available on the sub-net from other systems on the sub-net. By default, its value is set to 3.

Local update retry delay (minutes)

With the Local update retry delay (minutes) configuration parameter, you set the time interval between attempts Scrip 247 will make to obtain the newest virus definitions available on the sub-net from other systems on the sub-net. By default, its value is set to 60 minutes.

Report when anti-virus product is not installed

If you enable the Report when anti-virus product is not installed configuration parameter, on systems where Trend Micro anti virus is not installed Scrip 247 will post a log on the ASI server with this information.

You might want to use this configuration parameter periodically to find out which systems at a site do not have anti virus software installed. In this way, you would be able to intervene pro-actively and add anti virus protection to unprotected systems before a virus infects them.

Update Trend Micro anti virus now

With the Execute button, you can run Scrip 247 on demand, even if it's not enabled. In Scrip 247, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
