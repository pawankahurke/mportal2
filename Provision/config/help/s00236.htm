Definitions and important notes

RealVNC users please note. Although Scrip 236 supports UltraVNC, it will not interfere with an existing RealVNC installation. In fact, you can use your RealVNC client running in listen mode on the remote computer to take control of the target system where the UltraVNC server installed by Scrip 236 is running.

Please note that before you use Scrip 236 to initiate remote control of the system whose Scrip 236 configuration page you are accessing (target computer), you will need to run the remote control software in listen mode on the computer (remote computer) from which you will be controlling the target computer. It is important to ensure that only ONE remote control software client process is running at any one time on the remote computer.

This means that the remote control software needs to be installed on the remote computer (remember, this is the system that takes over the target computer) before you use Scrip 236.

If the remote computer is accessing a target computer at an external site, i.e. a site accessible via the Internet, and the remote computer does not have a public IP address, the entry for the Address of remote computer configuration parameter should be the public IP address or URL of the Internet gateway device used by the remote computer.

In addition, the firewall for the network where the remote computer is located should be configured to forward traffic incoming via port 5500 (the default port used by the remote control software) to the remote computer.

For example, suppose you want to configure Scrip 236 to initiate a remote control session from your computer (remote computer) on server X at customer site ABC. Let's also assume that your computer has a private IP address (e.g. 192.168.1.250), and that the Internet gateway at your site is a device with the service.provider.com URL.

The entry in the Address of remote computer configuration parameter would be service.provider.com (or its public IP address). In addition, the firewall on your network would be configured to forward incoming port 5500 (or whichever port you have specified in the Listener port (Blank defaults to 5500) configuration parameter) traffic to the device at 192.168.1.250 (the private IP address of your computer).

Please note that to identify the public IP address of your Internet gateway you can simply start an Internet browser session and enter the following http://checkip.dyndns.org/ in the browser's address box.

If Scrip 236 finds that the UltraVNC, or RealVNC process (winvnc.exe for both) is running when it starts up, it will stop it.

After completing a remote control session, if you wish to leave the UltraVNC server process running on the target system, simply close the remote control window. Do not terminate the process using the taskbar icon.

Before using Scrip 236, you should also increase the frequency of execution of Scrip 177 (Scrip Configuration Update) to ensure that execution of Scrip 236 will occur quickly after you trigger its execution from the ASI server. We suggest that Scrip 177 be configured to run once every three minutes. While it's not necessary, you may want to revert the Scrip 177 schedule to the settings you normally use after the remote control sessions are over.

You can configure Scrip 236 in two ways:

  • Accessing its configuration page either on your ASI server, or directly at a site, or

  • Using the remote control configuration wizard on the ASI server (https://asiserverdomain/main/config/remote.php)

Please note that whenever you change one of the configuration parameters of Scrip 236, the Scrip will run updating, if necessary, the UltraVNC configuration on the system whose Scrip 236 configuration you just changed. For example, if you disable the Place remote support icon on desktop configuration parameter, Scrip 236 will run and remove the remote support icon from the desktop of the system where you disabled the configuration parameter.

Connection testing

When Scrip 236 runs on the target computer it first tries to get a response from the remote computer. The Scrip will open and close the port using the remote computer listed in the the Address of remote computer configuration parameter (see below). If this attempt to establish a connection succeeds, Scrip 236 will connect to that remote computer.

The Scrip will try to connect with the first host listed in the Address of remote computer configuration parameter. If it receives a response, it will use that address to attempt a connection. If it doesn't get the expected response, Scrip 236 will attempt to establish a connection with next remote computer listed in the Address of remote computer configuration parameter. If none of the remote computers listed in the Address of remote computer configuration parameter respond to Scrip 236's request for a connection, the Scrip will post an event log with that information on the ASI server, and won't try to establish a connection.

Please note that At this point in time there is no way for Scrip 236 to know which device is responding to its connection request. This means that it is susceptible to “false positive”. If another program responds to Scrip's connection request, a “false positive” will result in as much as the Scrip will try connect with the device that responded.

Disable tray icon

As the name of the Disable tray icon configuration parameter suggests, enabling it will remove the remote control utility icon from the target computer's system tray.

Run UltraVNC as a service

When the Run UltraVNC as a service configuration parameter is enabled (its default state), the UltraVNC server will run as a service. This results in a number of benefits:

  • When the UltraVNC server runs a service on a system that is logged off, UltraVNC will not generate the display of the UltraVNC server properties control panel.

  • The ASI client will not attempt to open a remote session connection before the UltraVNC server starts running which would result in a failed remote control session

Default password

If Scrip 236 finds that the UltraVNC server password is blank, it sets it to the value you entered in the Default password configuration parameter.

We strongly advise you to change the default password the first time you use the remote control utility.

Impersonation enabled

When the Impersonation enabled configuration parameter is enabled, using the impersonation user id and password (and domain if necessary) you entered in Impersonation Username, Impersonation Password and Confirmation, and Impersonation Domain configuration parameters in the Scrip 43 configuration page, Scrip 236 can retrieve The remote control software installation executable from a shared resources on other systems on the network, accessed either as mapped drives(e.g. x:\shared folder), or using UNC (Universal Naming Convention) syntax (e.g. \\server\\shared folder).

Place remote support icon on desktop

When the Place remote support icon on desktop configuration parameter is enabled, an icon with the label entered in the Remote support icon label configuration parameter is placed on the desktop of the system where the Place remote support icon on desktop configuration parameter is enabled.

The icon placed on the desktop is the ASI icon.

By default, the Place remote support icon on desktop configuration parameter is disabled.

Remote support icon label (Blank defaults to 'Start remote support')

In the Remote support icon label configuration parameter, you can enter any text to replace the default remote support icon label (Start remote support).

Address of remote computer

In the Address of remote computer configuration parameter you should enter the IP address(es) or name(s) of the remote computer(s), where remote computer is as defined in the Definitions and important notes section above.

The IP addresses or names of remote computers must be be listed one per line. For example:

  • 192.168.0.1

  • 245.45.6.100

  • nanoheal.org

  • hfn.tellink.net

If you choose to specify a port for a remote computer, enter a colon after the name or IP address of the remote computer, and then enter the port number, with no spaces in between. For example:

  • 192.168.0.1:5800

  • tellink.net:2345

A port number doesn't have to be attached to the IP address or name of a remote computer listed in the Address of remote computer configuration parameter. If you enter only ports and no remote computer IP addresses or name, Scrip 236 will fail to connect the target computer to any device.

Scrip 236 will identify the port to use as follows:

  • Scrip 236 will use the port attached to the IP address or name of a remote computer listed in the Address of remote computer configuration parameter, if there is one.

  • If no port is attached to the IP address or name of a remote computer listed in the Address of remote computer configuration parameter, Scrip 236 will use the port listed in the Listener port configuration parameter.

  • If there are no entries in the Listener port configuration parameter, the Scrip will use 5500, the default port.

If Scrip 236 detects that the port entries in the Address of remote computer or Listener port configuration parameters are invalid, it will use 5500, the default port.

Please note that if the remote computer is accessing a target computer at an external site, i.e. a site accessible via the Internet, and the remote computer does not have a public IP address, the entry for the Address of remote computer configuration parameter should be the public IP address or URL of the Internet gateway device used by the remote computer.

In addition, the firewall for the network where the remote computer is located should be configured to forward traffic incoming via port 5500 (the default port used by the remote control software) to the remote computer.

For example, suppose you want to configure Scrip 236 to initiate a remote control session from your computer (remote computer) on server X at customer site ABC. Let's also assume that your computer has a private IP address (e.g. 192.168.1.250), and that the Internet gateway at your site is a device with the service.provider.com URL.

The entry in the Address of remote computer configuration parameter would be service.provider.com (or its public IP address). In addition, the firewall on your network would be configured to forward incoming port 5500 (or whichever port you have specified in the Listener port (Blank defaults to 5500) configuration parameter) traffic to the device at 192.168.1.250 (the private IP address of your computer).

Please note that to identify the public IP address of your Internet gateway you can simply start an Internet browser session and enter the following http://checkip.dyndns.org/ in the browser's address box.

Listener port (Blank defaults to 5500)

The Listener port (Blank defaults to 5500) configuration parameter lets you identify an IP port different from the default (5500) for the remote control software on the target computer to use to communicate with the remote computer.

The value of the Listener port (Blank defaults to 5500) configuration parameter can be different for each remote computer. In this way, you can have multiple systems at your site conducting remote control sessions with different target computers at the same time.

As described in the Address of remote computer section of this configuration help file, please keep in mind that you will need to configure the firewall at your site to forward traffic incoming to each of the IP ports entered in the Listener port (Blank defaults to 5500) configuration parameter for each of the target systems to the corresponding remote computer.

Run now

Clicking on the Execute button in the Run now parameter will cause Scrip 236 to run the remote control software in server mode on the system whose Scrip 236 configuration page you are accessing (the target computer).

Shutdown now

Clicking on the Execute button in the Shutdown now parameter will cause Scrip 236 to terminate execution of the remote control software running in server mode on the system whose Scrip 236 configuration page you are accessing.

Uninstall on shutdown

If the Uninstall on shutdown configuration parameter is set to local, clicking on the Execute button will cause Scrip 236 to:

  • Terminate execution of the remote control software running in server mode

  • Remove the remote control software

on the system whose Scrip 236 configuration page you are accessing.

If the Uninstall on shutdown configuration parameter is set to global, the above actions will be performed on every system on the sub-net.

Remote control software installation executable download URL

The current version of Scrip 236 supports only the VNC family of remote control utilities. For your convenience, we have placed a copy of the UltraVNC installation executable in the publicly accessible download folder on www.handsfreenetworks.com. The Remote control software installation executable download URL for the UltraVNC installation executable at this location is

http://www.handsfreenetworks.com/download/UltraVNC-Setup.exe

If you leave the Remote control software installation executable download URL parameter blank, Scrip 236 will automatically download the installation executable for UltraVNC from the above location.

In the Remote control software installation executable download URL configuration parameter you specify the location where the installation executable for the remote control software is located and the file name.

The path leading to the remote control software installation executable must be visible on all the machines you want to run the executable on. This means that if the executable is on a networked drive then the drive must be accessible on all the machines through an appropriately configured drive letter mapping.

If the installation executable is located on a networked drive, you can enter either the UNC name of the location, e.g. \\[server name]\[shared resource name]\, or the mapped drive letter if one has been assigned to the networked drive.

The remote control software installation executable can also reside on FTP or Web servers (HTTP or HTTPS).

FTP servers can be accessed via a URL scheme that uses a common syntax for IP-based protocols. The syntax of a command line for having Scrip 196 retrieve a patch executable from a FTP server is:

ftp://user:password@host:port/url-path.

The user, password, or port can be omitted, and defaults will be applied. The default user is anonymous. The default password is a fictitious email address (most FTP servers that allow anonymous login either ignore the password or require an email address). The default port is 21, which is the standard FTP port.

Url-path can merely be the name of the remote file (current working directory for the FTP server assumed), or it can denote a particular subdirectory path to the file, e.g., dir/dir/dir/filename.

host is either the name or IP address of the FTP server.

Scrip 236 captures all FTP commands executed when running a patch located on a FTP server. They will be logged automatically onto the ASI server if the patch download process fails.

Web servers can be accessed using HTTP or secure HTTP (HTTPS) via a URL scheme that uses a common syntax for IP-based protocols. The syntax of a command line for having Scrip 196 retrieve a patch executable from a Web server is:

http://user:password@host/url-path, or https://user:password@host/url-path

For publicly accessible Web pages, the user and password are not necessary, and should be omitted.

Url-path can merely be the name of the remote file (current working directory for the Web server assumed), or it can denote a particular subdirectory path to the file, e.g., dir/dir/dir/filename.

host is either the name or IP address of the Web server.

Scrip 236 captures all HTTP commands executed when running a patch located on a Web server. They will be logged automatically onto the ASI server if the patch download process fails.

Please note that capitalization of the name of the patch/software update executable in the path parameter of a Scrip 196 command line needs to match capitalization of the patch/software update name on the FTP or HTTP/HTTPS server. Otherwise, Scrip 196 will report a file not found error. Generally speaking, to minimize the probability of an error, it is advisable to use all-lower case names.

Each ASI server includes a folder called patches located in the publicly accessible https://ASIserverURL/main/pub directory. You can use the patches folder as a convenient location from which the remote control software installation executable can be downloaded by Scrip 236 using HTTP/HTTPS, without requiring authentication. A sample Scrip 236 Remote control software installation executable download URL for remote control software located in the folder patches would look something like this:

https//ASIserverURL/main/pub/patches/<remote control software installation executable file name>

Please remember that on each ASI server we have installed an FTP server and created a a user id and password with upload access to the patches folder to upload patches and installation executables to this folder.

Note on uninstall and install

If Scrip 236 detects an existing installation of the remote control utility, it will not change any of the existing configuration changes.

In some cases, re-installation of the remote control utility might not work correctly unless the target computer is first restarted. This is due to the fact sometimes uninstall of the remote control utility does not succeed in removing certain dlls that might be loaded at the time of the uninstall even though there is no active remote session.

Use intermediate server

By default, the Use intermediate server configuration parameter is enabled. The system with the IP address or host name entered in the Address of intermediate server configuration parameter will be used as the UltraVNC intermediate server. This means, that both the system taking control (remote system), and the system to be controlled (target computer) will create an incoming connection through ports 5500 and 5900 on the selected intermediate server computer.

With the the Use intermediate server configuration parameter enabled, the only networking configuration you will need to make is to open ports 5500 and 5900 for incoming traffic forwarded to your ASI server.

Although typically this is the case, you should also make sure that port 5500 is open in the outbound direction at the site where the target computer is located.

You will be able to take remote control of systems at your customer sites from anywhere without making any additional configuration changes.

Maintain connection with intermediate server

When the Maintain connection with intermediate server configuration parameter is enabled, every time the ASI client starts-up Scrip 236 will run to ensure that the UltraVNC server is running with an open outbound connection to the intermediate server.

This means that whenever a system is re-started (and every other time the ASI client re-starts), Scrip 236 will ensure that the UltraVNC server is running with an open outbound connection with the intermediate server.

Whenever you want to take remote control of a system where the ASI client is installed and running, and the Maintain connection with intermediate server configuration parameter is enabled, the remote control session will be established instantaneously.

Please note, however, that this functionality comes at the cost of a lower level of security as on every system where the Maintain connection with intermediate server configuration parameter is enabled, port 5500 will always be open for outbound traffic.

After you enable the Maintain connection with intermediate server configuration parameter, if you want to ensure that the UltraVNC server is running on a system with an open outbound connection to the ASI server, all you need to do is run Scrip 18 (Client Shut-down / Re-start), or Scrip 73 (System Restart).

Address of intermediate server

In the Address of intermediate server configuration parameter, you enter the name of the one or more computers housing the intermediate server software, which we call intermediate servers. For each entry, you can enter the netBIOS name, domain name, or IP address. If you enter multiple names (or IP addresses), each should be on a new line. If for any reason, Scrip 236 fails to establish a connection with the first system whose name or IP address is listed in the Address of intermediate server configuration parameter, Scrip will attempt to establish a remote control session via the intermediate server next on the list, and move down the list until it finds an intermediate server which accepts a connection.

If the Address of intermediate server configuration parameter is left blank, and the Use intermediate server configuration parameter is enabled, Scrip 236 will extract the domain name of your ASI server (the server that is being logged to) from the HandsFree event management server URL list configuration parameter in Scrip 43, and use it as the UltraVNC intermediate server.

If Scrip 236 cannot reach an intermediate server, it will post an event log on your ASI server reporting the failure with the following message Connection failed: Unable to find listening intermediate server client.

Connection ID (0 or 200-1999999)

In order for you to be able to take remote control of a system using the intermediate server you will need a unique identifier for that remote control session. This is because in order to handle multiple remote control session simultaneously, the intermediate server needs a unique connection ID number for each remote control session. In this way, it can distinguish among the remote control sessions taking place at any one time. The connection ID number has to be the same one for the remote system (the one taking control), and the target system (the one being controlled). In this way, the intermediate server can match them, and establish the link between them necessary for the remote control session.

The Connection ID (0 or 200-1999999) configuration parameter is an integer with a value between 200 and 1999999. When the value of The Connection ID (0 or 200-1999999) configuration parameter is set to the default value of 0, Scrip 236 will generate a unique connection ID number based on the UUID (Unique Universal ID) of the system where Scrip 236 is running.

Please note that because of the way it is generated, the connection id number for a system will always be the same. This means, that after the first time Scrip 236 runs on a system with the Use intermediate server configuration parameter enabled, you will know that system's connection id, and it will never change, unless that system's UUID changes.

You can change the value of the Connection ID (0 or 200-1999999) configuration parameter as long you enter an integer between 200 and 1999999.

If you decide to assign a different connection id to each system at your sites, you need to make sure that:

  1. You assign a unique connection id to each system

  2. When you enter the unique connection id for a system in the Connection ID (0 or 200-1999999) configuration parameter in its Scrip 236 configuration page, the Connection ID (0 or 200-1999999) configuration parameter is set to local.

If the value of the Connection ID (0 or 200-1999999) configuration parameter is not within the allowed range, Scrip 236 will post an event log on your ASI server reporting the error with the following message Error: Invalid ID number. No connection can be made. A valid ID number is between 200 – 1999999.

Initiate Connection Automatically

The Initiate Connection Automatically configuration parameter is used by the ASI server software to remind users how a remote control session was initiated the last time Scrip 236 ran. It has no effect on how Scrip 236 runs.

Please note that disabling or enabling the Initiate Connection Automatically configuration parameter via the Scrip 236 configuration page has no effect whatsoever on the way Scrip 236 runs. The value of the Initiate Connection Automatically configuration parameter is controlled by the ASI server software, and changed by user actions performed when using the remote control wizard on the ASI server.

If a remote control session is initiated by a system's user clicking on the remote support icon placed by Scrip 236 on a system's desktop, the Initiate Connection Automatically configuration parameter is disabled.

If a remote control session takes place by execution of Scrip 236 initiated on the ASI server, the Initiate Connection Automatically configuration parameter is enabled.
