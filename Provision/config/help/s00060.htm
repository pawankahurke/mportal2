    • 1,1,*:\Documents and Settings\*\Local Settings\Temp\,default

For your convenience, we offer six Scrips that are exact copies of each other to perform folder and file clean-up operations. They are Scrips 60, 217, 218, 219, 220, and 221.

In this way, you can manage clean-up operations of different classes of folders and files in a more straightforward, and less error-prone fashion.

Impersonation Support

On NT based systems (MS Windows NT4, 2000, XP, and Server 2003) the ASI client is logged in as local system administrator. This means that it cannot access shared resources on the local network.

In order to enable ASI client's access to shared resources on the local network, we have implemented impersonation. Impersonation lets the ASI client log onto the system where the shared resource is located as a user known by that system enabling it to access that shared resource.

You should enter the known user id and password in Scrip 43, as described below. With this information, the ASI client will be able to access shared resources on the local network as follows:

Impersonation enabled

When the Impersonation enabled parameter is enabled, , using the impersonation user id and password (and domain if necessary) you entered in Impersonation Username, Impersonation Password and Confirmation, and Impersonation Domain parameters in the Scrip 43 configuration page, Scrip 60 (and 217-221) can clean up folders that reside on shared resources on other systems on the the network, accessed either as mapped drives(e.g. x:\shared folder), or using UNC (Universal Naming Convention) syntax (e.g. \\server\\shared folder).

Directories or files to scan

Directories or files to scan configuration parameter entries have the following format with each value in an entry separated by a comma, and no spaces before or after each comma:

  • Entry status with possible values equal to 0, 1, or 2, corresponding respectively, to disabled, enabled, or invalid. Scrip 60 (and 217-221) sets the status of any entry where it finds an error to invalid.

  • Sub-directory deletion flag. If it is set to 1, Scrip 60 (and 217-221) will delete all sub-directories older than the number of days set in the Minimum age of files to be deleted (days) parameter, which are empty after Scrip 60 has deleted all files that match the criteria defined in the File groups entry used by the Directories to clean entry, and the aging criteria set in the Minimum age of files to be deleted (days) parameter. When this flag is set to 0, Scrip 60 (and 217-221) will ignore any sub-directory contained in the directory being cleaned.

  • A directory specification that may have an environment variable in it. For example

    • d:\database

    • %temp%

    • %temp%\vpmectmp

  • A file group name that matches one of the file group names in the File groups parameter

When specifying a directory, you can use wild cards (see section on directory wild cards at the end of this document).

This means that in a Directories to clean entry you could specify the following path:

  • *\Documents and Settings\*\Local Settings\Temp

A Directories or files to scan entry with the above path specification would delete files matching the file group specified in the entry, in all user profiles' temp folders on all logical drives on a system.

Please note that if a Directories to clean entry contains the full path for a file, you do not need to have a file group parameter.

The presence of a file group parameter in a Directories or files to scan entry, implies that the third parameter is a directory. In that case Scrip 60 (and 217-221) checks to see if the third parameter exists on the local machine as a directory. If it does, It will process it. If it doesn't, the Scrip will check to see if the third parameter exists as a file. If it does exist as a file the entry will be flagged as invalid and log posted by the Scrip will include an explanation of the error it found. If the third parameter doesn't exist as a file, and Scrip 60 (and 217-221) cannot find the directory. it assumes that the directory currently does not exist on the local machine, and leaves the Directories or files to scan entry enabled.

You can have as many Directories or files to scan configuration parameter entries as you want, each entered on a new line.

File groups

File groups configuration parameter entries have the following format with each value in an entry separated by a comma:

  • File group name followed by a comma

  • file masks used by Scrip 60 (and 217-221) to find files you want the Scrip to delete, separated by commas

When specifying file masks, you can use the standard Microsoft MS-DOS wild card characters ? and * in both the file name and extension.

You can have as many File groups configuration parameter entries as you want with each file group entered on a new line.

Delete read-only files

When you enable the Delete read-only files configuration parameter, Scrip 60 (and 217-221) will delete all files and directories that match the criteria you set in the File groups and Minimum age of files to be deleted (days) configuration parameters

Minimum age of files to be deleted (days)

In the Minimum age of files to be deleted (days) configuration parameter you enter the minimum age of deleted files measured in days. It can be any integer number equal to, or greater than 0.

Run folder clean-up now

With the Run folder clean-up now Execute button, you can run Scrip 60 (and 217-221) on demand, even if it's not enabled. The Run folder clean-up now Execute button can be used to execute the Scrip on the local system, or on all supported systems.

A note on temporary files clean-up

You should note that on Microsoft Windows NT based systems (NT4, 2000, XP, and Server 2003), there are two locations where temporary files accumulate:

  • [root drive]:\winnt\temp

  • [root drive]:\Documents and Settings\[user login]\Local Settings\Temp

The majority of temporary files will be found in the second location listed above.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.

File Deletion

Scrip 60 (and 217-221) deletes files permanently. It does not send them to the recycle bin.

Directory path wild card support

Certain repetitive system maintenance and management tasks require access to sub-directories that have the same name but have different parent directories. For example, clean-up of temp directories on systems running the Microsoft Windows NT based operating systems (NT4, 2000, XP, and Server 2003), requires access to each user's temp directory located in [drive}:\Documents and Settings\<user name>\Local Settings\Temp.

In order to make it easier for you configure Scrips to perform such repetitive tasks, we have implemented wild card directory support, allowing wild cards anywhere in a directory path. This means that in the example above, specification of the path for the temp directory on all fixed and network drives would be reduced to a single line:

*:\Documents and Settings\*\Local Settings\Temp\

Wild card directory support applies to any Scrip where you may need to specify a directory path, including Scrips 60 (and 217-221), 61, 100, 196, 222, 225, 229, and 230. It also extends to devices, meaning that you can use the wild card character for devices including:

  • Fixed drives (like hard disk partitions)

  • Removable drives (like floppies)

  • Network drives

  • CD-ROMs

  • RAM disk drives

  • Unknown drive types

For example:

\\*\c:\Windows\**\temp

would scan the temp directory found zero or more levels below the Windows directory on the c: drive on all reachable systems on the local network.

The notation used for wild card directory support is slightly different from the MS-DOS wild card notation. It consists of either * or **:

  • * matches exactly one directory level.

  • ** matches zero or more directory levels.

For example:

\Documents and Settings\**\Local Settings\Temp\

would cover all directory levels where the end path consists of \Local Settings\Temp\ including:

\Documents and Settings\Local Settings\Temp\

and

\Documents and Settings\<user name>\Local Settings\Temp\

while

\Documents and Settings\*\Local Settings\Temp\

would cover only one directory level above \Local Settings\Temp\

You can use multiple instances of * and/or ** in directory path specifications. When the directory wild card support software finds instances of multiple wild cards, it performs left-to-right scan that looks through directories

recursively. Please note that his means that multiple ** or * in a directory path specification can cause a lot of multiple scans through the same directory structure, which will take a lot of time, but it will return the correct result.
