Scrip 190 Configuration

Configuration Parameters and default values

  • Run as service enabled

  • Enable system tray icon disabled

  • File or command for system tray double-click empty

  • Parameters for system tray double-click empty

Run as service

The Run as service configuration parameter controls whether the ASI client runs as a service or an application. By default this configuration parameter is enabled. If it is disabled, the ASI client will run only when a user is logged in, and not all ASI functions will be available.

Enable system tray icon

The Enable system tray icon configuration parameter controls the display of the ASI client system tray icon in a system's system tray. You can enable or disable the Enable system tray icon configuration parameter while the ASI client is running and the change will be processed in real-time.

File or command for system tray double-click

When a user double-clicks on the ASI client system tray icon, the ASI client will open the file or execute the command you enter in the File or command for system tray double-click configuration parameter. The ASI client looks for the file or executable in the client directory. If you leave the File or command for system tray double-click configuration parameter empty, then the ASI client will not take any action when a user double-clicks on the ASI client icon in the system tray.

Parameters for system tray double-click

The Parameters for system tray double-click configuration parameter contains any command line parameters that you want the executable entered in the File or command for system tray double-click configuration parameter to use. If you enter a file name in the File or command for system tray double-click configuration parameter, you should leave the Parameters for system tray double-click configuration parameter empty.
