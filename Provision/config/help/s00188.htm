Scrip 188 Configuration

Default Configuration Parameters

  • Scrip disabled
  • Attachments to block list empty

Attachments to block

On systems it is enabled on, Scrip 188 detects and removes specific attachments and types of attachments entered in the Attachments to block list from any e-mail message received.

Specific files are identified by their complete name including extension. When screening for file types, MS-DOS wild card characters (?, *) are used to define the files masks for the types of attachments you want to delete from incoming e-mail messages. For example, *.vbs, *.com, pref?????.*.

When you want the Scrip to detect and remove more than one type of attachment, or more than one specific attachment, each attachment type or specific attachment's name should be on a separate line in the Attachments to block list.

Attachments that are removed are replaced with a text file with the following content:

The file attached to this message that was originally named "[file name]" has been removed due to the filtering policy set up by your system administrator.
