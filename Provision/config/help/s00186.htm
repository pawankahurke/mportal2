Scrip 186 Configuration

Default Configuration Parameters

  • Maximum concurrent Scrips Set to 10

Maximum concurrent Scrips

Scrip 186 lets you control the number of Scrips that the Automated Support Infrastructure (ASI) client will run concurrently.

You should be careful about setting the value of this parameter too high because of the potential negative impact it can have on system performance. Typically, setting Maximum concurrent Scrips to a value between 10 and 20 should not have a noticeable impact on system performance.

It should also be noted that because Scrips perform system maintenance, and problem prevention and resolution activities, and problems are exceptional events, at any one time few Scrips will be running concurrently on any one system.
