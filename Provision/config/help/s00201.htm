Scrip 201 Configuration

Default Configuration Parameters

  • Network monitor status status of ASI network driver components on local systems

  • Install network driver

  • Uninstall network driver

  • Load dynamic driver

  • Unload dynamic driver

  • Timeout (sec) for local TCP connections by default set to 5 seconds

  • Timeout (sec) for remote TCP connections by default set to 20 seconds

  • Timeout (sec) for local UDP connections by default set to 180 seconds

  • Timeout (sec) for remote UDP connections by default set to 360 seconds

  • Size of domain cache by default set to 20

  • Number of unmatched sends indicating disconnect set to 5

  • Additional HTTP port (E.g. port used by proxy server) by default set to 8080

Network monitor status

The Network monitor status configuration parameter is display only. Whenever the state of the ASI network driver changes, e.g. you make a driver configuration change by clicking on any of the Execute buttons in the Scrip 201 configuration page, the content of the Network monitor status configuration parameter will be automatically updated.

Because all Scrip 201 install/uninstall, load/unload actions can be either local or global, before executing any of them it's very important that you make sure that they are set to global or local, depending on what you want to do.

Install network driver

Clicking on the Execute button to the right of the Install network driver label, will install the ASI network driver.

Uninstall network driver

Clicking on the Execute button to the right of the Uninstall network driver label, will remove the ASI network driver. However, all files necessary for the installation of the network driver will still be in the ASI client directory.

PLEASE NOTE THAT REMOVING THE ASI NETWORK DRIVER USING THE EXECUTE BUTTON TO THE RIGHT OF THE UNINSTALL NETWORK DRIVER LABEL, WILL CAUSE ALL NETWORK CONNECTIONS ON THE SYSTEMS WHERE THE ASI NETWORK DRIVER IS BEING REMOVED FROM TO BE RESET.

Load dynamic driver

Clicking on the Execute button to the right of the Load dynamic driver label, will load the dynamic component ASI network driver. This configuration parameter lets you control whether or not to activate component of the ASI network driver that makes it possible for all Scrips that rely on the ASI network driver to work. Without this component TCP/IP networking event detection would not be enabled, and none of the network related problem resolution can take place. E-mail attachment filtering, file download blocking, and port probe detection would also not work.

The dynamic component of the ASI network driver is loaded by default when the network driver is installed.

Unload dynamic driver

Clicking on the Execute button to the right of the Unload dynamic driver label, will unload the dynamic component ASI network driver. This action should be taken only when diagnosing networking related issues. When you unload the dynamic component of the ASI network driver, all of the ASI client network related functions will be disabled.

Timeout (sec) for local TCP connections

The Timeout (sec) for local TCP connections parameter lets you control the amount of time that scrip 201 will wait before deciding that a timeout occurred during an attempt to establish a local connection with a TCP/IP port/service.

Timeout (sec) for remote TCP connections

The Timeout (sec) for remote TCP connections parameter lets you control the amount of time that scrip 201 will wait before deciding that a timeout occurred during an attempt to establish a remote connection with a TCP/IP port/service.

Timeout (sec) for local UDP connections

The Timeout (sec) for local UDP connections parameter lets you control the amount of time that scrip 201 will wait before deciding that a timeout occurred during an attempt to establish a local connection with a UDP port/service.

Timeout (sec) for remote UDP connections

The Timeout (sec) for remote UDP connections parameter lets you control the amount of time that scrip 201 will wait before deciding that a timeout occurred during an attempt to establish a remote connection with a UDP port/service.

Size of domain cache

Size of domain cache is the maximum number of URLs detected by the Scrip during network event detection that it stores locally for use during problem diagnosis and communication with a system's user when a network related problem occurs.

Number of unmatched sends indicating disconnect

The Number of unmatched sends indicating disconnect parameter lets you set the threshold in number of networking send/receive failures that trigger scrip 201 problem diagnosis and resolution functions. When the Scrip detects a number of sends from a system without a matching receive that is equal to or exceeds the above number, it will log this as a local disconnect event and begin problem diagnosis and resolution.

Additional HTTP port (E.g. port used by proxy server)

In Additional HTTP port (E.g. port used by proxy server), you can enter a port, in addition to port 80, that will be monitored for HTTP transactions by Scrip 189.

For example, if you use a proxy server, you should set the value of the Additional HTTP port (E.g. port used by proxy server) parameter to match the HTTP port on the proxy server.
