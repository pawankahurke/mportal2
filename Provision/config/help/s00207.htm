Scrip 207 Configuration

Configuration parameters and their default values

  • Scrip enabled disabled

  • Content distribution actions empty

  • Latest dates on the sub-net empty

  • Maximum random delay before update (minutes) by default set to 5

  • Maximum retries for obtaining local updates by default set to 3

  • Local update retry delay (minutes) by default set to 60

  • When enabled, Scrip 207 runs at 5 AM on Tuesdays and Thursdays

  • Update files now

Content distribution actions

Content distribution actions configuration parameter entries have the following format with each value in an entry separated by a comma, and no spaces before or after each comma:

  • Method used to determine which of two files is “newer”. Currently, it must be set 1, meaning that the value of a file's last modified date/time is used to determine which of two files is “newer”.

  • Specification of the complete path and file being distributed that may have an environment variable in it. For example

    • d:\database\db.mdb

    • %temp%\def.dat

    • %temp%\vpmectmp\*.jpg

Please note that when specifying a file, you can use standard MS-DOS wild cards, * ?.

If the file being distributed does not exist on a system on the sub-net, when Scrip 207 runs, it will place a copy of the file with the latest "last modified" date on the sub-net, on that system, and log the absence of the file on that system in an event log.

You can have as many Content distribution actions configuration parameter entries as you want, each entered on a new line.

Latest dates on the sub-net

The Latest dates on the sub-net configuration parameter is displayed for information only. It should not be modified under any circumstances. If it were to be modified, Scrip 207 would not function properly.

The purpose of the Latest dates on the sub-net configuration parameter is to keep a running list of the latest dates (currently a files's last modified date) for all systems where a copy of the file being distributed is located. As a result, when a machine is searching for the file(s) listed in the Content distribution actions configuration parameter on other systems on the sub-net, it simply accesses this list. It does not have to contact every machine on the sub-net. This minimizes local network traffic and bandwidth usage. From the list's entries a system finds out the dates of virus definitions available on the sub-net, and which system(s) has/have them.

The format of the Latest dates on the sub-net configuration parameter entries is as follows:

  • Method used to determine the file's age. Currently, it must be set 1, meaning that the value of a file's last modified date/time is used to determine which of two files is “newer”.

  • Specification of the complete path for the file being distributed

  • File name

  • Date expressed in seconds since 1970 (currently, the file's last modified date/time)

  • Date = <Month day, year hh:mm:ss>, where currently Date is modified date

For example:

  • 1,%windir%\*.abc,%windir%\a.abc,1159472580,modified date = September 28, 2006 15:43:00

Maximum random delay before update (minutes)

The Maximum random delay before update (minutes) configuration parameter helps minimize the likelihood that if only one system has the copy of the file(s) being distributed with the latest date (last modified), this system not be flooded with requests for the copy of the file(s) from other systems. The Maximum random delay before update (minutes) configuration parameter ensures that requests for a newer copy of the file(s) being distributed from other systems do not occur simultaneously, maxmimizing the probability the the system be able to service at least one of the requests. Once that happens, the number of systems that can provide the latest (based on last modified date) copy of the file(s) being distributed is automatically doubled (at least), ensuring that all requests will be fulfilled.

Maximum retries for obtaining local updates

As its name suggests, the Maximum retries for obtaining local updates configuration parameter lets you set the number of times Scrip 207 will attempt to obtain the latest (based on last modified date) copy of the file being distributed available on the sub-net from other systems on the sub-net. By default, its value is set to 3.

Local update retry delay (minutes)

With the Local update retry delay (minutes) configuration parameter, you set the time interval between attempts Scrip 207 will make to obtain the latest (based on last modified date) copy of the file(s) being distributed available on the sub-net from other systems on the sub-net. By default, its value is set to 60 minutes.

Update files now

With the Execute button, you can run Scrip 207 on demand, even if it's not enabled. In Scrip 207, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
