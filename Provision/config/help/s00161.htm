Scrip 161 Configuration

Configuration parameters – default values

  • Scrip disabled

  • Continuously monitor for changes enabled

  • Registry keys to be monitored empty

  • When enabled, by default Scrip 161 runs daily at 2.40 AM, once per hour between 6.40 AM and 6.40 PM, and at 9.40 PM

  • Check monitored items now

Continuously monitor for changes

When the Continuously monitor for changes configuration parameter is enabled, Scrip 161 will detect changes to the registry keys listed in the Registry keys to be monitored configuration parameter as they happen, and will prevent changes to the keys being monitored, if so configured. When this configuration parameter is disabled, Scrip 161 will only check for changes when it is scheduled to run. Please note that enabling this configuration parameter can impact performance if the registry keys being monitored are many, and are very active.

Registry keys to be monitored

The Monitored items parameter contains the list of directories and files monitored by Scrip 161. Each item to be monitored is entered on a new line. The format of an item in the list is:

  • 0 (disable) or 1 (enable) - this makes it possible to temporarily stop monitoring selected registry keys without deleting the corresponding registry key to be monitored command line from the list.

  • Sub-key monitor flag - If it is set to 1, Scrip 161 will monitor all sub-keys. When this flag is set to 0, Scrip 161 will ignore any sub-keys contained in the registry key being monitored.

  • Operating system – the value of this parameter can be 9x, if the operating system running on the system is either MS Windows 95, 98, 98SE, or Me, or nt, if the operating system running on the system is either MS Windows NT4, 2000, XP, or Server 2003.

  • Protection - If this flag is set to 1 Scrip 161 will prevent any changes to be made to registry keys being monitored. When this flag is set to 0, Scrip 161 will only detect and report changes made to monitored registry keys.

  • Registry key path - The full path for a registry key you want Scrip 161 to monitor

An example of Registry keys to be monitored configuration parameter entry is:

  • 1,0,1,NT,HKEY_LOCAL_MACHINE\SOFTWARE\Nico Mak Computing\WinZip - This command line is enabled. It covers the WinZip registry key ignoring all sub-keys, and monitors and prevents any changes

Please note that when a command line in the Monitored items parameter is disabled, the stored information about the registry key and its sub-keys is removed. When a command line is enabled, the information about the registry key and its sub-keys is re-generated.

Note about Scrip 161 execution

Scrip 161 runs when one of the following events occurs:

  • When the ASI client starts up. When this happens it checks for changes

  • When you click on the execute button to the right of the Check monitored items now label. When this happens it checks for changes.

  • At its scheduled time. When this happens it checks for changes.

  • When monitored item(s) change. When this happens it checks for changes.

  • When one of its configuration parameter changes. When this happens it updates the configuration.

  • When it is enabled/disabled. When this happens it updates the configuration.

Checking for changes means:

  • Determining if any files or directories have been removed, added or altered, and

  • Updating the stored information about any item that has changed

Updating the configuration means updating the stored information about monitored item(s) to match their current state. If any existing stored information for monitored item(s) is still relevant, then it is re-used.

If you enabled protection for a command line in the Monitored items parameter(value set to 1), instead of just updating the stored information (checksum contained in the ASI client's persistent state repository) for the item(s) in that command line, when an item changes, Scrip 161 will restore the item to its pre-change state by overwriting the changed item with the copy that was stored in the hidden directory 161 contained in the ASI client directory when the item was first added to a (part of) Monitored items parameter command line.

Please note that Scrip 161 will restore any protected file or directory with the backup copy stored in the not only if an attempt is made to change them but also if they are deleted.

If you change the copies of the files and/or directories you have configured Scrip 161 to monitor (and protect) that are stored in directory 161, the Scrip will not update the changed item's (or items') checksum information until it is disabled and then re-enabled, or the ASI client is restarted.

Check monitored items now

With the Execute button, you can run Scrip 161 on demand, even if it's not enabled. In Scrip 161, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
