Run list parameters

Scrip 192 helps you control the programs end-users run on their systems. It can be a powerful ally in ensuring system stability and end-user productivity.

However, like any other powerful tool, it needs to be used with care.

Specifically, this means that when Scrip 192 is used with the Run list options enabled, you should ensure that all the programs necessary for end-users to do their job be included in the Allow these programs to run and in the Commonly running programs lists.

Please note that if you enable the Use Run List parameters to allow execution of selected programs option and leave the Run List - Allow these programs to run list EMPTY, the system won't boot when the ASI client is running in as a service (the default mode of operation).
On Microsoft Windows ME based systems, users won't even get a login prompt.
On Microsoft Windows 2000 based systems, users will be able to login but will ony see a blue background.

To minimize potential end-user problems, we suggest the following approach:

  • On one system where the Automated Support Infrastructure (ASI) client is running and on which Scrip 192 is enabled, start all the applications you want end-users to be able to run
  • Through Scrip 192's configuration screen, enable the Include common programs parameter, and then click on the Execute button next to the Update list of common programs parameter. This will populate the Commonly running programs list with the name of all the programs running at that time. It will also populate the Commonly running programs list on all systems on the local network with the same programs.
  • Subsequently, you can use the Update list of common programs parameter to update the list of programs you want to allow end users to run.
  • Use the Allow these programs to run parameter to customize the list of programs end-users are allowed to run on a system-by-sytem basis

You should note that unless included explicitly either in the Commonly running programs or Allow these programs to run list, end-users will not be able to run executables such as:

  • setup.exe or any other installation executable
  • taskmgr.exe
  • Applets in the MS Windows Control Panel
  • autorun.exe
  • winhelp.exe, the program necessary to access most applications' help files
  • Screen savers (programs with .scr extension)
  • rundll32.exe necessary, among other things, to change a system's date and time

Before running Scrip 196 (Software Patch Application), you should disable Scrip 192 on all the systems you want to apply a software patch to.

Allow these programs to run

Here, you should enter the full name of the program(s), including extension, whose execution you want to allow. Location path should not be entered.

When more than one program is entered, each name should be on a separate line.

Scrip 192 detects the execution of the program(s) entered in the Allow these programs to run list by matching these names with those of processes that are active on the system.

Version 1.0 of Scrip 192 detects all types of executables except for those with the .com and .bat extensions.

Include common programs

Enabling this option, allows users to run the programs included in the Commonly running programs list.

Commonly running programs

This list is populated automatically by Scrip 192. To do this, enable the Include common programs option, and then click on the Execute button next to the Update list of common programs parameter.

Very important note on program inclusion / exclusion

The ASI client has a built-in list of programs whose execution it allows by default. This list cannot be modified by a user.

This also means that including these programs in the Stop these programs from running list will have no effect.

At present this list consists of two programs:

  • iexplore.exe
  • netscape.exe

The Scrip configurator can be accessed only with a browser. Accidental exclusion of the two programs listed above from the Allow these programs to run list or inclusion in the Stop these programs from running list would make it very difficult to access the Scrip configurator and, in all likelihood, require direct intervention by HandsFree Networks.

Update list of common programs

When you click on the Execute button next to the Update list of common programs parameter, Scrip 192 will automatically retrieve the name of the programs running on the local system at that time. Scrip 192 will also populate the Commonly running programs list on all systems on the local network with the same programs.

Stop list parameters

Stop these programs from running

Here, you should enter the full name of the program(s), including extension, whose execution you want Scrip 192 to terminate. Location path should not be entered.

When more than one program is entered, each name should be on a separate line.

Scrip 192 detects the execution of the program(s) entered in the Allow these programs to run list by matching these names with those of processes that are active on the system.

Version 1.0 of Scrip 192 detects all types of executables except for those with the .com and .bat extensions.

Version 1.0 of Scrip 192 allows the termination of multiple programs entered in the Programs to prevent from running window.

You should enter the full name of the executable, including extension. Location path should not be entered. You can put arguments after an executable in the list and the Automated Support Infrastructure (ASI) client will only stop the executable if it is run with those arguments.

This is one way to keep a particular script from running. The argument characters in the list are compared against the incoming argument characters (using a simple case insensitive compare). If the compare succeeds for the length of the arguments in the list then a match occurs otherwise it fails.

For example:

All comparisons are case insensitive.

There is no wildcard matching in either the arguments or the executable name. Spaces before the arguments are ignored; all other spaces are significant.

You should also note that Scrip 192 doesn't filter what is entered in the Stop these programs from running list. However, it does filter out the path from incoming command lines. For example, if the entry in Stop these programs from running is wscript.exe C:\WINDOWS\SAMPLES\WSH\NETWORK.VBS, and the incoming command line is D:\WIN2000\System32\WScript.exe "C:\WINDOWS\SAMPLES\WSH\NETWORK.VBS", Scrip 192 will not stop this process from running.

This simply means that you should avoid entering the full path of both the executable and any command line arguments when making entries in the Stop these programs from running list.

Scrip 192 requires that any executable in the Stop these programs from running list should be entered with its extension. For example, you should enter wscrip.exe, not wscript. This also means that Scrip 192 will not terminate the execution of a command line where the executable is entered without its extent.

Parameters common to stop list and run list modes of operation

Display a message when a program is terminated

If you enable this option upon termination of a program you don't want end-users to run, or an attempt to run a program end-users are not allowed to run, Scrip 192 will display a dialog box titled Executable control with either a default message or a custom message (see below).

If no custom text is entered in the Custom message text box, Scrip 192 will display the following default message:

Execution of the following program(s) was terminated:

followed by the name(s) of the program(s) whose execution was terminated or prevented.

Custom message text

Here you enter the optional content of the message you want displayed in the Custom message text box.
