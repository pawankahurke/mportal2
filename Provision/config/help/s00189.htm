Scrip 189 Configuration

Default Configuration Parameters

  • Scrip disabled
  • File types to block and files to substitute list empty

File types to block and files to substitute

On systems it is enabled on, Scrip 189 detects and removes the specific files and types of files entered in the File types to block and files to substitute list when a user attempts to download them.

Entries in the File types to block and files to substitute list have the follwing format:

  • Name of file or file mask whose download you want Scrip 189 to block followed by a comma
  • Name of file you want Scrip 189 to substitute blocked file with

Specific files are identified by their complete name including extension. MS-DOS wild card characters (?, *) are used to define the files masks for the types of files whose download you want to block. For example, *.vbs, *.com, pref?????.*.

When you want the Scrip to detect and block the downloading of more than one type of file, or more than one specific file, each file type or specific file's name should be entered on a separate line in the File types to block and files to substitute list.

Files that are removed are replaced with the file whose name is entered (without path) on the same line as the name of the specific file or file type whose download is prevented. Scrip 189 looks for this file in the Automated Support Infrastructure (ASI) client directory (:\Program Files\HandsFree\Client).

If no such file name is entered, the file whose download was prevented is replaced with a file whose content is zeros.

It's important to note that Scrip 189 only replaces the content of the file whose download was blocked, not its name. Unless the end-user is informed on a timely basis that the file download filtering is enabled, this can lead to repeated attempts to download a file, and consequent frustration on the part of the end-user.

To minimize the potential for this to happen, we have included in the ASI client directory a small program called block.exe (also in zipped form, block.zip) that when executed informs the end user about the file download blocking policy in effect with the following statement:

"Sorry, per company policy download of this file type is not allowed. Please contact your system administrator with any questions. Thank you."

You can use block.exe, or block.zip as the files that replace blocked files when blocking files of types .exe, or .zip, respectively.

You should also be aware that depending on the browser used by the end-user to attempt to download a file, and the type of replacement file you selected (e.g. .gif), a blocked file download attempt may result in the display of the replacement file's content on the end-user system's screen.

Browser caching affects use of Scrip 189 in another way. If you disable the Scrip and a user attempts to download a file whose download had been previously blocked by Scrip 189, the download operation will fail again until the browser cache is cleared either explicitly by the user, or it's recycled.

This happens because Scrip 189 gives the file it uses to substitute a file being blocked the same name. During the first attempt to download the file, when Scrip 189 was enabled, the browser cached the file used by Scrip 189 to substitute the file whose download was blocked. Even if the Scrip is disabled, subsequent attempts will fail until the browser cache is cleared.
