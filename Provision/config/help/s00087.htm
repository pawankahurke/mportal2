Scrip 87 Configuration

Default Configuration Parameters

  • Scrip enabled
  • Only use log server to get time enabled
  • Minimum difference (minutes) to report Scrip 87 reports differences of at least 15 minutes between the system's clock and a certified time server
  • Time Servers
    • time.nist.gov
    • time-a.nist.gov
    • time-nw.nist.gov
    • ntp2.usno.navy.mil
  • Scrip runs at 9am on Tuesday, Thursday and Saturday

Only use log server to get time

When this option is enabled, Scrip 87 gets the time from the system where the ASI log server is located using HPC. The default time servers normally used by the Scrip are ignored.

Using this option produces two main benefits:

  • We no longer put such a high load on a central time server.
  • We no longer require firewall changes to the firewall at an end-user site to allow traffic through port 37 (used exclusively for time server communication).

Minimum difference (minutes) to report

If the time reported by a system's clock is slower than the atomic time retrieved by Scrip 87 by a number of minutes greater than or equal to the value of the Minimum difference (minutes) to report paramter, Scrip 87 will reset the system's clock to match the atomic time.

Time Servers

Scrip 87 can use port 37 to access the default certified time servers to adjust a system's clock.

It tries to reach them in the order in which they are entered in the Time Servers list. If none of the time servers is available, the synchronization operation will be attempted again the next time Scrip 87 is scheduled to run.

You can modify the list of default time servers used by Scrip 87, adding or deleting time servers from the list. All newly added time server entries should be on a separate line.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1Minutes

0-60

2Hours 1-24
3Day of the Month 1-31, depending on the month; a value of * means that the Scrip should be run every day.
4Month 1-12; a value of * means that the Scrip should be run every month.
5Day of the Week 0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.
6Minute Randomizer N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.
7Conditional Execution N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.
8Failed Run Limit N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.

The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.

Time Servers

Scrip 87 uses the following Internet time servers to compare a system's time to:

  • time.nist.gov
  • time-a.nist.gov
  • time-nw.nist.gov
  • nist1.sjc.certifiedtime.com
  • ntp2.usno.navy.mil

It tries to reach them in the order listed above. If none of the time servers is available, the time comparison operation will be attempted again the next time Scrip 87 is scheduled to run.
