Scrip 248 Configuration

Default Configuration Parameters

  • Registry driver status status of ASI registry driver components on local system

  • Enable driver to be loaded on startup

  • Prevent driver from being loaded on startup

  • Unload dynamic driver

Registry driver status

The Registry driver status configuration parameter is display only. Whenever the state of the ASI registry driver changes, e.g. you make a driver configuration change by clicking on any of the Execute buttons in the Scrip 248 configuration page, the content of the Registry driver status configuration parameter will be automatically updated.

Because all Scrip 248 load/unload actions can be either local or global, before executing any of them it's very important that you make sure that they are set to global or local, depending on what you want to do.

Enable driver to be loaded on startup

Clicking on the Execute button to the right of the Enable driver to be loaded on startup label, will install the ASI registry driver the next time a system is restarted.

Prevent driver from being loaded on startup

Clicking on the Execute button to the right of the Prevent driver from being loaded on startup label, will disable the ASI registry driver. However, the registry driver can be enabled at any time by clicking on the Enable driver to be loaded on startup Execute button.

Unload dynamic driver

Clicking on the Execute button to the right of the Unload dynamic driver label, will unload the dynamic component ASI registry driver. When you unload the dynamic component of the ASI registry driver, all of the ASI client functions that use the ASI registry driver will not function any longer. This includes the malware protection and management functions (Scrips 27, 232, and 233, 240).
