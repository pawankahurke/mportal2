Scrip 199 Configuration

Default Configuration Parameters

  • Scrip disabled
  • Registry keys to monitor list empty

Scrip 199 will check for changes in the registry keys listed in the Registry keys to monitor list when one of the following is true:

  • The client is starting up
  • The scrip has just been enabled
  • The scrip is already enabled and the list of registry keys has changed

Registry keys to monitor

This parameter is used to list all the registry keys you would like Scrip 199 to monitor. The full name of each key should be entered on a separate line.

To facilitate the process of entering the name of each registry key you want monitored, and minimize the probability of making a mistake, you can follow these steps:

  • run regedit
  • Locate the key you want to monitor by using regedit's find function
  • Once you have found the registry key you want to monitor, click on Edit on the regedit menu, the click on Copy Key Name
  • Click the mouse in the Registry keys to monitor list box first open line
  • Paste the registry key copied by pressing simultaneously the ctrl and v keys
  • Repeat the above steps until all the registry keys you want to monitor are listed in the Registry keys to monitor list.

Please note that Scrip 199 will detect and report changes in the registry keys entered in the Registry keys to monitor list, and all their sub-keys. This means that selecting the registry key for a frequently used application like Microsoft Internet Explorer, with many sub-keys, will generate a large number of event logs, most of them not very relevant from a system management and support standpoint.

Because of the above consideration, in general it is advisable to select registry keys that address specific, and significant, system, and application configuration parameters.
