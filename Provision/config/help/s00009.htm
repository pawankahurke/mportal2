Scrip 9 Configuration

Configuration Parameters

  • Scrip enabled

  • /n command line parameter - Doesn't pause for additional user information. Displays summary screens

  • Automatically fix errors enabled

  • Disks to scan Scrip 9 checks and repairs all local drives (/all command line parameter)

  • Scrip runs at 3am on Mondays

  • Run Scandisk now

Disks to scan

You can change the value of this parameter by simply deleting the default value and entering the drive letter of each drive you want scanned followed by a colon, e.g. c:. The entry for each drive you want scanned should be on a separate line.

Microsoft Scandiskw Command Line Parameters

In order to scan drives selectively, the drive selection parameter can be changed from /all to [drive]: where [drive] should be replaced with the drive letter of each logical drive you want to have scanned.

When more than one drive letter is used, each [drive]: should be entered on a separate line.

Version 1.0 of Scrip 9 does not allow the addition of any command line parameters to the execution of Scandisk except for drive selection.

The /n command line parameter cannot be deleted.

Run Scandisk now

With the Execute button, you can run Scrip 9 on demand, even if it's not enabled. In Scrip 9, the Execute button can be used to execute the Scrip on the local system or on all supported systems.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.

Microsoft Scandisk Command Line Parameters

  • Drive - The drive or drives to test.

  • /Allfixeddisks - Test all fixed disks.

  • /Noninteractive - Doesn't pause for additional user information. Displays summary screens.

  • /Preview - Test but doesn't fix any errors.

  • /Oldfs - Changes the File System to normal Dos. Removes all long file names.Destructive, can't be undone.Must be run from a Command Prompt.

  • /Silent - Exits without displaying summary screens.

  • /Sageset:x - Display the System Agent-Aware Setting dialog box and store the settings in the SETn registry key.

  • /Sagerun:x - Runs in unattended mode using the System Agent-aware settings in the SETn registry key.

Scandiskw Command Line Parameters

  • Switch: /all Function: Checks and repairs all local drives.

  • Switch: /autofix Function: Fixes damage without prompting.

  • Switch: /checkonly Function: Checks a drive, but does not repair any damage.

  • Switch: /custom Function: Configures and runs Scandisk according to the Scandisk.ini file settings.

  • Switch: /nosave Function: With /autofix, deletes lost clusters rather than saving as files.

  • Switch: /nosummary Function: When used with /checkonly, or /autofix, it prevents Scandisk from stopping at the summary screen.

  • Switch: /surface Function: Performs a surface scan after other checks.

  • Switch: /mono Function: Configures Scandisk for use with monochrome display.
