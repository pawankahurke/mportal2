Scrip 176 Configuration

Configuration parameters and their default values

  • Scrip disabled

  • Report (rather than restart) stopped service(s)

  • Services to be monitored list empty

  • When enabled, by default Scrip 176 runs on the hour, every hour, seven days a week

  • Restart services now

Report (rather than restart) stopped service(s)

When the Report (rather than restart) stopped service(s) configuration parameter is enabled Scrip 176 will monitor the services listed in the Services to be monitored configuration parameter, and post a log onto the ASI server if any of them stop. It will not try to restart them.

Services to be monitored

Scrip 176 will re-start the services you enter in the Services list configuration parameter at the time indicated in the Schedule.

A service has two names, an internal name and a display name. Controlling a service programatically requires the internal name, but only the display name (user-friendly name) appears in the Service Control Manager UI.

It seems that in many cases the internal name and display name are the same, but they don't have to be. Therefore, Scrip 176 is designed to work with either. The entry made in "Service" list is initially assumed by Scrip 176 to be the internal name. If a service with that internal name is not found to be installed, Scrip 176 looks in the registry to see if the entry exists as a display name, and extracts the corresponding internal name and uses that.

Report services not found

When you click on the Report services not found Execute button, Scrip 176, even if disabled, will post a log onto the ASI server listing services included in the Services to be monitored configuration parameter that are not found on the system(s) where you run this command, depending on whether the Report services not found Execute button is set to be local or global.

Re-start services now

When you click on the Re-start services now Execute button, Scrip 176 will run even if not enabled, and will attempt to restart all the services you entered in the Services to be monitored configuration parameter. This command can be run on the local system, or on all systems at a site.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
