Currently enabled startup items on this machine

To facilitate and speed up the review of a system's start-up environment, start-up items currently enabled on a system are divided into two groups:

  • Currently enabled startup items on this machine - for review

  • Currently enabled startup items on this machine – known

The Currently enabled startup items on this machine - for review configuration parameter contains start-up items that do not match any of the entries in the Known items configuration parameter. The Currently enabled startup items on this machine – known configuration parameter contains start-up items that match entries in the Known items configuration parameter.

When reviewing a system's start-up environment, you can focus your review on the contents of the Currently enabled startup items on this machine - for review configuration parameter, ignoring the contents of the Currently enabled startup items on this machine – known configuration parameter. However, we suggest that periodically you review the content of the Currently enabled startup items on this machine – known configuration parameter to ensure that inadvertently one or more malware items have not been included in the Currently enabled startup items on this machine – known configuration parameter.

Currently enabled startup items on this machine configuration parameters entries can have one of the following formats:

  • RegKey, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • Key, followed by a comma with no spaces

  • Key variable name, followed by a comma with no spaces

  • Key variable value

or,

  • StartFolder, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • Start-up folder item path, followed by a comma with no spaces

  • Start-up folder item

or,

  • IniFile, followed by a comma with no spaces

  • User name, followed by a comma with no spaces

  • .ini file (either win.ini or system.ini) path, followed by a comma with no spaces

  • .ini file section, followed by a comma with no spaces

  • .ini file variable name, followed by a comma with no spaces

  • .ini file variable value

The Currently enabled startup items on this machine configuration parameter entries (for review and known) are display-only. They report the content of the disabled items in the start-up environment on the system whose Scrip 233 configuration page your are viewing. Editing the Currently enabled startup items on this machine configuration parameters (for review and known) has no effect, as its content will be regenerated when Scrip 233 runs.

Note on adding item entries and wild card support

Please note that identical items for different users show up as different entries in the Currently enabled items on this machine and Currently disabled items on this machine configuration parameters, and are handled as separate start-up items in the Items to enable and Items to disable configuration parameters. This allows you to enable or disable a given system configuration item only for a certain user or group of users.

To facilitate the addition of entries to the Items to enable or Items to disable configuration parameters, Scrip 233 supports the use of the * wild card character. For example, if you want to disabled occurrences of 

RegKey,Administrator,HKEY_USERS\S-1-5-21-1855000883-1368534861-315576832-10961\Software\
Microsoft\Windows\CurrentVersion\Run,TV Media,C:\Program Files\TV Media\Tvm.exe

for all users on one system, or all systems at a site, in the Items to disable configuration parameter you should enter:

RegKey,*,HKEY_USERS\S-1-5-21-1855000883-1368534861-315576832-10961\Software\
Microsoft\Windows\CurrentVersion\Run,TV Media,C:\Program Files\TV Media\Tvm.exe

Please note that the wild card character can be used only as a value in the User Name field.

Known items

The Known items configuration parameter contains all start-up environment items which you know to be safe. This does not mean that you may not want to disable items that are in the Known items configuration parameter. All it means is that you do not have to be concerned about any of them being malware.

You can add entries to the Known items configuration parameter manually making sure that they have the format described below. However, we advise against doing this because of the chance of inadvertent error.

The easiest and most reliable way to add entries to the Known items configuration parameter, is to copy safe entries in the Currently enabled startup items on this machine - for review configuration parameter, and paste them on a new line in the Known items configuration parameter.

After you add entries to the Known items configuration parameter, changes you have made (i.e. additions to the Currently enabled startup items on this machine – known configuration parameter, and deletions from the Currently enabled startup items on this machine – for review configuration parameter will take effect after you click on the the Update local variables only Execute button, or when you click on the Update startup items now Execute button.

Please note that since the Known items configuration parameter can be global, you can have one known items list for all your sites. If you leave the Known items configuration parameter set to global, any additions made to the parameter will immediately propagate to all systems at a site. In addition, you can use the export function to propagate the contents of the Known items configuration parameter from one system to all sites.

We suggest that you maintain a master known items list in an ASCII text file and update it as you find start-up items that are known to be safe. Periodically, you can copy the master list, paste it into the Known items configuration parameter for a system at one of your sites (at that point it will propagate automatically to all other systems at the site), and then export that system's Scrip 233 configuration to all other sites.

Update local variables only

Clicking on the Update local variables only Execute button, will cause Scrip 233 to update the content of the Currently enabled startup items on this machine configuration parameters (for review and known), and post an event log on the ASI server with the the up-to-date content of a system's start-up environment. This function can be useful for verifying that Scrip 27 (System Start-up Environment control) is protecting a system's start-up environment without having Scrip 233 clean-up the start-up environment, which would prevent you from verifying that Scrip 27 is working.

Update startup items now

Clicking on the Update startup items now Execute button, triggers Scrip 233 immediate execution, even if it's not enabled. The Update startup items now Execute button can be used to execute the Scrip both on the local system, or on all systems at a site. When you click on the Update startup items now Execute button, Scrip 233 runs immediately, regenerating the content of the Currently enabled startup items on this machine (for review and known), and Currently disabled startup items on this machine configuration parameters. It also enables/disables the start-up items identified in the Startup items to enable and Startup items to disable configuration parameters, respectively.

Note about Scrip 233 execution

Scrip 233 runs when one of the following events occurs:

  • When the ASI client starts-up

  • When you click on the Update startup items now Execute button, or on the Update local variables only Execute button

  • When a user logs on. Having Scrip 233 run when a user logs on has the side effect of having it run on ASI client start-up. This is because the user that is currently active is seen as a new user when the ASI client starts up.

  • When it is enabled. When this happens it updates the configuration.

  • Whenever the schedule changes

  • At its scheduled time

When Scrip 233 runs the Startup items to enable and Startup items to disable configuration parameters are processed first. After the appropriate start-up items are enabled or disabled as per the configuration, then the content of the Currently enabled startup items on this machine and Currently disabled startup items on this machine configuration parameters is updated.

Note on enabling and disabling start-up items

The easiest way to disable a start-up item is to copy it from the Currently enabled startup items configuration parameter, and paste it into the Startup items to disable configuration parameter. To re-enable an item you can move it (cut and paste) from the Startup items to disable configuration parameter to the Startup items to enable configuration parameter. Depending on your needs, in order to disable a start-up item on all systems at a site you only need to add an entry in the Startup items to disable configuration parameter on one system.

While this method of enabling and disabling start-up items is certainly not the most elegant, it is straightforward and efficient. For example, in order to disable a system configuration item on all systems at a site you only need to add an entry to the Start-up items to disable configuration parameter on one system.

When performing the above task, please pay special attention to the local/global settings of each variable whose value you are changing. Otherwise, you run the risk of making changes to all systems at a site, when you don't want to, or making changes on only one system when, in fact, the changes should apply to all systems at a site. You should use the same care when executing Scrip 233 on-demand using the Execute button.

Schedule

The Schedule parameter has a command line with eight fields, each entered on a new line.

The table below describes each field in detail.

Field Number

Field Name

Field Values and Description

1

Minutes

0-59; a value of * means that the Scrip should run every minute

2

Hours

0-23; a value of * means that the Scrip should run every hour

3

Day of the Month

1-31, depending on the month; a value of * means that the Scrip should be run every day.

4

Month

1-12; a value of * means that the Scrip should be run every month.

5

Day of the Week

0, Sunday - 6, Saturday; a value of * means that the Scrip should be run every day of the week.

6

Minute Randomizer

N; the size of the time interval in minutes over which the Scrip should be randomly executed. We multiply N*60, then calculate a random number X between 1 and N*60, and run the Scrip X seconds after its scheduled time.

7

Conditional Execution

N; N=1 means that if the Scrip is past due, it will be run immediately, N=2 means that If the Scrip is past due, it will be run at the next scheduled execution time. If the value is set to 2, the actual Scrip execution time will be within 3 minutes (either way) of the scheduled time. This is necessary to prevent the case where a process will not be executed if it is a little slow to start, or if the scheduler starts it a few seconds too early.

8

Failed Run Limit

N, where N is the number of times that Scrip execution is allowed to fail before notification is sent to the support provider; N=0, the Scrip will NEVER report a failure, N=1, the Scrip will report EVERY failure, N=2, the Scrip will report if it failed twice in a row, and so on.



The Minutes, Hours, Day of the Month, Month, and Day of the Week fields can have multiple values and ranges of values separating them with commas.

For example, a 1,3,5 entry for the Minutes parameter would mean that a Scrip would be executed at minutes 1, 3, and 5.

A 10-14 entry for the Hours parameter would mean that a Scrip would be executed starting at 10 AM and ending at 2 PM.

Multiple value and range entries can be used together separated by commas.
