Scrip 178 Configuration

Default Configuration Parameters

  • Scrip enabled
  • Automatically allow access disabled

Automatically allow access

When the Automatically allow access option is enabled, as a failure to access a shared resource on an NT based (Microsoft Windows NT4, 2000, XP, or Server 2003) happens, the server will create an account using the name of the account that was logged in at the time the failure occurred. Scrip 178 will create the account without a password, and with normal (anonymous) access rights on the server. This means that the the first time a user tries to access a shared resource on a server it will fail, but from then on, it will work.
