General notes

When they are configured to be sent by e-mail, event notifications, event reports and asset reports by default will be sent to the recipients designated as the default recipients for the user account where the event notifications, event or asset reports are being produced. Default recipients for a user account are configured using the Admin module reachable by clicking on the admin link in the tools navigation bar on any page on the ASI server. You can also configure event notifications, event or asset reports to be sent to recipients other than the default recipients. These are called custom recipients. In all item pages, parameters related to default e-mail recipients are labeled Default Recipients. Parameters related to custom recipients are labeled either E-mail Recipients, or Recipients.



Edit option parameters

State átop

Asset reports can be in one of three states, enabled, disabled, or invalid. Invalid state denotes an asset report where an execution error occurred. The State edit parameter lets you change the state of a group of asset reports.

Please note that if you set the state of a group of asset reports to Invalid,

The State edit parameter can have the values listed below.

No Change Disabled Enabled Invalid

Output átop

Asset reports can be sent to recipients via e-mail (default option), or posted on the information portal. The Output edit parameter lets you change the value of the Output configuration parameter for a group of asset reports. The Output edit parameter can have the values listed below.

No Change Email Information Portal

Default Recipients átop

You can configure asset reports to be sent to the default e-mail address(es) for a user login. This is the default option. The Default Recipients edit parameter lets you change the value of the Default Recipients configuration parameter for a group of asset reports. This edit parameter can have the values listed below.

No Change No Yes

Links átop

When defining an asset report, you can choose to configure the names of the systems events are reported from as links to those systems' asset detail information. The Links edit parameter lets you enable or disable this configuration option for a group of asset reports. This edit parameter can have the values listed below.

No Change No Yes

Details átop

When defining an asset report, you can choose to have the report include the complete results of the asset query used by the asset report (default option). With the Details edit parameter you can include or exclude asset query results for a group of asset reports. This edit parameter can have the values listed below.

No Change No Yes

Format átop

The Format edit parameter lets you change the format of a group of asset reports selecting a new format from one of the values listed below.

No Change HTML w/o Charts HTML w/ Pie Charts MHTML w/ Pie Charts HTML w/ Bar Charts MHTML w/ Bar Charts HTML w/ Column Charts MHTML w/ Column Charts

Skip Owner átop

When you configure a global asset report, you can choose to NOT have the report sent to the owner of the report (the user logged onto the ASI server when the notification is created). You do this by enabling or disabling the Do not generate for owner asset report configuration parameter.

The Skip Owner edit parameter lets you change the value of the Do not generate for owner configuration parameter for a group of asset reports. This edit parameter can have the values listed below.

No Change No Yes

Tabular átop

When defining an asset report, if you select the Details in by-system tables configuration option, detail asset information for each system included in that asset report will be presented in a separate table, improving report readability. The Tabular edit parameter lets you enable this configuration option for the asset reports you are editing. This edit parameter can have the values listed below.

No Change No Yes

Include User átop

When you configure an asset report you can choose to include the user login under which the report is run in the subject line of the e-mail message used to distribute the report. If you use this option in a global asset report configuration, when the report is run for each user on the ASI server (as global reports always are), the e-mail message for each user-specific report would include the user login in the subject line.

The Include User edit parameter lets you change the Include User option for a group of asset reports. Its possible values are listed below.

No Change No Yes

Include Text átop

When you configure an asset report you can choose to include custom text in the subject line of the e-mail message used to distribute the report. The Include Text edit parameter lets you change the Include Text option for a group of asset reports. Its possible values are listed below.

No Change No Yes

Subject Text (update) átop

If the Include Text configuration parameter is enabled, you can choose to change the custom text to be included in the subject line of the e-mail message used to distribute the report for a group of asset reports by enabling the Subject Text (update) edit parameter. This edit parameter can have the values listed below.

No Change Update

Subject Text (value) átop

If the Subject Text (update) edit parameter is enabled, in the Subject Text (value) edit parameter you enter the new custom text to be included in the subject line of the e-mail message used to distribute the report for a group of asset reports. This edit parameter can have the values listed below.

E-mail Recipients (update) átop

You can choose to change the recipients for a group of asset reports by enabling the Recipients (update) edit parameter. This edit parameter can have the values listed below.

No Change Update

E-mail Recipients (value) átop

If the Recipients (update) edit parameter is enabled, in the Update Recipients (value), you enter the updated comma delimited (no spaces) list of the e-mail addresses of the recipients for a group of asset reports.
