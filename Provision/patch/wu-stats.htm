Search and Display Options Panel Parameters

Search Options

Any field used to filter software update status entries will be displayed in the software update table. In order to have a field displayed in the table even if you do not use it as a search parameter, select the (all) option from its pull down list in the Search Options panel. If you do not want a field displayed in the table, simply select (not displayed) from its pull down list in the Search Options panel.

Dates can only be selected in daily increments up to two weeks from the current date, weekly increments from two weeks up to one month, monthly increments from the second month up to six months, and up to one year beyond six months. You cannot select an end date different from the current date.

Name Contains

In the Name Contains search parameter, you can enter a single string of alphanumeric characters. The search performed will be a non-case sensitive sub-string match. Name is the field that contains the name of a software update.

Site

The Site search parameter lets you select software updates applicable to individual sites.

Machine

The Machine search parameter lets you select software updates applicable to one system.

In order to select software update status for a single system you need to first select the site it belongs to from the Site pull down list in the Search Options panel, click on the Submit Search button, then select the system whose update status you are interested in by selecting it from the Machine pull down list in the Search Options panel.

Update

Update, e.g. Security Update for Windows 2000 (KB841533) (x86 en 5.0.2195 SP4.0), is a name composed of:

  • The Title of the update (the value of the title field in the software update detail information page accessed when you click on an Update column entry in the Microsoft Update – Status page.)

  • Item string that identifies the software update uniquely. The Item string has the following format:

  • processor locale OS Build ServicePack level

Initially, newer Microsoft XP software updates will not have Item string information. Their Item string will consist of: en 0.0.0 SP0.0. For each software update, Scrip 237 constructs the Update field composed (Title plus Item string) and sends it to the ASI server. if the software update title already exists but the Item string is different (i.e. if the versions of an update are different) then the ASI server adds a new software update entry with the same Title and a different Item string.

Status

The Status search parameter lets you select updates by the value of the status filed as reported by Scrip 237. At present, values for the Status field include:

  • automatic

    Currently, this state is not used by Scrip 237.

  • pending install

  • pending uninstall

  • scheduled install

  • scheduled uninistall

  • pending download

All of the above states indicate the next action Scrip 237 will take for that software update.

A pending state is when Scrip 237 is just about to perform the operation, be it downloading, installing, or uninstalling.

A scheduled state is when Scrip 237 needs to wait until a specific point in time before performing a task. These states are not used presently since Scrip 237 reports this information in even greater detail as part of its logs. They're in place in case we need to make them active at some future date.

  • Disabled

A software update has disabled status when it is declined via the Decline updates wizard on the ASI server.

  • error

    When a software update installation fails, its status will change to error, last error date will be set to the time Scrip 237 discovered the failure, and last error text will Update failed to install. In future, we will add other error states. For example, providing error codes and the corresponding error text.

Installation errors are the only kind of error that changes a software update status to error. This is true for both Microsoft AU clients. The old Microsoft AU client (iuengine.dll version equal to, or lower than 5.4.3790.21) does not report the error code explicitly although there may be an error code in the Windows Update client log output.

The new Microsoft AU client (iuengine.dll version higher than 5.4.3790.21) tries to report the error code explicitly as an hresult. Scrip 237 reports these error codes in its event logs. You could lookup the meaning of error codes reported by the Microsoft AU client in a number of sources including this one: http://inetexplorer.mvps.org/windows_update_codes.htm

  • installed

    Scrip 237 reports the software update as successfully installed.

  • uninstalled

    Scrip 237 does not currently use this field.

  • downloaded

    Scrip 237 reports the software update as successfully downloaded.

  • detected

    Scrip 237 reports the software update as detected.

  • needs reboot

    When a software update is listed as being in this state, the system must be restarted before Scrip 237 can take any further actions.

  • potential installation failure

    When a software update install operation partially succeeds, Scrip 237 will report the software update as having a potential installation failure status. Ultimately, this state will be followed by either in installed or error state.

  • superseded

  • A software update in superseded state has been superseded by another software update, and should not be installed. For instance, suppose you decline the installation of Microsoft Windows XP SP2, then software updates that were superseded by Microsoft XP SP2 are detected by Scrip 237. If Microsoft XP SP2 were then approved, those software updates would be given superseded status.

If a software update simply disappears for any reason, Scrip 237 will assign it a superseded status. A software update would disappear if at one time Scrip 237 detected it and reported it to the ASI server, and then during some future detection cycle the update is not detected again.

The above date related search parameter fields let you select software updates by the date of the last action performed, or state detected, by Scrip 237.

Date of last change

The Date of last change search parameter facilitates the search for software update configuration changes that occurred in the recent past The Date of last change represents the last time a software update configuration change for a software update occurred on the ASI server. For example, when the schedule and notification configuration for a software update was changed, or which systems it was approved for.

Date of next action

Currently, this field is not used by Scrip 237.

Error

The Error search parameter lets you select software updates that have a non-empty error status. Currently, Scrip 237 gives a software update error status when its installation fails. Scrip 237 will try to install a software update which has error status again if it actually exists and it is configured to do so.

Download Source

The download source search parameter can be either the Microsoft server or the WUS server (SUS). Usually, it is the full URL of the server.

Update Size

The Update Size search parameter lets you filter updates by size in the following ranges:

  • 0 - 100 KB

  • 100 KB - 500 KB

  • 500 KB - 1 MB

  • 1 MB - 10MB

  • 10 MB - 50 MB

  • over 50 MB

Please note that you cannot select updates by size across the above ranges. For example, you cannot search for updates with size ranging between 100 Kbytes and 1 Mbyte.

Machine Group

You can select either one of the built-in machine groups, a machine group defined by another user, or a machine group that you defined when using the software update wizards. Note that if you select a machine group defined by another user, you cannot modify it. The ASI built-in machine groups are:

  • All This category should contain only one group, All. The group should contain every machine logging to the server.

  • Sites This category contains one machine group for every site currently logging to the server. The site groups have the same name as the site, and each site group should contain one member for each machine at the associated site.

  • Machines This category should contain one group for each machine currently logging to the server. Each of these groups should contain a single machine. The name of the group is "$SITE:$HOST" where $SITE represents the site name and $HOST represents the machine name.

Note that the All machine group is the default selection for the Microsoft Update – Status page. You can select individual members of the Sites group (i.e. individual sites) using the Sites search parameter, and individual members of the Machines group (i.e. individual systems) using the Machines search parameter.

Update Group

The Search Options panel includes the Update Group selection parameter. Currently, none of the software update wizards give you the option to define software update groups. The wizards define software update groups automatically in order to perform software update tasks. The names of these software update groups, e.g. Wiz_CRIT_PG $MGRP , are not meaningful to you. The membership of these software update groups changes every time the software update task they were created for is executed again. ASI features a number of built-in software groups:

  • All This patch category should contain only one group, All. The group should contain every patch known to the server.

  • Updates This category contains one patch group for every patch known to the server. Each group will have the same name as its associated patch, and should contain just that one patch.

Type

The software update type built-in group includes all Microsoft software update types. The group has the following six members:

  • Update

  • Service Pack

  • Rollup

  • Security

  • Critical

  • Mandatory

Please note that since the type software update group is built-in you cannot change it in any way.

MS Name

MS Name, e.g. 841533_W2K_SP5_WinSE_125872, is one of the names Microsoft gives an update. It's just an entry in the XML file on the old Microsoft AU client (iuengine.dll version equal to, or lower than 5.4.3790.21). On the new Microsoft AU client (iuengine.dll version higher than 5.4.3790.21), this field does not exist. This means, for example, that new XP patches will not have an MS Name.

Display Options

If you want to sort software update table entries by more than one parameter (e.g. site/machine/update), you should use the Sort-by options in the Display Options panel. To sort table entries by a single parameter, simply click on the column heading corresponding to that parameter (e.g. Status). Clicking once sorts entries in descending order, twice sorts them in ascending order.

You can control the number of entries displayed in an item page with the Page Size parameter. Its possible values are listed below.

5 10 20 25 50 75 100 150 200 250 500 750 1000
