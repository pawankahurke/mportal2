Search and Display Options Panel Parameters

Any field used to filter software update entries will be displayed in the software update table. In order to have a field displayed in the table even if you do not use it as a search parameter, select the (all) option from its pull down list in the Search and Display Options panel. If you do not want a field displayed in the table, simply select (not displayed) from its pull down list in the Search and Display Options panel.

Dates can only be selected in daily increments up to two weeks from the current date, weekly increments from two weeks up to one month, monthly increments from the second month up to six months, and up to one year beyond six months. You cannot select an end date different from the current date.

If you want to sort software update table entries by more than one parameter (e.g. site/machine/update), you should use the Sort-by options in the Search and Display Options panel. To sort table entries by a single parameter, simply click on the column heading corresponding to that parameter (e.g. Status). Clicking once sorts entries in descending order, twice sorts them in ascending order.

Name Contains

In the Name Contains search parameter, you can enter a single string of alphanumeric characters. The search performed will be a non-case sensitive sub-string match.

Update Group

The Search and Display Options panel includes the Update Group field. Currently, none of the software update wizards give you the option to define software update groups. The wizards define software update groups automatically in order to perform software update tasks. The names of these software update groups, e.g. Wiz_CRIT_PG $MGRP , are not meaningful to you. The membership of these software update groups changes every time the software update task they were created for is executed again. ASI also features a number of built-in software groups:

  • All This patch category should contain only one group, All. The group should contain every patch known to the server.

  • Updates This category contains one patch group for every patch known to the server. Each group will have the same name as its associated patch, and should contain just that one patch.

Detected

Filtering software updates using the Detected search parameter, lets you quickly identify the software updates that have been detected as applicable to the systems you selected for the update wizard action, but have not yet been downloaded or installed.

Installed

Filtering software updates using the Installed search parameter, lets you quickly identify the software updates that have been installed on the systems you selected for the update wizard action, without having to access the MUM status page (https://asiservername/main/patch/wu-stats.php).

Type

The software update type built-in group includes all Microsoft software update types. The group has the following six members:

  • Update

  • Service Pack

  • Rollup

  • Security

  • Critical

  • Mandatory

Please note that since the type software update group is built-in you cannot change it in any way.

Internal Priority

Scrip 237 assigns internal priorities to Microsoft software updates. Internal priority values tell you which updates Scrip 237 treats as more important than others. The higher the value of the internal priority, the more important the update is. More important updates must be installed or declined before less important updates can be installed or declined.

For example, suppose that Microsoft software update A has an internal priority of 4. At one time on one machine, Scrip 237 detected update A. It was displayed on the MUM status page as detected. Then, sometime later, before update A was installed, Microsoft released update B which has an internal priority of 5. Once Scrip 237 detects update B, it gives waiting status to update A, and update B becomes detected instead. Update A will not be installed (transition to detected state) until update B is installed.

Below, is the list of internal priority values used by Scrip 237 (Microsoft Update Management):

  • 0 - Drivers

  • 1 - Updates

  • 3 - Service Packs

  • 4 - Update Rollups

  • 4 - Security Updates

  • 4 – Critical Updates

  • 5 – Mandatory Updates

Mandatory and critical software updates are always detected, downloaded, and installed before any other software updates are even detected. Service packs will not be detected, downloaded, or installed until both of the following conditions are met:

  1. All mandatory updates are installed.

  2. All critical, security, and update rollups are either declined or installed. If an update is declined, Scrip 237 will ignore the update and proceed down to one with a lower internal priority, if there are none with equal internal priority.

Please note that since drivers have the precedence value of 0, they will never be detected, downloaded, or installed.

Update Size

The Update Size search parameter lets you filter updates by size in the following ranges:

  • 0 - 100 KB

  • 100 KB - 500 KB

  • 500 KB - 1 MB

  • 1 MB - 10MB

  • 10 MB - 50 MB

  • over 50 MB

Please note that you cannot select updates by size across the above ranges. For example, you cannot search for updates with size ranging between 100 Kbytes and 1 Mbyte.

The values for each of the following search parameters in the Search and Display Options panel comes from different sources depending on the version of the Microsoft AU (automated update) client installed on the machines the information comes from. Please note that the values for all of these fields are extracted from the software update records downloaded from Microsoft. Scrip 237 does not use or change any of these values, and they cannot be changed on the ASI server either.

Date

The Date field has two meanings, depending on the version of the Microsoft AU client. This parameter is machine-independent.

On machines where the Microsoft AU client iuengine.dll version is equal to, or lower than 5.4.3790.21, this date is the file last modified timestamp of the update installation executable. It can only be found after an update has been downloaded on the machine. After download completes, Scrip 237 reports this date to the server.

On machines where the Microsoft AU client iuengine.dll version is higher than 5.4.3790.21, this date is the last deployment changed date property of an update, retrieved by a COM API call. Scrip 237 reports this date as part of the detection process.

Platform

On machines where the Microsoft AU client iuengine.dll version is equal to, or lower than 5.4.3790.21, the value of the Platform field comes from details.xml. Usually, it's equal to ver_platform_win32_nt.

On machines where the Microsoft AU client iuengine.dll version is higher than 5.4.3790.21, Scrip 237 reports the Platform field as blank.

Component

On machines where the Microsoft AU client iuengine.dll version is equal to, or lower than 5.4.3790.21, the value of the Component field comes from details.xml. It specifies the part of the operating system the software update applies to.

On machines where the Microsoft AU client iuengine.dll version is higher than 5.4.3790.21, Scrip 237 reports the Component field as blank.

Priority

On machines where the Microsoft AU client iuengine.dll version is equal to, or lower than 5.4.3790.21, the value of the Priority field comes from details.xml. Scrip 237 does not use it.

On machines where the Microsoft AU client iuengine.dll version is higher than 5.4.3790.21, the Priority field is always 0. Scrip 237 does not use this field.

Please note that Scrip 237 does not use the value of the Priority field to prioritize the download of software updates, as the Microsoft AU client does. You have control over both the download and installation of software update via the Approve updates wizard.

Uninstallable

If an uninstall string exists for an update, then the value of the Uninstallable field is reported as true to the ASI server. The value of the Uninstallable field is machine independent. Once any machine installs an update and detects the uninstall string in the registry, it will set this value to true on the ASI server in the Microsoft Update Status page on the ASI server. The meaning of the uninstall string is the same for all versions of the Microsoft AU client.
