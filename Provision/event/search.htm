General notes

Throughout the help pages, when you encounter the word item, it refers to event filters, event notifications, event reports, asset queries, and asset reports as a group.

In all item pages, you can sort table entries by clicking on the column header corresponding to the field you want to sort entries by (e.g. Name). Clicking once sorts entries in descending order, twice sorts them in ascending order.

By default, all available search parameters are displayed in the Search Options panel on item pages. If you wish to limit the number of search parameters available for use, click on the Less button just below the Display Options panel. At this point in time, you cannot choose the search parameters available for use when clicking on the Less button.

Clicking on the Reset button on any item page will cause the same item page to be displayed in its default state. No values are assigned to any search parameters, and items are displayed by name in ascending alphabetical order, 20 per page.

Any search option parameter used to filter entries in an item page will be displayed in the item table on that page. In order to have a field displayed in the table even if you do not use it as a search parameter, or if it's a display-only field, select the (all) option from its pull down list in the Search Options panel. If you do not want a field displayed in the table, simply select (not displayed) from its pull down list in the Search Options panel. The exception to this rule is the Event Filter search parameter in the Event Reports page which when used is not displayed.

Using multiple search parameters to select items on an item page will result in a search where only items that satisfy ALL search parameter conditions will be selected (all search parameter used are chained together with AND). Increasing the number of search parameters used to select items is equivalent to refining a search. For example, you could first search for all asset reports with the word operating in their title, and then refine the search by selecting the value enabled for the State search parameter.

All searches using <field name> Contains (e.g. Name Contains) search parameters are case insensitive.

In the Event Reports and Asset Reports pages, the Filter Contains and Query Contains search parameters let you base a search on matching a string of characters in the name of a filter or query, not its content.

In all item pages, item name entries (e.g. Name and Query in the Asset Reports page) are linked to the corresponding item detail pages. For example, clicking on an entry in the Name column in the Asset Reports page will take you to the detail page for that asset report opened in a new window. There, you will find all the detail information about the report.

When they are configured to be sent by e-mail, event notifications, event reports and asset reports by default will be sent to the recipients designated as the default recipients for the user account where the event notifications, event or asset reports are being produced. Default recipients for a user account are configured using the Admin module reachable by clicking on the admin link in the tools navigation bar on any page on the ASI server. You can also configure event notifications, event or asset reports to be sent to recipients other than the default recipients. These are called custom recipients. In all item pages, parameters related to default e-mail recipients are labeled Default Recipients. Parameters related to custom recipients are labeled either E-mail Recipients, or Recipients.



Search option parameters

Name Contains átop

In the Name Contains search parameter, you can enter a single string of alphanumeric characters. The search performed will be a non-case sensitive sub-string match. Name is the field that contains the name of an event filter.

Scope átop

The Scope search parameter lets you select event filters based on their scope. The scope of an item controls the availability of the item to one or all users. Global items are available to all users and can be created only by users with global rights. Local items are available only to the users that created them. When a user edits a global item he/she does not own, a local copy of that item is automatically created containing the user's edits. If that local item is subsequently deleted, the global item by the same name becomes once again available to the user that had created the local item.

The Scope search parameter can have the values listed below.

(not displayed) (all) Local Global

Filter átop

Filter is a display-only field. It contains the clause(s) used to retrieve events of interest. You can choose to have this field displayed in the Event Query Filters page.

(not displayed) (all)

Owner átop

Owner is a display-only field. It contains the user that created the item. You can choose to have this field displayed in the Event Query Filters page.

(not displayed) (all)

Filter Contains átop

In the Filter Contains search parameter, you can enter a single string of alphanumeric characters. The search performed will be a non-case sensitive sub-string match. Filter is the field that contains the event selection clause(s) specified in an event filter.
