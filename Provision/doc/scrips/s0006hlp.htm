Scrip 6 - Memory Statistics

Scrip 6 reports a system's memory utilization statistics on demand, or on a schedule for each of the following classes of memory:

  • Physical memory

  • Virtual memory

  • Swap space

the following utilization statistics are reported:

  • Total (KBytes)

  • In use (KBytes)

  • Percentage in use (%)

  • Free (KBytes)

  • Percentage free (%)

In addition, the following statistic is reported:

  • Page reads per second

Note that memory utilization measurements are made at the time the Scrip is run. However, the "page reads per second" statistic must be averaged over a specific time period. That time period is one of the Scrip 6's configuration parameters, in mSec, with the default value set at 15000 Msec.

Scrip 6 can be used as an automated early warning system for low memory conditions. For page reads per second you can set a threshold that will trigger a Scrip log whenever page reads per second exceed it. This is recognized as one of the better indicators of degraded system performance due to low memory conditions.

In addition, for each type of memory monitored, users can set available memory thresholds as a percentage of the total amount of memory. When available memory falls below a threshold, a Scrip log is automatically created and uploaded to the log server. A support provider can then set up a notification that scans the log database every ten minutes and notifies him/her whenever a Scrip 6 log is found signaling a low-memory condition.

The table below describes the information contained in the logs produced by this Scrip.


Client time:

Date and time at which event was detected by the Client on the end-user system (e.g. 2001-03-02 19:21:36)

Server time:

Date and time at which event log was received and stored in the SQL database on the Web log server (e.g. 2001-03-02 19:21:53)

Scrip number:

Number of Scrip (no particular relevance)

Customer:

Name of supported customer, individual department or site

Machine:

Name of system as assigned at the time of Windows configuration

Username:

Name of user when event occurred

Client version:

Client version number (e.g. 0.700.2048.LE)

Client size:

Size of Client in bytes

Priority:

Priority of event ranging between 1 (lowest) and 9 (highest)
