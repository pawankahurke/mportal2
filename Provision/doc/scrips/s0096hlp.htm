Scrip 96 - Processor Statistics

Scrip 96 reports a system's processor statistics on demand, or on a schedule. Some of these statistics are available for both MS Windows NT based systems (NT4, 2000, XP, or Server 2003) and MS Windows 9x based systems (Windows 95/98/Me), and some are only available for MS Windows NT based systems.

On both MS Windows NT, and MS Windows 9x based systems, the following statistic is reported:

  • Processor utilization (%)

On MS Windows NT based systems only, the following statistics are reported:

  • Interrupt rate (per second)

  • Average queue length (items)

Note that all of these statistics must be averaged over a specific time period called Sampling Period. That time period is fixed for MS Windows 9x based systems. For MS Windows NT based systems, it's one of Scrip 96's configuration parameters.

The table below describes the information contained in the logs produced by this Scrip.


Client time:

Date and time at which event was detected by the Client on the end-user system (e.g. 2001-03-02 19:21:36)

Server time:

Date and time at which event log was received and stored in the SQL database on the Web log server (e.g. 2001-03-02 19:21:53)

Scrip number:

Number of Scrip (no particular relevance)

Customer:

Name of supported customer, individual department or site

Machine:

Name of system as assigned at the time of Windows configuration

Username:

Name of user when event occurred

Client version:

Client version number (e.g. 0.700.2048.LE)

Client size:

Size of Client in bytes

Priority:

Priority of event ranging between 1 (lowest) and 9 (highest)

Description:

Description of event that took place (e.g. symptom detected, problem resolved or automated procedure executed)

Type:

Dialog box type (Error, Warning, Information, Question)

n/a

n/a

n/a

n/a
