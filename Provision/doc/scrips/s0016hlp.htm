Scrip 16 - File / Folder Deletion Dialog Box Creation

Scrip 16 detects and reports a user's attempt to delete MS Internet Explorer-related history or cookie files.

It does not report whether the attempted deletion was completed or the user chose to cancel the operation.

Scrip 16 will not detect an attempted deletion of the entire Internet Explorer History or Cookie folders located in the MS Windows directory. It would be detected by Scrip 21.

This Scrip is triggered by one of the following actions:

  • Attempted deletion of an Internet Explorer history file located in the folder \Windows\History accessed directly either through "My Computer" or Windows Explorer.
  • Attemtped deletion of an Internet Explorer history file accessed through Internet Explorer, by clicking on View, Explorer Bar, History, or by clicking on the History icon on the Internet Explorer toolbar.
  • Attempted deletion of an Internet Explorer cookie file accessed through Internet Explorer, by clicking on Tools, Internet Options, Settings, in the Temporary Internet Files section, and View Files.

The table below describes the information contained in the logs produced by Scrip 16.


Client time: Date and time at which event was detected by the Client on the end-user system (e.g. 2001-03-02 19:21:36)
Server time: Date and time at which event log was received and stored in the SQL database on the Web log server (e.g. 2001-03-02 19:21:53)
Scrip number: Number of Scrip (no particular relevance)
Customer: Name of supported customer, individual department or site
Machine: Name of system as assigned at the time of Windows configuration
Username: Name of user when event occurred
Client version: Client version number (e.g. 0.700.2048.LE)
Client size: Size of Client in bytes
Priority: Priority of event ranging between 1 (lowest) and 9 (highest)
Description: Description of event that took place (e.g. symptom detected, problem resolved or automated procedure executed)
