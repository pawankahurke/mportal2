Scrip 17 - Executable Detected

Scrip 17 detects and reports the execution of programs included in the Executables to detect list in its configuration file.

In some cases, executables whose execution you may want to detect are entered as parameters in the command line of a script or another executable.

For this reason, you can also enter the name of the executable in the Command Line arguments to detect list in Scrip 17's configuration file.

Note that Command Line arguments to detect entries don't have to be executables. You could use this parameter to enter any sequence of characters in the command line of an executable you want the Scrip to detect.

The table below describes the information contained in the logs produced by this Scrip.


Client time: Date and time at which event was detected by the Client on the end-user system (e.g. 2001-03-02 19:21:36)
Server time: Date and time at which event log was received and stored in the SQL database on the Web log server (e.g. 2001-03-02 19:21:53)
Scrip number: Number of Scrip (no particular relevance)
Customer: Name of supported customer, individual department or site
Machine: Name of system as assigned at the time of Windows configuration
Username: Name of user when event occurred
Client version: Client version number (e.g. 0.700.2048.LE)
Client size: Size of Client in bytes
Priority: Priority of event ranging between 1 (lowest) and 9 (highest)
Description: Description of event that took place (e.g. symptom detected, problem resolved or automated procedure executed)
Type: n/a
Path: n/a
Executable: n/a
Version: Version of program whose execution was reported

(e.g. 5.00.2134.1)

Size: Size of program whose execution was reported in bytes

(e.g. 72464)

ID: n/a
Window title: n/a
String 1: Creation date of of program whose execution was reported

(e.g. Monday, December 31, 1979 19:00:00)

String 2: Client location on system that reported the event

(e.g. C:\PROGRAM FILES\...\CLIENT\CUST.EXE)

Text 1: n/a
Text 2: n/a
Text 3: n/a
Text 4: Location of program whose execution was reported

(e.g. Executable File: C:\WINNT\regedit.exe)
