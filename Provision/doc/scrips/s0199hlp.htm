Scrip 199 - Registry Change Detected

Scrip 199 detects and reports changes made to the registry. It covers all keys listed in the Registry keys to monitor parameter in the configuration entry for this Scrip, and their sub-keys.

Most changes made to applications' and the system's configuration and environment result in registry changes. This means that the registry changes frequently. To avoid generating a large volume of event logs, most of which are not very useful, it is important to be as specific as you possibly can when identifying the registry keys you would like Scrip 199 to monitor.

For example, if you want to monitor changes in Outlook Express account settings, it is probably more advisable to select the key for specific accounts (e.g. HKEY_CURRENT_USER\Software\Microsoft\Internet Account Manager\Accounts\00000001), rather than the entire Internet Account Manager key (HKEY_CURRENT_USER\Software\Microsoft\Internet Account Manager).

The table below describes the information contained in the logs produced by this Scrip.


Client time: Date and time at which event was detected by the Client on the end-user system (e.g. 2001-03-02 19:21:36)
Server time: Date and time at which event log was received and stored in the SQL database on the Web log server (e.g. 2001-03-02 19:21:53)
Scrip number: Number of Scrip (no particular relevance)
Customer: Name of supported customer, individual department or site
Machine: Name of system as assigned at the time of Windows configuration
Username: Name of user when event occurred
Client version: Client version number (e.g. 0.700.2048.LE)
Client size: Size of Client in bytes
Priority: Priority of event ranging between 1 (lowest) and 9 (highest)
Description: Description of event that took place (e.g. symptom detected, problem resolved or automated procedure executed)
Type: Dialog box type (Error, Warning, Information, Question)
