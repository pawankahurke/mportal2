Scrip 177 - Scrip Configuration Update

Scrip 177 retrieves Scrip configuration changes made using the server based ASI site management facility, and propagates them to all systems on the local network.

Here is how it works:

  • It makes a call to the server where the ASI site management facility is located, passing it the client revision level, the PS revision level, the machine name, and the site name.
  • The ASI site management facility returns an indication of what to do. It will tell the client whether or not the client revision level, and the PS revision level matches. After receiving this information, the ASI client takes the following two steps, in order:
    • If the client revision level doesn't match, the ASI client sends the Scrip templates to the ASI site management facility.
    • If the PS revision level doesn't match, the ASI client performs a Scrip configuration update operation retrieving the updated Scrip configuration(s) from the ASI site management facility and propagating it/them to all systems on the local network.
