Scrip 26 - Executable Usage Profiler

Scrip 26 monitors the programs executed on a system on the local network from the time it was first enabled.

The list of the processes compiled by Scrip 26 can be used to identify changes in a system's configuration brought about by the installation of applications that don't use common installation executables.

This list would also be useful in identifying and analyzing system useage patterns.

The first time execution of a program is detected by Scrip 26, an event is logged on the log database.

The table below describes the information contained in this log.

Another log of the execution of this program is recorded on the log database if its execution takes place more than 30 days since the last time it was executed.

The entry for a program in the list of executables maintained by Scrip 26 contains the last time execution of the program was detected by the Scrip.


Client time: Date and time at which event was detected by the Client on the end-user system (e.g. 2001-03-02 19:21:36)
Server time: Date and time at which event log was received and stored in the SQL database on the Web log server (e.g. 2001-03-02 19:21:53)
Scrip number: Number of Scrip (no particular relevance)
Customer: Name of supported customer, individual department or site
Machine: Name of system as assigned at the time of Windows configuration
Username: Name of user when event occurred
Client version: Client version number (e.g. 0.700.2048.LE)
Client size: Size of Client in bytes
Priority: Priority of event ranging between 1 (lowest) and 9 (highest)
Description: Description of event that took place (e.g. symptom detected, problem resolved or automated procedure executed)
Type: n/a
Path: Location of program whose execution is being monitored

(e.g. C:\PROGRAM FILES\INTERNET EXPLORER\IEXPLORE.EXE)

Executable: Name of program whose execution is being monitored

(e.g. IEXPLORE.EXE)

Version: Version of program whose execution is being monitored

(e.g. 5.50.4522.1800)

Size: Size of program whose execution is being monitored

(e.g. 60688)

ID: n/a
Window title: n/a
String 1: Creation date of program whose execution is being monitored

(e.g. Thursday, February 1, 2001 23:00:00)

String 2: Client location on system that reported the event

(e.g. C:\PROGRAM FILES\...\CLIENT\CUST.EXE)

Text 2: n/a
