Scrip 191 - TCP/IP Connectivity Problem Management

Scrip 191 detects, diagnoses, and manages the resolution of TCP/IP connectivity problems.

Failure points covered by Scrip 191 include:

  • Local connectivity
  • Internet access gateway
  • DNS server
  • Target service port - Scrip 191 covers ALL TCP/IP ports / services

Scrip 191 uses the ASI network driver to detect and diagnose TCP/IP connectivity problems on all ports, at a low level before they are detected by application software.

If the port of the service which failed is included in the User Interface enabled on failure of these ports list, Scrip 191 will inform the end-user about the problem with three dialog boxes:

  • One dialog box that informs the user that a connectivity problem occurred
  • A second dialog box that tells the end-user which problem occurred, and offers to notify him/her when the service is available again, and
  • If the end user clicked on the "OK" button in the second dialog box, a third dialog box that will notify the user that the failed service is back online.

The number of notifications about the availability of a service generated by Scrip 191 is a configurable parameter you can set in the Scrip's configuration page. If the service whose availability is being monitored is not back online when the number of notifications is exceeded, Scrip 191 will display a last message with instructions to the end-user on what to do next. The content of this message is also a configurable parameter that you can access via the Scrip's configuration page.
